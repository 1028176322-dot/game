## 图例 / 状态定义（全文统一）

为避免"缺失 / 未实现 / 未接入 / 未挂载"混用，全文统一如下四类状态：

| 符号 | 状态 | 含义 |
|---|---|---|
| ✅ | 已实现并接入 | 类/接口已真实实现，且已注册并由运行时实际使用 |
| 🟡 | 已实现未接入 | 类/接口已实现（或已注册），但**未真正接入运行时节点/调用链** |
| 🟠 | 已接入但未完成 | 仍被运行时使用，但**尚未退化为设计要求的壳层 / 未完成组件化** |
| 🔴 | 未实现 | 代码中不存在对应实现（仅令牌、注释或规划） |

一致性等级（替代无依据的"百分比"，用于总体结论与完成度概览）：

| 等级 | 星标 | 含义 |
|---|---|---|
| 高度一致 | ★★★★★ | 设计要素已完整实现并接入运行时 |
| 中度一致 | ★★★★☆ | 主要实现，少量接入或接口缺口 |
| 部分一致 | ★★★☆☆ | 实现存在但未接入/未壳化，或有关键缺口 |
| 基本未完成 | ★★☆☆☆ | 仅结构占位、或完全未实现 |

> 说明：凡可精确计数的项，以 `分子/分母` 形式给出（如 `11/12` 服务已注册）；无法计数的主观判断以等级表达，避免无依据的百分比。
