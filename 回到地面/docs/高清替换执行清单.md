## 资源分级说明

| 等级 | 含义 | 预估数量 | 处理动作 |
|------|------|:--------:|----------|
| **S** | 玩家每局必看：角色/Boss/核心HUD/主界面 | ~80 | `replace` 第一批高清替换 |
| **A** | 高频UI/图标/技能/商店 | ~120 | `replace`/`resize_export`/`merge_atlas` |
| **B** | 低频怪物/特效/状态图标 | ~150 | `keep`/`defer`，按需处理 |
| **C** | raw/bak/test/无用/占位 | ~60 | `move_source`/`delete`，清理 |

### 7 种处理动作

| 动作 | 含义 | 示例 |
|------|------|------|
| `replace` | 同名高清替换 | 角色 idle、Boss attack、主界面背景 |
| `keep` | 保留现状 | 已清晰/体积合理/低频但仍使用 |
| `merge_atlas` | 合并图集 | 小图标、按钮状态、同类 UI 零件 |
| `resize_export` | 从母版导出运行尺寸 | 1024 图标只显示 96px → 导出 256 |
| `move_source` | 移出运行目录 | AI 母版、raw、PSD、bak |
| `delete` | 删除 | 测试图、无引用占位图、废弃资源 |
| `defer` | 暂缓 | 低频资源，等包体稳定后再处理 |

## 阶段 3：S 级 — 核心战斗高清替换 (5项 · ~2h) 🔴

**范围：**
```
characters/      (5角色×7动作 = 35帧，48px → 192/256px)
monsters/S级     (每个区域精选 2-3 种常见怪优先替换)
bosses/          (miniboss 30 + finalboss 30帧，64px → 192/256px)
核心 HUD/技能图标  (玩家每局必看的部分)
```

**规格升级目标：**

| 类型 | 当前单帧 | 替换尺寸 |
|------|:--------:|:--------:|
| 普通角色 | 48px | 192px |
| 重要角色 | 48px | 256px |
| 普通怪物 | 48/64px | 128px |
| 精英/Boss | 64/96px | 192/256px |

**执行项：**

- [ ] **3-1** 角色高清化（AI 母版 1024+ → 运行时 192-256px，严格等比 4x）
- [ ] **3-2** 怪物高清化（S 级常见怪优先，每个区域 2-3 种）
- [ ] **3-3** Boss 高清化（先替换 idle/attack/skill，再替换 death/phasechange）
- [ ] **3-4** 核心 HUD/技能图标高清化
- [ ] **3-5** 验收：10 项标准（Cocos 导入/动画/碰撞/血条/包体/内存）

> **关键约束**：同角色所有动作保持同单帧画布；保持帧数和排列方向；保留 `.meta`

## 每批替换的 10 项验收标准

1. Cocos Console 无导入错误
2. `.meta` UUID 未大面积重建
3. 场景/Prefab 无 Missing SpriteFrame
4. 角色动画帧数正确，无跳帧
5. 脚底/影子/血条位置正常
6. 怪物/Boss 战斗中大小符合设计
7. UI 图标在高 DPR 设备上清晰
8. 背景无强拉伸
9. 透明边缘无黑边/白边
10. 包体/内存无异常暴涨
