# 跑马灯系统 · 详细规格

> **版本**: v1.0 | **日期**: 2026-06-25 | **设计者**: GameDesigner
> **系统层**: 变现层 P3 | **依赖**: 战斗系统 (战斗结束事件)
> **变现模式**: 激励广告 (每点亮 1 格看 1 次广告)

---

## 1. 设计目的

**玩家体验**: "反正战斗结束了, 顺手点一下, 不亏"

**核心机制**: 战斗结束后, 显示一个 3 格进度条。每点亮 1 格需要看一个激励广告。3 格全亮 → 免费获得 1 把钥匙。

**垃圾时间漏斗**:
```
战斗结束 → 等待结算动画 (3 秒)
    ↓
跑马灯弹出 → 玩家: "点一下广告试试?"
    ↓
如果是 → 看 30 秒广告 → 点亮 1 格
    ↓
如果不是 → 关闭 → 继续前进
    ↓
下次战斗结束 → 再次弹出
    ↓
3 格全亮 → 获得 1 把钥匙!
```

## 2. UI/交互

### 2.1 界面布局

```
战斗结束后, 底部弹出:

┌──────────────────────────────────┐
│                                  │
│   宝箱收集进度                    │
│                                  │
│   [■] [■] [□]   2/3             │
│                                  │
│   再点亮 1 格就能领 1 把钥匙!      │
│                                  │
│   ┌──────┐  ┌──────┐            │
│   │ 点亮  │  │ 跳过  │           │
│   │ 看广告 │  │      │           │
│   └──────┘  └──────┘            │
│                                  │
└──────────────────────────────────┘

交互:
  - 点击"点亮" → 播放 30 秒激励广告 → 格子变亮
  - 点击"跳过" → 关闭面板, 该格保留 (下次可继续点)
  - 3 格全亮 → 自动弹出"恭喜获得钥匙!" → 钥匙加入背包
```

### 2.2 状态矩阵

| 格数 | 显示 | 可操作 |
|------|------|--------|
| 0/3 | [□][□][□] | 任一点亮 |
| 1/3 | [■][□][□] | 点第 2 格 |
| 2/3 | [■][■][□] | 点第 3 格 (用户接近奖励)|
| 3/3 | [■][■][■] → 领奖 | 自动弹出奖励 |

## 3. 进度规则

```
进度积累:
  - 每次战斗结束 → 进度 +1
  - Boss 战斗结束 → 进度 +1 (同上)
  - 每层进入新楼层 → 进度不清零 (跨层积累)
  - 死亡/觉悟结算 → 进度不清零 (保留到下一局?)
  
→ 决策: 跑马灯是否跨局? 
  建议: 不跨局。因为"上一局没点完的广告, 下一局接着点"
  会降低玩家"必须点完"的动力。
  
  规则: 本局内跨层积累, 但结算/死亡后清零。
```

---

## 4. 广告 fallback

```
点亮失败: 
  广告加载失败 → 按钮显示"加载失败, 重试", 保留当前进度
  3 次重试均失败 → 自动跳过错失该格, 保留其他格进度
  
断网状态:
  检测到断网 → 按钮置灰, 显示"网络不可用"
  恢复网络 → 按钮恢复可点击
  
玩家拒绝广告推送:
  微信允许关闭个性化广告
  → 仍然正常显示广告, 只是内容非个性化
  → 不影响收益
```

## 5. 变现预期

```
每局战斗数: 约 8~12 场 (3 层)
每局跑马灯触发: 约 8~12 次

如果每次战斗后都弹出:
  - 前期 (1-2 层): 玩家好奇, 点击率高 (60%)
  - 后期 (3 层+): 玩家想快点过关, 点击率下降 (30%)

预期点击率: 约 40%
平均每局点亮格数: 约 3~4 格 (看 3~4 次广告)
每 2~3 局获得 1 把钥匙 (3 格 = 1 钥匙)

收入影响:
  每局额外 3~4 次广告观看
  对"人均 6 次/日"目标贡献约 50%
```
