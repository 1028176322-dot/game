> **版本**: v2.1 | **日期**: 2026-07-09 | **设计者**: GameDesigner
> **所属项目**: 回到地面 (BackToGround) | **系统层**: 核心层 P0

## 0. 设计师笔记

### 0.1 玩家体验目标

```
进入房间 → 怪物蜂拥而至 → 英雄自动攻击最近的敌人
    ↓
玩家控制移动走位 —— 躲技能、卡地形、找角度
    ↓
看到敌人头顶跳出"爆燃!"、"超导!"等元素反应
    ↓
血量危险 → 走位撤退 / 喝药 / 放技能控场
    ↓
怪物全清 → 捡宝 → 前往下一房间
```

**核心感受**：眼睛盯着战场，手指控制走位，大脑在规划"下一个技能怎么放"——**一手操作的流畅 + 一手构筑的快感**。

### 0.2 好玩假设

> **"自动攻击解放操作，让玩家专注于走位和技能时机的判断"**

实时自动攻击的核心乐趣不是"点得多快"，而是：
1. **走位决策**：往哪躲、站哪打、怎么利用地形
2. **Build 驱动的战斗风格**：你的装备和技能决定了自动攻击的表现形式
3. **元素反应的自然触发**：边走边打，combo 自己发生

### 0.3 与 AP 回合制的核心差异

| 维度 | AP 回合制 (v1.0) | 实时自动攻击 (v2.0) — **当前** |
|------|-----------------|-------------------------------|
| 战斗节奏 | 玩家思考→行动→等待 | **实时持续** —— 没有"等待" |
| 操作重心 | 分配 AP 给移动/攻击/技能 | **走位 + 时机性技能释放** |
| 决策密度 | 每回合 1 次大决策 | **每秒 1 次小决策**（往哪走）|
| 技能使用 | AP 够就能放 | **有冷却 CD**，需要判断时机 |
| 防御方式 | 主动按防御按钮 | **走位躲避**（规避伤害）|
| 战斗时长 | 5~15 回合（3~8 分钟）| **30~90 秒** 一场战斗 |

## 2. 自动攻击系统

### 2.1 Mechanic: 自动攻击

```
Mechanic: 自动攻击

Purpose: 玩家不操作时也能造成伤害，让"出手"这件事自动化，
         玩家只需关注走位和技能时机

Player Fantasy: "我的角色自己就在战斗，我是战场指挥官"

Input: 无（系统自动执行）
Output: 每隔 1.0 秒，对当前锁定目标造成一次普攻伤害

Success Condition: 攻击命中目标，伤害正常计算
Failure State: 目标被障碍物完全遮挡 → 攻击被阻挡（弹道消失）
               目标死亡 → 自动切换到最近敌人

Edge Cases:
  - 当前锁定目标死亡 → 自动切换至最近敌人（0.1 秒内完成）
  - 房间内无敌人 → 停止自动攻击
  - 攻击目标超出 Range → 角色自动追击（但玩家可拉回走位）
  - 玩家在翻滚中 → 翻滚期间自动攻击不停止
  - 距离越远命中率越低 → 超出 Range 后每格 -20% 命中率

Tuning Levers:
  - 攻击频率 (当前: 1.0 秒/次)
  - 距离命中衰减 (当前: 每超 Range 1 格 -20%)
  - 自动索敌范围 (当前: 全房间可见)

Dependencies: 伤害公式, 元素系统
```

### 2.2 自动攻击的 Build 变化

自动攻击的基础行为可以被 Build 系统改变：

| 核心能力 | 对自动攻击的影响 |
|----------|----------------|
| 二段斩 | 每次攻击额外攻击 1 次（60% 伤害）|
| 弹射箭 | 攻击弹射至第 2 个目标（50% 伤害）|
| 吸血光环 | 每次攻击回复 15% 伤害量 |
| 元素附魔 | 每次攻击附带对应元素状态 |

**核心**：自动攻击不是"无聊的挂机"——你每 1 秒看到的是你的**Build 在表演**。

## 3. 主动技能系统

### 3.1 技能总表

技能不再消耗 AP，而是有**冷却时间 (CD)**：

| 技能 | CD | 效果 | 适用场景 |
|------|----|------|----------|
| 冲刺冲锋 | 5 秒 | 快速移动 3 格，路径上敌人受 50% 伤害 | 追击/逃跑/穿怪 |
| 元素爆发 | 8 秒 | 对周围 2 格所有敌人造成 150% ATK 的元素伤害 | 被包围时清场 |
| 护盾 | 6 秒 | 生成持续 2 秒的护盾，吸收 10 点伤害 | 预判高伤害技能 |
| 治疗波 | 10 秒 | 回复 25% HP + 清除 1 个负面状态 | 残血保命 |
| 减速领域 | 7 秒 | 在当前位置生成持续 3 秒的减速场 | 风筝远程怪 |
| 锁定射击 | 4 秒 | 对当前目标造成 200% ATK 的必中伤害 | 快速斩杀残血怪 |

**玩家同一时间最多携带 2 个主动技能**（Build 决定）。技能通过以下方式获得：
- 角色初始自带（如战士带护盾，弓手带锁定射击）
- 强化房选择（从 3 个技能中选 1 个）
- 遗物提供（如"回响之珠"使治疗波 CD 减半）

### 3.2 Mechanic: 技能释放

```
Mechanic: 技能释放

Purpose: 给玩家在自动攻击之外的"操作时刻"——决定什么时候放大招

Player Fantasy: "就是现在！"——时机很重要

Input: 点击对应技能按钮
Output: 技能效果立即触发（可能有动画前摇 0.2 秒）
        技能进入 CD

Success Condition: 技能效果生效
Failure State: 技能 CD 中 → 按钮显示 CD 倒计时，不可点击
               技能释放时玩家死亡 → 技能中断

Edge Cases:
  - 治疗波在满血时使用 → 提示"HP 已满"，不消耗 CD
  - 冲刺冲锋撞到障碍物 → 停在障碍物前，不穿墙
  - 护盾在被攻击瞬间开启 → 护盾吸收本次伤害
  - 元素爆发范围内无敌人在技能动画后又走入 → 不会发生（即时检测）
  - 技能 CD 期间升级获得该技能 → 重置 CD

Tuning Levers:
  - 各技能 CD 时长
  - 技能伤害倍率
  - 技能范围

Dependencies: Build 系统
```

## 4. 翻滚/闪避系统

### 4.1 Mechanic: 翻滚闪避

```
Mechanic: 翻滚闪避

Purpose: 玩家唯一主动的防御手段——不是按防御按钮，而是靠操作走位

Player Fantasy: "看到了，躲开了"——操作感的核心来源

Input: 双击空地 / 点击翻滚按钮
Output: 角色沿移动方向翻滚 1 格，耗时 0.5 秒，
        期间前 0.3 秒为无敌帧（无视所有伤害）
        翻滚 CD 3 秒

Success Condition: 翻滚期间避开了一次致命攻击
Failure State: 滚动撞到障碍物 → 停在障碍物旁
                翻滚结束后立即受到伤害（无敌帧已过）

Edge Cases:
  - 连续翻滚 → 不能（CD 3 秒）
  - 翻滚途中被阻断 → 不会被阻断，翻滚强制完成
  - 翻滚进荆棘地板 → 翻滚后触发荆棘伤害（在无敌帧之后）
  - 翻滚进治疗符文 → 翻滚后触发治疗（同上）

Tuning Levers:
  - 无敌帧时长 (当前: 0.3 秒)
  - 翻滚 CD (当前: 3 秒)
  - 翻滚距离 (当前: 1 格)

Dependencies: 网格系统
```

## 5. 伤害公式

### 5.1 基础公式（与 v1.0 相同）

```
原始伤害 = ATK(攻击者) + 骰子(1~6)
减伤值  = floor(DEF(目标) × 0.5)
实际伤害 = max(原始伤害 - 减伤值, 1)

// 暴击判定
if (骰子结果 == 6):
    实际伤害 = floor(实际伤害 × 1.5)
```

### 5.2 距离衰减（新增）

```
if 攻击者与目标的距离 > 攻击者 Range:
    命中率 = 100% - (超出格数 × 20%)
    // 命中失败时：提示"MISS"，无伤害，不触发元素附着
```

**设计意图**：近战玩家必须贴身打，远程玩家不能太远白嫖。1 格≈安全，2 格≈还行，3 格≈大概率 miss。

### 5.3 验证场景

| 场景 | ATK | DEF | 距离 | 骰子 | 实际伤害 |
|------|-----|-----|------|------|---------|
| 近战打贴身怪 | 5 | 0 | 1(=Range) | 4 | 9 |
| 近战打 2 格怪 | 5 | 0 | 2(>Range) | 4 | MISS(60%命中) |
| 远程打 3 格怪 | 6 | 1 | 3(=Range) | 5 | 10 |
| Boss 打近战玩家 | 8 | 3 | 1 | 4 | 9 |
| 远程 Boss 打玩家 | 7 | 2 | 4(>Range) | 3 | MISS |

## 6. 地形系统（适配实时战斗）

### 6.1 地形总表

| 地形 | 通行 | 效果 | 实时战斗中的行为 |
|------|------|------|-----------------|
| 空地 | 可 | 无 | — |
| 障碍物 | 不可 | 阻挡移动和弹道 | 弹射无法穿越，玩家需绕路 |
| 荆棘地板 | 可 | **站在上面每秒扣 1 HP** | 每次触发伤害间隔 1 秒 |
| 治疗符文 | 可 | **站在上面每秒回复 2 HP** | 每次回复间隔 1 秒 |
| 高台 | 可(减速) | **站在上面 Range +1，但移速 -30%** | 射程变长但变成活靶子 |
| 暗区 | 可 | **进入后自动隐身 2 秒**（离开后消失） | 怪物丢失目标，转向攻击其他 |

### 6.2 地形与战斗的交互

```  
实时战斗中，地形效果按时间 tick 检测（每 0.5 秒）:

tick 事件:
  - 检测每个角色当前所在格子
  - 如果在治疗符文上 → 回复 2 HP
  - 如果在荆棘地板上 → 扣 1 HP
  - 如果在暗区 → 标记为隐身状态
  - 如果在高台 → 标记为高台状态（Range+1，移速-30%）

地形 tick 与攻击/技能完全独立 —— 你站着不动就回血/扣血，
移动是避开还是利用地形，是全凭玩家走位控制的。
```

## 7. 怪物 AI（实时版）

### 7.1 AI 主循环（实时）

```
每个怪物运行独立的 AI 协程，每 0.5 秒决策一次:

loop:
    wait 0.5s
    
    // 1. 检查自身状态
    if stunned: skip this tick
    if still in attack animation: skip this tick
    
    // 2. 更新目标
    target = getNearestPlayer()  // 默认总是追玩家
    
    // 3. 选择并执行行为
    distance = getDistanceTo(target)
    
    if distance <= attackRange:
        attack(target)
    elif distance <= moveRange:
        moveToward(target)
    else:
        charge(target)  // 快速接近
```

### 7.2 6 种 AI 行为（适配实时）

| AI 类型 | 行为 | 实时战斗中的表现 |
|--------|------|-----------------|
| 冲锋型 | 持续追玩家，近身猛攻 | 全程追着你，攻击频率高(0.8秒/次)|
| 远程型 | 保持 2~3 格距离射击 | 你靠近它后退，你后退它追到射程边缘 |
| 防御型 | 原地架盾，偶尔攻击 | 站在关键位置当肉盾，需要绕后 |
| 召唤型 | 每 4 秒召唤 1 个小怪，上限 3 只，总数 > 8 时禁召 | 不先杀它就会越来越多人海，但有上限 |
| 自爆型 | 进入房间后 5 秒自爆 | 倒计时——5,4,3,2,1 ——躲开！|
| 精英型 | 混合 2 种行为+BOSS技能 | 每个 Boss 单独设计 |

### 7.3 实时战斗的怪物决策差异

```
自爆型在实时模式下的设计:
  - 进入战斗后显示倒计时 5 秒（屏幕中间大字 5...4...3...）
  - 自爆者全身变红，向玩家冲刺（加速 50%）
  - 倒计时结束时 → 范围 2 格爆炸，伤害 = 剩余 HP × 2
  - 玩家选择: ① 5 秒内击杀它 ② 翻滚躲开爆炸 ③ 跑出 2 格范围

远程型在实时模式下的设计:
  - 始终试图保持 3 格距离
  - 玩家靠近 → 后退（玩家追不上就别追了，换目标）
  - 玩家远离 → 追击到射程内
  - 射击时会预判玩家走位（而不是打当前位置）
```

### 8.1 状态机

```
[探索中] ──进入房间──→ [战斗开始]
                            │
                            ├─→ [实时战斗中] ←───────────┐
                            │    │ 自动攻击 每1秒         │
                            │    │ 怪物AI 每0.5秒        │
                            │    │ 地形tick 每0.5秒      │
                            │    │ 玩家走位 自由          │
                            │    │ 技能CD 独立倒计时       │
                            │    │ 元素检测 每0.5秒       │
                            │    │                        │
                            │    ├──全清──→ [房间清空奖励]  │
                            │    │              │          │
                            │    ├──死亡──→ [觉悟战暂停]    │
                            │    │              │ 复活?    │
                            │    │              ├──是──→ ──┘
                            │    │              │
                            │    │              └──否──→ [结算]
                            │    │
                            │    └──玩家出门?─→ [逃跑成功]
                            │                    (弃战，无奖励)
                            │
                            └──────────────────────────────┘
```

### 8.2 觉悟战（实时版适配）

```

Mechanic: 觉悟战（实时模式）

Purpose: 与回合制相同——死亡不是终点，是选择。
         实时模式下，HP=0 时游戏立即暂停（不像回合制还有怪物行动待处理）

Input: 玩家 HP=0
Output: 游戏暂停 → 弹出觉悟战界面
        选项 A: 看 30s 广告 → HP 恢复到 50%，在当前房间继续战斗
        选项 B: 不接受 → 进入觉悟结算

Edge Cases（实时模式特有）:
  - 玩家在翻滚中被杀死 → 回放显示"最后一击来自哪里"
  - 复活后所有怪物位置不变，但重置仇恨（怪物重新锁定)
  - 复活后有 1 秒无敌时间（防止一复活又死）
  - 多只怪物同时攻击导致死亡 → 觉悟战中显示击杀回放
```

| 普通战斗活过率 | > 85% |
| Boss 战胜率 | 50~60% |
| 无重伤通关率（HP>50%）| < 30% |
| 平均每局死亡次数 | 1~2 次 |
| 翻滚使用率 | 每次战斗至少使用 1 次 |

### 9.3 [HYPOTHESIS] 平衡假设

```
1. 自动攻击 1.0 秒间隔
   → 预期: 玩家 ATK=4, 平均伤害 ≈ 7.5, 小怪 HP=6 → 约 2 次攻击击杀
   → 击杀时间 ≈ 2 秒 — 太快? 太慢?
   → [HYPOTHESIS] 需要 feel test

2. 翻滚无敌帧 0.3 秒
   → 预期: 玩家用翻滚规避一次 Boss 重击
   → 0.3 秒 ≈ 人的平均反应时间 (200~300ms)
   → [HYPOTHESIS] 合理，需要练习但可达

3. 技能 CD 4~10 秒
   → 预期: 一场 60 秒战斗，每个技能可用 6~15 次
   → CD 太短 → 技能变成自动循环，降低操作感
   → CD 太长 → 玩家舍不得用
   → [HYPOTHESIS] 4~10 秒范围合理，具体数值依技能调整
```

## 10. 战斗系统的"破解"标准（实时版）

| 破坏模式 | 判定标准 | 严重等级 |
|----------|----------|----------|
| **无限翻滚** | 翻滚 CD 为 0，无限无敌 | ❌ 灾难 |
| **原地站撸通关** | 不动也能过任何战斗（吸血太强）| ⚠️ 严重 |
| **技能无限连** | CD 减少叠加到 0 秒 | ❌ 灾难 |
| **怪物打不到玩家** | 玩家用障碍物无伤卡怪 | ✅ 可接受(策略) |
| **自爆必杀** | 自爆伤害高到 100% 秒杀 | ⚠️ 严重 |
| **自动索敌对玩家不利** | 自动攻击锁定错误目标导致死亡 | ❌ 灾难 |
| **翻滚穿墙** | 翻滚穿过了障碍物 | ⚠️ BUG |
| **房间战斗超过 3 分钟** | 伤害太低/Build 太差 | ⚠️ 严重(软锁)|

## 11. Playtest 计划（实时版）

### 11.1 原型测试

```
目标: 验证"自动攻击+走位"的手感是否流畅

准备:
  - 使用现有游戏引擎或原型工具（如 GameMaker/Unity 原型）
  - 2D 俯视图，6×6 网格
  - 玩家: 蓝色方块，可摇杆移动
  - 怪物: 红色方块，冲锋型 AI
  - 自动攻击: 每 1 秒发射白色弹丸

测试脚本:
  1. 进入房间，观察自动攻击是否自然
  2. 走位尝试: 绕障碍物、穿荆棘地板、站治疗符文
  3. 翻滚测试: 躲避一次怪物攻击
  4. 技能测试: 使用冲刺冲锋跨过 3 格
  5. 记录每场战斗耗时
  6. 测试后问卷: "走位顺手吗? 翻滚有成就感吗?"
```

### 11.2 验收标准

```
[ ] 玩家无需点攻击按钮就能自动攻击
[ ] 移动摇杆响应延迟 < 50ms
[ ] 翻滚无敌帧可被玩家感知和利用
[ ] 自动攻击锁定目标符合直觉（打最近敌人）
[ ] 一场战斗 < 2 分钟
[ ] 玩家在战斗中至少主动走位过
[ ] 怪物 AI 表现出行为差异（能认出不同怪物类型）
[ ] 元素反应在战斗中可见并可感知
```
