## 1. 目标与范围

### 1.1 设计目标
把现有"物理连续穿越 6×6 房间"的地牢,重做为《酋长你别跑》式的**节点路线图肉鸽**:进层看见整张节点地图 → 选路 → 点节点进入独立 3D 俯视场景打完/交互 → 回地图选下一节点。战斗本身不动(手动 ARPG 手感 + 元素反应 + 闪白/震屏/弹道)。

### 1.2 设计支柱(不可妥协)
1. **战略选择感** — 看全图、做路线赌注。
2. **单层清晰** — 路线一眼可见,不迷路。
3. **战斗零改动** — 复用现有 ARPG 战斗层。
4. **移动端稳健** — 断线可恢复,性能不崩。

### 1.3 与 ai-tasks 基座的关系
`docs/ai-tasks/`(Demo0–6 + §5.5–5.13) 已落地:`GameContext`/`LifecycleManager`/`Logger`/`ConfigDatabase`、`CameraBrain`、`ICollisionService`/`KinematicMover`、`SaveManager`(`ISaveManager`)、`SaveService`(`core/save`)、`ReplayRecorder`(FNV-1a 种子确定性)、`SkillGraph`、`DungeonGenerator`(多房间连通图)、`GameBootstrap`(服务注册 + lifecycle)。本重做**复用**这些基座。

> ⚠️ `GameBootstrap` 当前注册完服务即 `destroyAll()`(§8.4),非正式容器,本版 §8.4/§14.1 已将其纳入 P0 修复授权。

### 1.4 3D 主线 / 2D 兜底
- **正式目标路径 = 3D encounter**:节点进入后由 `SingleRoomEncounterBuilder` 造单房 → 3D 俯视场景战斗。
- **2D 仅开发兜底**:3D 底座未就绪时用 2D fallback 调试节点流程,**不作为发布形态**。

## 2. 核心循环 (Core Loop)

### Moment-to-Moment (0–30 秒)
- **Action**: 在节点地图上点一个**可达**节点。
- **Feedback**: 节点高亮脉冲 + 镜头推近 + 进入加载。
- **Reward**: "选择被兑现"的掌控感。

### Session Loop (5–30 分钟,单层)
- **Goal**: 从 `start` 走到 `boss`,沿途吃 Elite/Event/Shop/Rest 节点。
- **Tension**: 分支赌注——高风险 Elite 还是稳妥 Rest。
- **Resolution**: 击杀 Boss → `floor_cleared` → 下一层或结算。

### Long-Term Loop (小时–周)
- **Progression**: 多层累积 build / 解锁。
- **Retention Hook**: 每日种子挑战(同种子同图,复用 `ReplayRecorder` 确定性)。

## 3. 节点路线图数据结构

> **v0.4.2 修正(③)**: 以下纯数据类型**统一存放于 `assets/scripts/core/save/RouteSaveTypes.ts`**(纯数据,无 Cocos / 无 controller 依赖),`dungeon/route/` 与 `core/save/SaveTypes.ts` 均从此文件 import。此举消除"若 `SaveTypes.ts` 直接 import `dungeon/route` 的 `NodeRouteMapDefinition` 等 → `core/save → dungeon/route` 倒挂依赖"的隐患。本处为可读性展示结构,正式定义见 §10.3。

### 3.1 静态定义(生成一次,不可变)
```ts
export type RouteNodeType =
    | 'start' | 'combat' | 'elite' | 'event'
    | 'treasure' | 'shop' | 'rest' | 'upgrade' | 'boss';

export interface NodeRouteNodeDef {
    id: string;
    row: number;
    col: number;
    type: RouteNodeType;
    parents: string[];
    children: string[];
}

export interface NodeRouteMapDefinition {
    floorIndex: number;
    rows: number;
    nodes: Record<string, NodeRouteNodeDef>;
    edges: Array<{ from: string; to: string }>;
    startNodeId: string;
    bossNodeId: string;
    seed: number;
}
```

### 3.2 运行时状态(单点真相)
```ts
export type RouteRunPhase =
    | 'map_select' | 'node_entering' | 'encounter_running'
    | 'reward_pending' | 'node_resolved' | 'floor_cleared' | 'run_settled';

export interface NodeRouteRuntimeState {
    currentNodeId: string;
    reachableNodeIds: string[];
    completedNodeIds: string[];
    phase: RouteRunPhase;
}
```

### 3.3 UI 视图状态(派生,不存储)
```ts
export type RouteNodeViewState = 'current' | 'completed' | 'reachable' | 'locked';

export function getNodeViewState(
    nodeId: string,
    state: NodeRouteRuntimeState,
): RouteNodeViewState {
    if (state.currentNodeId === nodeId) return 'current';
    if (state.completedNodeIds.includes(nodeId)) return 'completed';
    if (state.reachableNodeIds.includes(nodeId)) return 'reachable';
    return 'locked';
}
```

## 4. 生成算法与验证规则

### 4.1 算法
- `rows = 10 + floorIndex`(`[PLACEHOLDER]` 调)。
- 每行 2–4 节点,`start` 独占第 0 行,`boss` 独占末行。
- 边生成: 每个节点连 1–2 个下一行节点,保证**至少 2 条**从 start 到 boss 的不重复路径。
- 节点类型分布(按权重 `[PLACEHOLDER]`): combat / elite / event / treasure / shop / rest / upgrade / boss。
- 全部随机来自 `routeSeed`(§10),禁 `Math.random`。

### 4.2 验证规则 (`NodeRouteValidator`)
- **连通性**: 从 `start` BFS 必达 `boss`。
- **无环**: 边的 `to.row > from.row` 恒成立。
- **保底**: 每行至少一节点被前一行覆盖。
- **Boss 可达(v0.4.1 修正, 放宽)**:
  ```ts
  // Boss reachable if ANY parent is completed (standard roguelike: clear one parent -> enter next row)
  const bossReachable = def.nodes[def.bossNodeId].parents
      .some(p => state.completedNodeIds.includes(p));
  ```
  > 不要求"所有父节点都完成"——那等于强制清完整层,削弱路线选择感。若未来设计确需"清完整层",作为可选 `requireAllParents: boolean` 开关(默认 `false`)。

### 4.3 "Broken" 定义
- 全单节点路径(无选择) → broken。
- Boss 不可达(无任何父节点可完成路径) → broken。
- 任意节点无出边且非 Boss → broken。

## 8. 节点执行层:SingleRoomEncounterBuilder + 3D 主线

### 8.1 ❌ 禁止误用 DungeonGenerator roomCount:1
`DungeonGenerator.generate(seed, zone, { roomCount: 1 })` 不可用:其 `line 54` 将 `roomCount` clamp 到 `[3,12]`,传入 1 实际变 3,且它是"多房间连通图"生成器,职责不匹配单节点。

### 8.2 ✅ SingleRoomEncounterBuilder(类型冲突修正)
> **v0.4.1 修正**: 原 `SingleRoomEncounterOptions.type: Constants.RoomType` 与返回 `RoomLayout.type`(`DungeonGenerator.RoomType`) 是**两套不兼容类型**。现引入局部 `EncounterRoomLayoutType` 对齐 `DungeonGenerator.RoomType`,`build()` 直接产 `RoomLayout`,不再混用 `Constants.RoomType`。
> **v0.4.2 修正(⑥)**: 移除 `rewardRoomType` 字段——奖励类型归属 `RouteNodeTypeAdapter`(§6.2)/ `NodeRewardResolver`(§9.3),不应混进"只负责造房间几何"的 builder options。
> **v0.4.3 优化**: `EncounterRoomLayoutType` 不再手写 union,改为从 `DungeonGenerator` 导入别名 —— 一旦 `DungeonGenerator.RoomType` 改动,此处天然同步,不会出现 union 与 `RoomLayout.type` 失配。

```ts
// v0.4.3: alias into DungeonGenerator.RoomType, no hand-written union
import type { RoomType as EncounterRoomLayoutType } from './DungeonGenerator';

export interface SingleRoomEncounterOptions {
    roomId: string;
    layoutType: EncounterRoomLayoutType;   // drives RoomLayout.type (== DungeonGenerator.RoomType)
    zone: string;
    seed: number;
    minSize?: number;                       // default 7
    maxSize?: number;                       // default 11
}

export class SingleRoomEncounterBuilder {
    build(opts: SingleRoomEncounterOptions): RoomLayout {
        const rng = new Rng(opts.seed);
        return {
            id: opts.roomId,
            type: opts.layoutType,          // EncounterRoomLayoutType -> RoomLayout.type OK
            gridX: 0,
            gridY: 0,
            width: oddSize(rng, opts.minSize ?? 7, opts.maxSize ?? 11),
            height: oddSize(rng, opts.minSize ?? 7, opts.maxSize ?? 11),
            connections: [],
        };
    }
}
```
- 文件位置:`assets/scripts/dungeon/SingleRoomEncounterBuilder.ts`(非 `route/`)。
- 不破坏 `DungeonGenerator` 多房间职责,符合 ai-tasks 契约边界。

### 9.2 修正:事件化(上下文注入, v0.4.3)

```ts

// v0.4.4 修正:RouteNodeType / RouteEncounterContext 均属 core/save/RouteSaveTypes 纯数据类型
//   RoomFlowController 位于 assets/scripts/run/,相对路径为 ../core/save/RouteSaveTypes
//   禁止从 dungeon/route 引(会倒挂)、禁止 ../../core/...(会落到 assets/scripts/ 上层不存在处)
import type { RouteNodeType, RouteEncounterContext } from '../core/save/RouteSaveTypes';

// v0.4.4: RouteEncounterContext 不再在本文件内定义,改由 RouteSaveTypes 统一提供(见 §10.3)
//   export interface RouteEncounterContext { nodeId: string; nodeType: RouteNodeType; startedAt: number; }

export class RoomFlowController {
    private _routeCtx: RouteEncounterContext | null = null;
    private _encounterKills = 0;

    // v0.4.3: 进场景前调用;传 null 表示离开 route 上下文
    setRouteEncounterContext(ctx: RouteEncounterContext | null): void {
        this._routeCtx = ctx;
        this._encounterKills = 0;
    }

    onBattleVictory(mode: 'legacy' | 'route' = 'legacy'): void {
        if (mode === 'route') {
            // v0.4.3: payload 全部来自 _routeCtx,不引用任何临时字段
            const ctx = this._routeCtx;
            if (!ctx) {
                console.warn('[RoomFlowController] route victory without RouteEncounterContext');
                return;
            }
            eventBus.emit('route:encounter_complete', {
                nodeId: ctx.nodeId,
                nodeType: ctx.nodeType,         // RouteNodeType
                result: 'victory',
                elapsed: Date.now() - ctx.startedAt,
                kills: this._encounterKills,
            });
            return;
        }
        this._rewardService.grantRoomClearRewards(roomType);
    }
}
```

> `kills` 由 `RoomFlowController` 在战斗内自增(`onMonsterKilled` 计数);`elapsed` 由 `startedAt` 派生。其余字段一律来自 `RouteEncounterContext`,实现者**不得**给 `RoomFlowController` 新增 `_currentNodeId` / `_currentNodeType` 之类字段。

```ts
// assets/scripts/core/save/RouteSaveAdapter.ts (implements RouteSavePort)
import { SaveService } from './SaveService';
import { RouteRunSnapshot, RouteSavePort } from './RouteSaveTypes';

export class RouteSaveAdapter implements RouteSavePort {
    // v0.4.2: 用单例,禁 new;v0.4.3: 归属 core/save,不在 dungeon/route
    private readonly _save = SaveService.instance;

    saveRoute(snapshot: RouteRunSnapshot): boolean {
        // v0.4.3: 无 active run 直接返回 false;新 RunSave 只能由 RunCoordinator 创建
        const run = this._save.loadRun();
        if (!run) {
            console.warn('[RouteSaveAdapter] no active RunSave; skip route save');
            return false;
        }
        run.route = snapshot;
        return this._save.saveRun(run);
    }

    loadRoute(): RouteRunSnapshot | null {
        return this._save.loadRun()?.route ?? null;
    }

    clearRoute(): void {
        const run = this._save.loadRun();
        if (run) { run.route = undefined; this._save.saveRun(run); }
    }
}
```
