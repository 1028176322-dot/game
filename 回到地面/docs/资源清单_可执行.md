### 1.4 依赖关系

- **3D 资产依赖**：每项 3D 资产的 `depends[]` 取自 `art_3d_manifest.json` 的 `source_2d`（其源 2D 精灵表）。生产前须确认源 2D 已就绪（见 §1.2 的 2D 遗留列）。
- **2D 资产依赖**：icon/ui/background 多为独立；tile 为模块化拼装；角色/怪物/Boss 特效依赖对应骨骼动画 clip（§16.5 socket）。
- **跨轨依赖**：3D 资产入库前，其 `depends[]` 中引用的 2D 资源（如角色待重做的 34 个）须先完成，否则 `asset_validate` 判 FAIL。

## §3 执行步骤（可执行级）

**通用驱动方式**：读取 `resource_inventory.json`，筛选 `target != "verify"` 的项，按 `priority` 升序逐条执行其 `skill_command`，完成后将 `current_status` 置为相应终态并写入进度文件（§8）。

> **提示词规范**：所有 AI 出图类（backgrounds / icons / ui 等）的 prompt 撰写规范—9 块结构、分类策略、技术约束、安全合规、模块调色板—详见 `docs/资源提示词设计策略.md`。运行 `generate` 前建议先确认 `prompts.json` 中对应 key 的 prompt 已按策略文档对齐。

### 3.1 目标=regenerate_2d_256（角色精灵表修正）
1. 确认 `prompts.json` 中该 key 的 prompt 指定 `256x1024`（若仍写 192×768，先更新 prompt 文案，不破坏其他消费方）。
2. `python tools/art_pipeline.py generate --resource <rel> --force`（强制重生成）。
3. 自检：PNG 256×1024、RGBA、4 帧竖排、体积 ≤ 角色预算。
4. **视觉审核（人工确认）** → 用户说"可以/入库"方可继续。
5. `python tools/art_pipeline.py import --all`（默认 2d 模式）。

### 3.2 目标=generate（背景缺失）
1. `python tools/art_pipeline.py generate --resource <bg key>`（Agnes）。
2. 后处理：PNG 母版 + JPG 运行时双路径，JPG ≤ 350KB。
3. 安全词过滤：禁 skull/骨/血/心。
4. **视觉审核（人工确认）** → 入库。

### 3.3 目标=produce_3d（3D 资产生产，外部 Blender）
1. Blender 制作 `.glb`（命名 `CHR_/MON_/BOSS_/FX_/TILE_` + PascalCase）。
2. 导出 ASTC 贴图 → Cocos 导入生成 `.prefab`。
3. 编写 `<NAME>.assetmeta.json`（naming/budget/sockets/colliders/depends/lifecycle/version/author/date/reviewer）。
4. `python tools/asset_validate.py assets/resources/models/<NAME>.assetmeta.json` → 必须 PASS。
5. **asset_validate 门禁（人工确认报告）** → `python tools/art_pipeline.py import --mode 3d --resource <NAME>`。

### 3.4 目标=reconcile（UI 去重）
1. `python tools/art_pipeline.py audit --category ui` 列出磁盘文件。
2. 逐项比对 `prompts.json`：孤儿/重复/程序化冗余 → 标记保留或删除（删除需用户确认）。

| 3D 特效 | ≤4500(1500)/512²/ASTC6 / ≥8 clip | 同上 | `asset_validate.py` |
| 3D 地块 | ≤2500/512²/ASTC6 / LOD100·60·30 | 同上 | `asset_validate.py` |

> 数值权威源：`art_quality_budget.json → rules3d`（§16.3）。任何工具禁止硬编码，统一从此读取。

## §6 人工确认节点

| 节点 | 触发 | 确认方式 | 不确认后果 |
|---|---|---|---|
| 视觉审核 | 每个 2D 资源生成/重生成后 | 用户明确说"可以"或"入库" | 不得进入 import，资源停留 candidate |
| asset_validate 报告 | 每个 3D 资产生成 `.assetmeta.json` 后 | 用户审阅 PASS 报告 | FAIL 项禁止入库，须修复或降级 lifecycle |
| 入库审批 | import 执行前 | 用户确认批次 | 禁止自动入库 |
| 删除/去重 | UI 核对发现孤儿文件 | 用户确认删除清单 | 不删除，保留并标记 |

> 任何节点未获确认，AI 不得继续执行后续步骤（强制门禁）。

## §7 最终入库

**2D 入库流程**：candidate → 备份旧文件至 `art_source/textures_review/backup/<ts>/` → 复制到 `assets/resources/textures/<cat>/<file>` → 更新 `game_assets.json`/`ui_assets.json`（如新增）→ 跑 `npm.cmd run validate:all`。

**3D 入库流程**：`.glb` → `assets/resources/models/<NAME>.glb`；`.prefab` → `assets/resources/prefabs/<NAME>.prefab`；**绝不进 `textures/`**。登记 `game_assets.json`（type=Prefab/SkeletalAnimation/Material），`lifecycle=已批准`。

**数据格式要求**：
```json
// game_assets.json 条目（3D 示例）
"CHR_Archer_A": {
  "assetId": "prefabs/CHR_Archer_A", "type": "prefab", "category": "characters",
  "safeReview": false, "lifecycle": "已批准",
  "tri": 2800, "bones": 28, "textureSize": 512, "lod": [100,60,30], "clips": 7
}
// .assetmeta.json（asset_validate 输入）
{ "name":"CHR_Archer_A", "bucket":"characters", "naming":{"pattern":"..."},
  "budget":{"maxTri":3000,"maxBones":30,"textureSize":512},
  "sockets":[...], "colliders":[...], "depends":[...], "lifecycle":"已批准",
  "version":1, "author":"...", "date":"2026-07-10", "reviewer":"..." }
```
