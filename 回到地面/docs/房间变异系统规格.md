# 房间变异系统 · 详细规格

> **版本**: v1.0 | **日期**: 2026-06-25 | **设计者**: GameDesigner
> **系统层**: 内容层 P1 | **依赖**: 地牢生成系统 (变异挂载在楼层上)
> **变异数量**: 12 种, 每层随机 1~2 个

---

## 1. 设计目的

**玩家体验**: 进入新楼层时——"这一层的规则是什么？"

不是"怪物变强了"这种千篇一律的难度递增，而是**规则层面的改变**: 这一层吸血强, 下一层攻速快, 再下一层全是毒。玩家不是单纯堆数值，而是根据当前规则**调整策略**。

**核心感受**: "血月层我该走吸血 Build, 淘金热层我该贪金币"——每层的"最优解"不同。

### 2.2 变异效果规格

```
变异效果必须实现为对战斗系统的"参数覆盖":

interface RoomMutation {
  id: string;                    // M01~M12
  name: string;
  layerDuration: 'floor';        // 持续整层
  effects: {
    // 对战斗参数的修正 (覆盖 / 百分比 / 绝对值)
    playerAtkMod?: number;       // ±
    playerDefMod?: number;
    playerSpeedMod?: number;     // 百分比
    playerAttackSpeed?: number;  // 百分比
    playerHealEffect?: number;   // 百分比
    playerLifesteal?: number;    // 百分比
    monsterAtkMod?: number;
    monsterDefMod?: number;
    monsterHpMod?: number;       // 百分比
    monsterSpeedMod?: number;
    skillCdMod?: number;         // 百分比
    goldDropMod?: number;        // 百分比
    visionRange?: number;        // 格数
    // 特殊效果 (布尔开关)
    elementStorm?: boolean;      // 奥术风暴
    earthquake?: boolean;        // 地震
    mist?: boolean;              // 薄雾
    unstableSpace?: boolean;     // 不稳定空间
    countdown?: number;          // 倒计时 (秒)
  };
}
```

## 3. 变异生成算法

### 3.1 变异选择

```
每层选择 1~2 个变异:

1. 第 1 个变异: 从 12 种中权重随机
   - 权重 = 基础权重 + 层数修正 + DDA 修正
   
2. 第 2 个变异: 30% 概率出现
   - 从剩余 11 种中选 (排除第 1 个)
   - 层数 ≥ 3 时概率提升至 50%
   - 层数 ≥ 5 时概率提升至 70%

3. 变异互斥检查:
   - 如果新选变异与已有变异互斥 → 重新抽取
   - 连续 3 次互斥 → 只保留 1 个变异
```

### 3.3 互斥表

```
两个变异不能同时出现 (互相抵消或导致游戏无法进行):

互斥对:
  血月 + 淘金热 = 治疗-50% 但金币×2 → 不互斥, 但玩家很痛苦
  黑暗降临 + 倒计时 = 看不清还要限时 → 不互斥, 但太折磨
  
  实际互斥:
  薄雾 + 地震 = 迷雾+地形变化 → 玩家完全迷失
  → 已确认互斥

  倒计时 + 地震 = 限时+地形变化 → 玩家无法规划路线
  → 已确认互斥

【核心原则: 可以难, 但不能无解】
```

## 4. Mechanic: 变异效果

```
Mechanic: 房间变异

Purpose: 改变本层的战斗规则, 让每层的体验不同
Player Fantasy: "这层的规则不一样了, 我得换个打法"

激活时机: 进入楼层时立即激活
持续时长: 整层
移除时机: 离开楼层 (进入下层/死亡/通关)

变异效果的触发方式:
  - 参数覆盖: 战斗系统在每次进行计算时读取当前变异参数
  - 特殊效果 (奥术风暴/地震等): 通过游戏 tick 每 0.5 秒触发
  
Edge Cases:
  - 变异效果与玩家 Build 效果叠加 → 加法叠加, 不是乘法
    例: 血月(吸血×2) + 吸血光环(15%) = 15% × 2 = 30% 吸血
  
  - 变异效果为负值时, 最小值不能低于 -50%
    例: 血月(治疗-50%) → 治疗药水效果 10HP = 5HP
  
  - 多个变异叠加时, 同类效果相加
    例: 淘金热(金币×2) + 无其他金币修正 = ×2
    例: 时空扭曲(攻速+20%) + 回声(技能CD-20%) = 攻速+20%, 技能CD-20%

  - 变异导致战斗不可能完成 → DDA 机制保护 (见 DDA 规格)
    例: 倒计时(60秒) + 怪物血量特别高 → 降低怪物 HP 30%

Tuning Levers:
  - 各变异权重
  - 变异效果数值
  - 互斥规则
  - 第 2 变异出现概率

Dependencies: 战斗系统 (参数覆盖), 地牢生成系統 (楼层持有者)
```

## 5. 对战斗的影响速查

| 变异 | 玩家策略调整 | 战斗影响 |
|------|-------------|----------|
| 黑暗降临 | 站在空地打, 找视觉出口 | 远程怪更难定位 |
| 血月 | 优先拿吸血能力 | 药水价值降低 |
| 奥术风暴 | 带元素附魔或元素抗性 | 每 3 秒全屏受伤 |
| 时空扭曲 | 积极进攻 (攻速更快) | 怪物追不上你 |
| 地震 | 不要依赖固定地形 | 走位策略需动态调整 |
| 淘金热 | 尽可能多触发战斗 | 怪物更肉, 但更值钱 |
| 虚弱诅咒 | 优先清小怪叠 ATK | 前期刮痧, 后期爽 |
| 狂乱 | 速战速决, 避免持久战 | 怪的攻击很高但很脆 |
| 回声 | 多放技能输出 | 技能 Build 爆发期 |
| 薄雾 | 大胆走荆棘路线 | 负面地形变免费高速路 |
| 倒计时 | 快速战斗, 不打没必要的怪 | 时间压力 |
| 不稳定空间 | 利用全场元素触发反应 | 无元素 Build 劣势 |
