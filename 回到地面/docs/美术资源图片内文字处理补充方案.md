# 美术资源图片内文字处理补充方案

> 项目：回到地面  
> 适用范围：AI 生成美术资源、现有 PNG 资源、UI/图标/背景/角色/Boss/特效、后续所有进入 `assets/resources` 的视觉资源  
> 目标：避免图片内出现不可控英文、乱码、伪文字、水印、Logo、数字和可读字符，降低微信小游戏审核、本地化、UI 维护和画质风险。

---

## 1. 结论

普通美术资源上出现英文或可读文字，**原则上不建议保留**。

正确做法是：

```text
美术图只负责图形、材质、装饰、图标、背景
游戏文案由 Cocos Label / 富文本 / 本地化表渲染
Logo 和极少数标题字单独管理
AI 生成资源默认禁止文字、字母、数字、水印、招牌
```

图片内文字会带来这些问题：

1. 本地化困难：中英文、繁体、后续改文案都要重新出图。
2. 审核风险：AI 可能生成敏感词、错误词、伪英文、宗教/暴力/赌博暗示。
3. 视觉不统一：中文 UI + 英文贴图会显得混乱。
4. 画质风险：小图压缩后英文会糊，像乱码。
5. 维护困难：文字画死在 PNG 里，无法动态改字号、颜色、语言和样式。

## 2. 适用资源范围

必须检查：

```text
assets/resources/textures/ui
assets/resources/textures/icons
assets/resources/textures/backgrounds
assets/resources/textures/characters
assets/resources/textures/monsters
assets/resources/textures/bosses
assets/resources/textures/effects
assets/resources/arts/placeholders
```

后续如果增加，也必须检查：

```text
assets/resources/spine
assets/resources/prefabs
assets/resources/fonts
```

其中字体本身不是问题，但字体文件、Logo 图、标题字图必须登记用途和授权。

## 3. 图片内文字分类

### 3.1 必须处理

| 类型 | 示例 | 处理 |
|---|---|---|
| 英文单词 | START、SHOP、POWER、FIRE、DANGER | 重做或抹除 |
| 中文文字 | 开始、商店、胜利、失败 | 原则上也不画死，改 Label |
| 数字 | 1、2、HP+10、x2 | 改代码/Label 渲染 |
| 水印 | AI 平台水印、签名、Logo | 必须重做 |
| 伪英文 | AI 生成的乱码英文、假字母 | 必须重做或抹除 |
| 招牌/海报 | 背景里的英文标牌、墙上海报 | 视清晰度处理，能读则处理 |
| 敏感词 | blood、kill、death、casino 等 | 必须重做 |

### 3.2 可以保留但要登记

| 类型 | 条件 | 处理 |
|---|---|---|
| 游戏 Logo | 专门设计、需要品牌露出 | 单独登记为 logo 资源 |
| 抽象符文 | 不可读，不像真实文字 | 可保留，标记 decorative |
| 极少数标题字 | 例如主标题图 | 单独管理，不混入普通 UI |
| 图形化符号 | 箭头、十字、元素符号、星形 | 可保留 |

### 3.3 不确定资源

如果看起来像文字，但无法判断：

```text
action=review_text
status=pending
note=疑似伪文字/招牌/符号，需要人工确认
```

不要直接放行。

## 4. CSV 字段补充方案

在 `textures_audit_manifest.csv` 或后续 `art_assets_manifest.csv` 中增加以下字段：

```csv
has_text,text_type,text_language,text_action,text_note
```

字段含义：

| 字段 | 示例 | 说明 |
|---|---|---|
| `has_text` | true/false | 图片内是否有可读或疑似文字 |
| `text_type` | english/chinese/number/logo/watermark/fake_text/signage/decorative/none | 文字类型 |
| `text_language` | en/zh/unknown/none | 语言 |
| `text_action` | regenerate/paint_over/replace_with_label/keep/review | 处理动作 |
| `text_note` | 背景左上角有 DANGER | 说明位置和风险 |

示例：

```csv
path,has_text,text_type,text_language,text_action,text_note
textures/ui/main/start_button.png,true,english,en,replace_with_label,按钮文字 START 应由 Label 渲染
textures/icons/skills/icon_fireball.png,true,english,en,regenerate,图标上有 FIRE 字样
textures/backgrounds/bg_room_shop.png,true,signage,en,paint_over,背景招牌有 SHOP
textures/ui/splash/splash_logo.png,true,logo,zh,keep,游戏标题 Logo 允许保留
textures/effects/combat/fx_hit_normal.png,false,none,none,keep,
```

## 5. 处理动作标准

### 5.1 `regenerate`

适用：

```text
图标上有英文
角色/Boss 身上有字母
AI 生成水印
伪英文大面积出现
背景文字太多无法自然抹除
```

要求：

```text
重新生成无文字版本
保持原路径、尺寸、透明边界、锚点策略
通过 PNG mode 检查
通过人工视觉检查
```

### 5.2 `paint_over`

适用：

```text
背景墙上的小标语
UI 装饰边角上的小字母
不影响主体结构的小文字
```

要求：

```text
自然修补材质
不能留下糊块
不能破坏透明边缘
修补后再次压缩和验收
```

### 5.3 `replace_with_label`

适用：

```text
按钮文字
面板标题
卡牌标题
数值
商店价格
技能名称
胜利/失败提示
```

做法：

```text
PNG 只保留底图
Cocos 使用 Label 渲染文字
文字走 TextManager / text.json
字号、颜色、描边由 UI 组件控制
```

### 5.4 `keep`

适用：

```text
游戏 Logo
确认不可读的符文
纯图形符号
明确作为品牌资产的标题字
```

要求：

```text
必须在 text_note 说明原因
Logo 类资源单独放入 logo/category
不作为普通按钮/图标混用
```

### 5.5 `review`

适用：

```text
疑似文字
疑似水印
疑似敏感词
模糊招牌
AI 伪字母
```

要求：

```text
不能进入 approved 状态
必须人工看图确认
```

## 6. AI 生成提示词规范

### 6.1 通用负面约束

所有 AI 美术生成提示词都必须加入：

```text
no text, no letters, no words, no numbers, no logo, no watermark, no signature, no signage, no poster, no readable characters
```

中文补充：

```text
画面中不要出现任何文字、字母、数字、英文单词、中文、Logo、水印、签名、招牌、海报或可读字符。
```

### 6.2 图标类提示词

```text
pure game icon, symbol only, no text, no letters, no numbers, no words, no watermark
```

中文：

```text
纯图形图标，只表现符号和物体，不要任何文字、字母、数字、水印。
```

### 6.3 UI 类提示词

```text
blank UI button background, no label text, no letters, no numbers, no logo, game UI frame only
```

中文：

```text
只生成按钮底图或面板底图，不要按钮文字、标题文字、数字或任何可读字符。
```

### 6.4 背景类提示词

```text
environment background only, no signs, no posters, no banners, no readable text, no letters, no numbers
```

中文：

```text
只生成环境背景，不要招牌、海报、横幅、文字、字母、数字或可读标记。
```

### 6.5 角色/Boss 类提示词

```text
character sprite, no text on clothes, no symbols resembling letters, no logo, no watermark
```

中文：

```text
角色身上不要出现文字、字母、编号、Logo、水印或类似字母的装饰。
```

## 7. 审计流程

### 第 1 步：自动扫描

可以先用脚本做基础筛查：

```text
文件名命中 logo/title/text/start/shop/victory/defeat 等关键词
OCR 识别是否存在英文/数字
检查 AI 水印常见区域
检查背景招牌区域
```

注意：OCR 只能辅助，不能完全替代人工。AI 伪文字经常 OCR 不出来。

### 第 2 步：人工快速过图

重点看：

```text
UI 按钮
升级卡牌
技能/遗物图标
主界面背景
商店背景
死亡结算背景
Boss 图
角色衣服/武器
背景墙面/旗帜/招牌
```

### 第 3 步：CSV 标记

所有有文字或疑似文字的资源必须填：

```text
has_text=true
text_type=...
text_action=...
status=pending
```

### 第 4 步：处理资源

按照 `text_action` 执行：

```text
regenerate
paint_over
replace_with_label
keep
review
```

### 第 5 步：复查

复查通过后：

```text
has_text=false
```

或者 Logo/标题字：

```text
has_text=true
text_action=keep
status=approved
```

## 8. UI 改造配合

如果当前按钮图、面板图、卡牌图带文字，应改成：

```text
按钮底图 PNG
+ Cocos Button
+ Cocos Label
+ text.json 文案
```

示例：

```text
错误：
btn_start.png 里写死 START

正确：
btn_primary.png 只做按钮底
Label.string = T('ui.start')
```

文本表：

```json
{
  "ui": {
    "start": "开始游戏",
    "shop": "商店",
    "upgrade": "强化",
    "continue": "继续"
  }
}
```

好处：

```text
中文/英文可切换
文案可热改
按钮可复用
压缩后更清晰
审核风险更低
```

## 9. 微信小游戏注意事项

微信小游戏环境下要特别保守：

```text
不要让 AI 随机生成英文标语
不要出现疑似赌博、血腥、恐怖、宗教、诱导点击词汇
不要出现外部平台 Logo 或水印
不要出现无法解释的英文缩写
不要让按钮图片出现误导性文字
```

推荐所有审核可见文案都走：

```text
text.json
TextManager
Cocos Label
```

这样后续如果要修改合规词，只改配置和 UI，不需要重新出图。

## 10. 验收标准

每批美术资源入库前必须检查：

```text
普通 UI 图不含文字
普通图标不含文字
普通背景不含可读招牌/海报
角色和 Boss 身上不含字母/Logo/编号
没有 AI 水印
没有伪英文乱码
Logo 资源已单独登记
所有文字类内容都能在 text.json 中找到
CSV 中 has_text/text_action/status 已填写
```

硬性失败：

```text
图片内出现英文按钮文字
图片内出现水印/签名
图片内出现敏感英文词
图片内出现不可解释的 AI 乱码大字
图片内文字与游戏实际语言不一致
```

## 11. 推荐落地顺序

### Phase 1：补字段

在 `textures_audit_manifest.csv` 和后续 `art_assets_manifest.csv` 中增加：

```csv
has_text,text_type,text_language,text_action,text_note
```

### Phase 2：先查高风险目录

优先检查：

```text
textures/ui
textures/icons
textures/backgrounds
arts/placeholders
```

### Phase 3：处理第一批

处理顺序：

```text
水印/Logo 误生成
按钮和 UI 英文
图标英文
背景招牌
角色/Boss 身上字母
```

### Phase 4：更新 AI 提示词模板

把 `no text/no letters/no watermark` 写进所有生成脚本和美术需求文档。

### Phase 5：入库门禁

每次资源入库前检查：

```text
PNG mode
尺寸
体积
SpriteFrame
是否有文字
是否有水印
```

---

## 12. 最终原则

```text
能用 Label 的文字，不画进图片。
能用配置管理的文案，不固化进 PNG。
能用符号表达的图标，不写英文。
AI 生成图默认无文字。
Logo 和标题字作为例外单独管理。
```
