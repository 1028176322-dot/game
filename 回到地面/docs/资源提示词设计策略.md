## 1. 统一风格底座（所有资源共享的 6 个核心块）

> 来源：`ART_RESOURCE_RULES.md` §1.1 / §15。这 6 块是「统一感」的来源，**任何资源提示词都必须前置这 6 块**，差异只发生在第 7 块（专属描述）。

| 块名 | 职责 | 英文范本（直接复用） |
|------|------|----------------------|
| **STYLE_CORE** 统一视觉方向 | 锁定整体画风，防止跨资源风格漂移 | `Bright colorful cartoon game style, clean shapes, saturated colors, soft highlights, consistent cute cartoon visual identity, suitable for mobile game asset collection.` |
| **QUALITY_CORE** 发布质量标准 | 锁定清晰度 / 描边 / 可读性，避免低质 | `crisp confident outlines, clean alpha-ready edges, high-resolution details, smooth color gradients, polished lighting, readable silhouette, balanced contrast for small mobile screens.` |
| **SAFE_COPY_CORE** 运行时无文字 | 强制「图上无字」，文字全交 Cocos Label | `ABSOUTELY NO TEXT, NO WATERMARK, NO SIGNATURE, NO WRITING OF ANY KIND. Clean fantasy symbols only.` |
| **SAFE_WORLD_CORE** 安全主题词 | 限定可出现的元素池，规避敏感/恐怖 | `flowers, leaves, mushrooms, crystals, coins, lanterns, books, tents, tools, clouds, stars, paw marks, soft magic sparkles, friendly animal shapes, clean adventure props.` |
| **CONSISTENCY** 一致性声明 | 声明与本游戏既有素材同语言 | `consistent with the integrated forest-and-gem interface style used by the current game.` |
| **PRODUCTION_NOTE** 制作说明 | 声明输出形态（透明/不透明/JPG/PNG） | `full opaque rectangular scene for JPG export.` / `clean transparent edges, PNG RGBA export.` |

**风格总纲（中文）**: 明亮卡通动物冒险 + 温暖手绘动画感 + 森林宝石一体化 UI。圆润友好造型、柔光、饱和自然色、清晰移动端可读性、合家欢安全幻想基调。
**禁止风格**: pixel art / dark fantasy / chunky pixels / 低分辨率 / hard sci-fi / realistic horror。

## 2. 通用 Prompt 结构模板（对齐现有 prompts.json 的 9 块）

现有 `prompts.json` 实为「9 块顺序拼接」。新写/重写任何 AI 提示词，均按此顺序：

```
[1] STYLE_CORE         统一视觉方向        ← 固定（§1）
[2] QUALITY_CORE       发布质量标准        ← 固定（§1）
[3] SAFE_COPY_CORE     运行时无文字规则    ← 固定（§1）
[4] SAFE_WORLD_CORE    安全主题词          ← 固定（§1）
[5] dimensions_clause  画布尺寸            ← 由类别决定（§3 表）
[6] format_clause      格式说明            ← 由类别决定（PNG/JPG/RGBA/RGB）
[7] specific           分类/资源专属描述   ← 唯一差异点（§5 各模板）
[8] consistency        风格一致性声明      ← 固定（§1）
[9] production_note    制作说明            ← 由类别决定
```

> 正向描述铁律（`ART_RESOURCE_RULES.md` §1.2）：**禁止** `no/do not/avoid/without/blood/skull/bone/heart/death/horror/english/letters/watermark` 等负向/危险词出现在发给 API 的正向 prompt 中。危险词只进本地 `riskFilter` 做扫描，不发送。

## 3. 引擎技术规范约束总表

> 数值权威源：`ART_RESOURCE_RULES.md` §3 / §9.2 / §15 / §16；3D 预算以 `assets/resources/config/art_quality_budget.json → rules3d` 为准。

### 3.1 保留 2D / 程序化类（AI 出图或脚本生成）

| 类别 | 推荐尺寸 | 格式/色彩 | 体积推荐→人工上限 | 命名 | 特殊约束 |
|------|----------|-----------|------------------|------|----------|
| backgrounds 背景 | 1920×1080 | **JPG / RGB**（不透明） | 300–800KB → 1.5MB | `bg_{scene}_{region}.jpg` | 满屏无透明；中心留干净战斗区；母版 PNG + 运行时 JPG 双路径 |
| icons 图标 | 192×192 | **PNG / RGBA** | 15–60KB → 180KB | `icon_{name}.png` | 单物体、粗描边、64px 可辨、四周透明留白 |
| ui（装饰/图标类） | 192×192 起 | PNG / RGBA | 按组件（§9.2） | `ui_{module}_{obj}.png` | 装饰框类画布 ×1.8 留边（ORNAMENT_TYPES） |
| ui（AI 出图组件） | 192×192 起（面板/卡片按 9-slice 放大母版） | PNG / RGBA | 按钮 20–100KB→250KB；面板 80–250KB→800KB | `btn_/panel_/card_/slot_/...` | **AI 出图**（prompts.json），9-slice 边距（§3.4），四角须透明 |
| tiles（2D 无缝） | 96×96 | PNG / RGBA 或 RGB | 5–40KB → 120KB | `tile_{region}_{kind}.png` | 四边可平铺（tile_score 检测） |

### 3.2 3D 建模类（Blender → .glb/.prefab，不走 AI）

| 类别 | 预算（取 rules3d） | 命名前缀 | 贴图/压缩 | 特殊约束 |
|------|--------------------|----------|-----------|----------|
| characters 角色 | 2000–3000 Tri / 20–30 骨 / 512² / ASTC 6×6 / LOD 100·60·30% / ≥5 clip | `CHR_{Hero}_A.glb` | 512² ASTC | 原点脚底 +Y-up+面朝+Z；命中帧 AnimEvent；Socket/Collider/LOD 齐全 |
| monsters 普通怪 | 1000–2500 Tri / 18–30 骨 / 512² / LOD 100·60·30% / ≥5 clip | `MON_{Region}_{Name}.glb` | 512² | 同上 |
| bosses 小 Boss | 4000–7000 Tri / 30–60 骨 / 512² / LOD 100·50% | `BOSS_{Name}_{NN}.glb` | 512² | 终 Boss 名含 `Final`；详见 5.6 |
| bosses 终 Boss | 5000–9000 Tri（建议 7000）/ 40–80 骨 / 1024² / ASTC 8×8 / LOD 100·50·20% / ≥8 clip | `BOSS_{FinalName}_{NN}.glb` | 1024² | 最高规格 |
| effects（3D 粒子） | ≤80 粒子 / ≤2 DrawCall / 256² / ≤1.5s（普通）；Boss 特效 ≤300 / ≤4 / 512² / ≤2s | `FX_{Name}.prefab` | — | 见 5.7 |
| tiles（3D 模块件） | 2m×2m 模块 / 512² / LOD 100·50% | `TILE_{Region}_{Module}.glb` | 512² | Module ∈ Floor/Wall/HighGround/Thorn/Corner/Edge/Slope/Ramp |

> **3D 资产未过 `asset_validate.py` 一律禁止入库**（命名 → 预算 → Socket/Collider → depends[] → lifecycle 全检）。

## 4. TapTap 合规与知识产权约束

### 4.1 审核红线（TapTap 宽松模式，对应 §7.2）
| 红线 | 提示词 / 制作中的落实方式 |
|------|--------------------------|
| 无真实血腥 | 受击统一「闪白 + 火花 + 星点」，绝不用红色血液/内脏/器官写实描摹 |
| 无真实恐怖 | 骷髅/怪物可存在但必须卡通化（圆润、饱和色、可爱比例） |
| 无尸体刻画 | 击败 = 化光点 / 闪白消散 / 卡通倒地（不刻画细节） |
| 敏感词放宽但谨慎 | 主题可用「骷髅/死亡/恶魔」，但**正向 prompt 仍禁止**做写实正向描述，避免 AI 出写实图 |
| 命名安全替代 | 可用：骨兵/炎魔/暗影/翠毒/冰晶/爆裂；禁止：骷髅/恶魔/死亡/亡灵/地狱/腐烂/吸血/自爆/食尸（命名层仍走替代词） |

### 4.2 知识产权 / 侵权规避
- **原创动物造型**：5 英雄为原创 Q 版动物（鹿/狮/兔/猫/熊），不临摹任何现有 IP 角色轮廓。
- **不复制现实品牌/Logo/地标**：UI 装饰只用抽象几何、叶卷、爪印、宝石、星花。
- **音乐/音效商用授权**：AI 生成 BGM（如 Suno）须确认平台商用授权或替换为无版权曲库（见 §8）。
- **字体**：游戏内文字走 `text.json` + 系统/授权字体，美术图上禁止嵌字（SAFE_COPY_CORE）。

### 4.3 安全词屏蔽清单（正向 prompt 禁用，仅进 riskFilter）
`blood / skull / skeleton / bone / corpse / organ / anatomical / heart / horror / grimdark / gore / wound / injury / death(文件名用 journey summary 替代) / english / letters / watermark / signature`。

### 4.4 高风险命名自动替换（脚本层，不出现在最终文件名）
`death→journey_summary`，`skull→paw_gem`，`bone→wood_trim`，`heart→vitality_crystal`，`berserker→brave_vanguard`，`lifesteal→vitality_aura`，`critical→star_impact`。

## 5. 分类型提示词设计策略（核心）

> 每类均给出四要素：**用途说明 / 风格关键词 / 技术参数约束 / 差异化变量**，并附模板与示例。

### 5.1 背景场景 `backgrounds`（AI 出图 · JPG）
- **用途说明**：全屏不透明战斗/事件/房间底图，角色与怪物以独立 sprite 叠加其上；需预留干净中心战术区。
- **风格关键词**：top-down 俯视地面透视、三维边缘装饰向上隆起、温暖手绘、区域主色、空旷中心。
- **技术参数约束**：1920×1080、**JPG/RGB 不透明**、体积 300–800KB（上限 1.5MB）；中心 60% 留空；满屏无透明边。
- **差异化变量（6 区域调性）**：

| 区域 | 地面 | 边缘装饰 | 主色 |
|------|------|----------|------|
| forest | 苔藓落叶白花 | 古树丛、灌木 | 暖绿 |
| catacombs | 风化灰岩苔藓纹 | 残破石柱、苔碑 | 灰绿 |
| volcano | 黑曜岩熔岩纹 | 火山岩、熔池 | 橙黑 |
| tundra | 冰霜石砖霜花 | 蓝白冰棱雪坡 | 蓝白 |
| swamp | 淤泥绿潭盘根 | 枯树芦苇、荧光菇 | 墨绿 |
| abyss | 紫黑符文砖 | 黑曜晶簇符柱 | 紫黑 |

- **模板**：`[STYLE_CORE][QUALITY_CORE][SAFE_COPY_CORE][SAFE_WORLD_CORE] Target canvas: 1920x1080. Runtime format: full opaque JPG. Asset role: {scene} backdrop for {region}. Scene concept: top-down {region} arena, {ground}, {edge_decor} at left/right edges, clean empty center tactical area. Composition: top-down floor perspective, clean center for sprites. [CONSISTENCY][PRODUCTION_NOTE]`
- **示例（forest/combat，已存在于 prompts.json，风格一致）**：`"...Asset role: full-screen combat backdrop for the forest region. Scene concept: top-down overhead view of a forest combat arena floor, rich mossy ground texture with scattered autumn leaves and tiny white flowers, strong three-dimensional thick ancient tree trunks and dense bushes rising upward at left and right edges... clean empty center play area, characters rendered as separate game sprites."`

### 5.2 UI 图标 `icons`（AI 出图 · PNG 192×192）
- **用途说明**：独立物品/功能图标（钥匙、金币、魂石、治疗、冲刺等），在 64px 仍须可辨；无文字徽标。
- **风格关键词**：单物体居中、粗描边、内高光、卡通体积感、透明留白。
- **技术参数约束**：192×192、**PNG/RGBA**、体积 15–60KB（上限 180KB）；不透明度比 ≥0.15；装饰框类画布 ×1.8 留边。
- **差异化变量**：按 ICON_SAFE_MAP 16 种已知图标各给安全描述（如 `coin→gold round coin with leaf emblem`，`heal→green cross leaf potion`）；稀有度用色环（普通白 / 精良蓝 / 史诗紫 / 传说金）。
- **模板**：`[STYLE_CORE][QUALITY_CORE][SAFE_COPY_CORE][SAFE_WORLD_CORE] Target canvas: 192x192. Runtime format: PNG RGBA transparent. Asset role: standalone icon for {function}. Specific: one {object}, thick outline, simple inner highlight, recognizable at 64px, centered transparent margin, no badge label. [CONSISTENCY][PRODUCTION_NOTE]`

### 5.3 UI 组件 `ui`（AI 出图 · PNG RGBA）
- **用途说明**：按钮/面板/卡片/输入框/槽位/进度条/徽章/头像框等界面骨架与装饰，**全部由 Agnes AI 出图**（`prompts.json` 现有 173 条 `ui/...` 条目，覆盖 btn_/panel_/card_/slot_/input_/badge/avatar 等）。
- **风格关键词**：卡通动物冒险 UI 语言——圆润木纹 / 叶角 / 宝石点缀 + 蓝灰紫金冒险调色板；粗描边、柔光、清晰可读性；9-slice 边框须**四角完全透明、中心留空**给 Label。
- **技术参数约束**：PNG/RGBA；单组件画布 192×192 起（面板/卡片按 9-slice 放大母版，见 §3.4：按钮 32/18、主按钮 44/22、长条卡片 44/18、大面板 64/64）；体积 按钮 20–100KB→250KB、面板 80–250KB→800KB（§9.2）；不透明度比 ≥0.02；套 `ORNAMENT_TYPES` 留边 ×1.8。
- **AI 提示词模板**（9 块顺序，UI 专属 specific 块）：
  ```
  [STYLE_CORE][QUALITY_CORE][SAFE_COPY_CORE][SAFE_WORLD_CORE]
  Target canvas: {W}x{H}  (组件按 9-slice 放大母版：按钮 240x80 / 面板 360x200 / 槽位 92x92 / 卡片 260x96).
  Runtime format: PNG RGBA with clean alpha, 9-slice safe (fully transparent corners, empty center for Label).
  Asset role: {module} UI {component} for the {screen} screen.
  Shape concept: rounded wooden and leafy {component}, soft bevel, gem corner accent, {module_palette} palette, consistent blue-gray violet gold adventure UI.
  [CONSISTENCY][PRODUCTION_NOTE]
  ```
- **差异化变量（模块调色板）**：splash 暖木金 / main 森林绿 / shop 宝石蓝 / battle 战斗红 / equipment 铁灰紫 / death 暗夜蓝 → 写入 specific 块的 `{module_palette}`，保证「一家人但不撞脸」。
- **注意**：AI 出的 UI 框架须保证 **四角透明 + 中心留空**，否则 9-slice 拉伸会拉花；极简几何遮罩如需兜底可走程序化（回滚路径 `tools/ui_kit_generator.py`，当前 UI 不调用）。

### 5.4 角色 `characters`（3D 建模 Brief · 非 AI）
- **用途说明**：5 英雄（Archer 鹿 / Warrior 狮 / Mage 兔 / Assassin 猫 / Berserker 熊）逐帧动画实体；横版 Roguelike 主角。
- **⚠️ 非 AI 出图**：本类走 Blender→`.glb`→Cocos Prefab，**不写 AI 提示词**，改为下方「建模 Brief」。
- **建模 Brief 模板**：
  ```
  概念: Q版动物英雄 {职业}，圆润卡通、饱和色、友好比例。
  造型: {动物} 形态，{识别色} 主色，{武器/法器}，清晰头-身-肢分离。
  技术: 2000–3000 Tri / 20–30 骨 / 512² ASTC6×6 / LOD 100·60·30% / ≥5 clip
        (idle/attack/walk/hurt/journey_summary/skill/phasechange)。
  约束: 原点脚底+Y-up+面朝+Z；命中帧 AnimEvent；Socket(武器/特效)·Collider·LOD 齐全。
  风格: 与 backgrounds/icons 同一卡通语言；禁止写实/恐怖。
  ```
- **差异化变量（职业识别色）**：

| 英雄 | 动物 | 识别主色 | 武器/法器 |
|------|------|----------|-----------|
| Archer | 鹿 | 棕金 | 弓 |
| Warrior | 狮 | 红 | 剑盾 |
| Mage | 兔/羊 | 蓝紫 | 法杖 |
| Assassin | 猫 | 暗紫 | 双匕 |
| Berserker | 熊 | 橙 | 巨斧 |

- **技术参数约束**：见 §3.2；命名 `CHR_{Hero}_A.glb`；过 `asset_validate` 方可入库。

### 5.5 普通怪物 `monsters`（3D 建模 Brief · 非 AI）
- **用途说明**：36 个区域小怪，战斗中与角色同屏；128px 仍须可读。
- **建模 Brief**：`概念: 卡通区域小怪 {Name}，单只、夸张圆润、友好街机比例。造型: {区域元素} 主题（如 forest 蘑菇/slime，swamp 毒蛙）。技术: 1000–2500 Tri / 18–30 骨 / 512² / LOD 100·60·30% / ≥5 clip。约束: 同 5.4 技术约束；命名 MON_{Region}_{Name}.glb。`
- **差异化变量**：6 区域各 6 只，区域调性（见 5.1 表）+ 怪物原型（植物/元素/野兽）区分。

### 5.6 Boss `bosses`（3D 建模 Brief · 非 AI）
- **用途说明**：42 个（12 终 Boss + 30 小 Boss），高辨识剪影、夸张体型。
- **建模 Brief**：`概念: 巨型卡通 Boss {Name}，大剪影、夸张形状、清晰头-身-肢。造型: {区域} 主题 + 压迫感但可爱化。技术(小Boss): 4000–7000 Tri/30–60骨/512²/LOD100·50%；技术(终Boss): 5000–9000 Tri(建议7000)/40–80骨/1024² ASTC8×8/LOD100·50·20%/≥8 clip。约束: 同上；命名 BOSS_{Name}_{NN}.glb，终Boss 名含 Final。`
- **差异化变量**：终 Boss 用 1024² + 8 clip + 最高骨数；小 Boss 用 512² + 较低预算；区域调性区分配色。

### 5.7 特效 `effects`（3D 粒子为主 / 可选 2D sprite sheet）
- **用途说明**：元素反应 / 打击 / 增益等视觉反馈（burn/freeze/conduct/melt/radiance/shatter…）。
- **3D 轨道（默认）**：`FX_{Name}.prefab` 粒子系统，≤80 粒子/≤2 DrawCall/256²/≤1.5s（Boss 特效 ≤300/≤4/512²/≤2s）。**不写 AI 提示词**，给 VFX Brief：
  `概念: 卡通 {元素} 特效，中心 bold 形状、简洁粒子簇、低色数、高可读性。约束: 无 gritty 烟雾；命名 FX_{Name}.prefab；过 asset_validate。`
- **2D 保留（如坚持 sprite sheet）**：256×1024 竖排 4 帧 PNG/RGBA，体积 80–250KB（上限 700KB），套 AI 模板：
  `Target canvas: 256x1024 vertical 4-frame sheet. Runtime format: PNG RGBA. Asset role: cartoon VFX {effect}, bold center shape per frame, simple particle clusters, limited color count, high readability.`

### 5.8 地块 `tiles`（3D 模块件 + 2D 程序化无缝）
- **用途说明**：战斗场地拼装单元（Floor/Wall/HighGround/Thorn/Corner/Edge/Slope/Ramp），每区域 24 件。
- **3D 模块件（默认）**：`TILE_{Region}_{Module}.glb`，2m×2m/512²/LOD100·50%，Blender 制作，**不写 AI 提示词**。
- **2D 无缝贴图（程序化）**：96×96 PNG，四边可平铺（tile_score 检测），由 `art_pipeline.py generate` 的 Pillow 程序化生成，区域自然地面色。

## 6. 风格统一 + 差异化的治理方法

### 6.1 统一层（保证「一家人」）
- **共享 6 块**（§1）前置所有 AI 提示词。
- **主色板**：森林绿 / 宝石蓝紫 / 暖木 / 金 四色贯穿 UI 与场景。
- **造型语言**：圆润、粗描边、饱和、柔光、Q 版比例，禁止写实/像素/暗黑。
- **无文字**：所有可读文字由 Cocos Label 提供（SAFE_COPY_CORE）。

### 6.2 差异层（保证「不撞脸」）
- **区域调性**（6 套主色 + 元素，见 5.1 表）：forest 暖绿 / catacombs 灰绿 / volcano 橙黑 / tundra 蓝白 / swamp 墨绿 / abyss 紫黑。
- **职业识别色**（5 英雄，见 5.4 表）：鹿棕金 / 狮红 / 兔蓝紫 / 猫暗紫 / 熊橙。
- **品质/稀有度色阶**（图标与掉落）：白→蓝→紫→金 四级。
- **尺寸/帧数差异化**：背景满屏 JPG、图标 192 单图、角色 256×1024 竖排、Boss 384+ 大图、特效 256×1024，帧数与画幅天然区分资源类型。

### 6.3 一致性校验
- AI 提示词：`prompts.json` 共享块一致性由 `normalize_prompts_positive_style.py` 保证；`riskFilter` 扫描危险词。
- 3D 资产：`asset_validate.py` 校验命名/预算/Socket/Collider/depends/lifecycle 一致性。
- 全量：`npm.cmd run validate:all` + `python tools/encoding_audit.py --ci`（issues=0, p0=0）。

## 7. 落地与门禁（如何执行）

> **资源计划**：需要生成/重做哪些资源、优先级顺序、当前状态，见 `docs/资源清单_可执行.md`（含 8 节可执行计划 + 619 项资源清单）。本策略文档 §§1–6 为该清单中 AI 出图类资源提供提示词模板与约束。

### 7.1 AI 出图类（backgrounds / icons / ui）
```
写入 prompts.json → art_pipeline.py generate --category {x}
  → 母版存 art_source/textures_review/master/
  → 【人工视觉确认门禁】用户说"可以/入库"方可继续
  → runtime_candidates 后处理 → validate → 备份 → import
  → 更新 assets.json / ui_assets.json / game_assets.json
  → npm.cmd run validate:all
```

### 7.2 3D 建模类（characters / monsters / bosses / effects / tiles）
```
Blender 制作 .glb → Cocos 导入生成 Prefab
  → art_pipeline.py validate --mode 3d --category {x}  (调 asset_validate.py)
  → 未过禁止入库 → 修复或降级 lifecycle
  → import --mode 3d --category {x}  (.glb→models/ .prefab→prefabs/)
  → 更新 assets.json(3D类型) → validate:all
```

### 7.3 强制门禁
- **人工确认**：生成/校验通过资源，**必须用户明确确认**才能 import，禁止自动跳过。
- **审核优先**：合规红线 > 体积/性能。
- **文档一致**：每次批量后刷新 `docs/progress/resource_status.md`（资源进度唯一权威）。

## 8. 音频 / 音效附录（扩展建议，当前 pipeline 未覆盖）

> 当前 `art-pipeline` 不含音频生产。以下为前瞻设计 Brief，待接入时落地 `audio/` 目录与命名。

### 8.1 音效（SFX）设计 Brief 模板
```
用途: {攻击/受击/技能/UI点击/掉落/胜利} 反馈音
风格: 卡通合成音、短促明亮、与区域调性同音色（forest 木琴 / volcano 低鼓）
技术: WAV/OGG、单音 ≤0.5s、批量 ≤1MB/bundle、移动端 -3dB 峰值
合规: 原创合成或授权曲库；无现实枪声/血腥拟声
命名: sfx_{category}_{event}.wav
```

### 8.2 背景音乐（BGM）设计 Brief 模板
```
用途: {主菜单/战斗/区域/结算} 循环 BGM
风格: 温暖卡通冒险、区域主调性、135–150 BPM（战斗稍快）
技术: OGG/MP3、循环无缝、单曲 ≤2MB、淡入淡出 2s
合规: Suno 等 AI 生成须确认商用授权，否则替换无版权曲库
命名: bgm_{scene}_{region}.ogg
```

### 8.3 目录建议
```
assets/resources/audio/
├── sfx/     # 音效
└── bgm/     # 背景音乐
```
接入后须在 `ART_RESOURCE_RULES.md` 增补音频格式/体积/命名章节（单一权威源原则）。
