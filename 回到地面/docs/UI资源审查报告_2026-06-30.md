## 1. 总结论

当前 UI master 目录共有 92 张 PNG，技术格式层面全部是 `RGBA`，没有发现 `P` 模式问题。

但是视觉语义问题较多，不能直接整体进入 runtime 替换。主要问题是：

- 部分背景/面板中心出现 X、巨大爪印、头像、无关装饰，影响运行时文本和按钮布局。
- 装备槽被生成成带具体物品的图标，不是空槽位。
- HUD 资源出现 X 或语义不对。
- upgrade 图标大量退化成“爪印徽章”，技能/遗物/升级语义丢失。
- 少量地图图标使用问号、勾号、脸部图案，和“无烘焙文字/符号、语义清晰”的规则冲突。
- `splash_bg.png` 是一个相框式插画，不像完整启动页背景。

必须先重做问题资源，再进行最终替换。

## 6. 根因分析

### 6.1 UI 通用提示词过强

当前 UI prompt 大量强调：

```text
paw-print motifs
rounded animal-ear corner shapes
gem dots
stars
leaf curls
```

这对按钮、边框、面板是好的，但对技能/遗物/升级图标是灾难。AI 会把所有东西都画成爪印徽章，导致语义丢失。

### 6.2 upgrade 图标不应归入通用 UI 规则

`ui/upgrade/icon_*.png` 虽然在 ui 目录，但本质是“图标资源”，不应套用面板/按钮装饰规则。

建议在生成脚本中单独识别：

```python
if path.startswith("ui/upgrade/icon_"):
    category_prompt_type = "semantic_icon"
```

并使用 icon 专用规则：

```text
single semantic game icon object only;
subject must directly match filename;
small decorative frame is optional and must not dominate;
no paw badge as main subject unless filename explicitly means paw.
```

### 6.3 空槽位提示词不够硬

装备槽提示词虽然写了 empty center，但仍被生成具体物品。需要加入硬规则：

```text
EMPTY SLOT CONTRACT:
draw the frame only;
inside area must be transparent or plain empty;
do not draw equipment, item, face, paw icon, symbol, weapon, armor, ring, boot, gem, or placeholder object inside.
```

### 6.4 背景/面板没有中心留白约束

`main_bg`、`shop_bg`、`panel_bg` 都说明中心区域没有被严格保护。

应加入：

```text
CENTER CONTENT AREA CONTRACT:
center 70 percent of the asset must remain clean, low-detail, and usable for runtime UI text/buttons;
no large symbol, paw mark, character, icon, X mark, close icon, logo, or badge in the center.
```
