## 1. 问题结论

原有 effects 资源确实整体偏糊。

根因不是单纯压缩体积，而是旧生成链路天然会制造柔边：

```text
Agnes 生成黑底发光图
-> LANCZOS 缩放到 192x768 / 192x1152
-> 黑底亮度转 alpha
-> 保留大量半透明柔光边缘
-> 游戏里缩放后进一步发糊
```

旧版审计指标显示：

```text
多数 effects 半透明像素占比：60% - 95%
alpha 层级：约 227 档
单张 master 体积：160KB - 380KB
```

这类素材更像视频光效或黑底叠加光效，不适合当前“清晰、卡通动物风、移动端小屏可读”的游戏资源。

## 2. 已执行整改

## 5. 后续禁止事项

effects 资源后续不要再使用以下方案：

```text
黑底发光图 + luminance to alpha
AI 生成 1024x1024 后强缩到 192x768
大量半透明 glow 边缘
多列小缩略图式 sprite sheet
为了压缩体积强行降色导致糊块
```

尤其禁止恢复旧逻辑：

```python
img = img.resize((tw, th), Image.Resampling.LANCZOS)
img = apply_black_matte_alpha(img)
```

这条路线会重新造成柔边和糊感。

## 6. 如果要继续优化视觉

建议只在程序化生成器中增强，不要回到 AI。

可优化方向：

```text
1. 每类特效增加 1-2 个硬边辅助形状。
2. 增加少量 180 alpha 的外描边，但 alpha 层级控制在 3 档以内。
3. 为每类特效设置更明确的动画阶段。
4. 增加单帧中心主体尺寸，确保手机小屏可读。
5. 保持一列多帧，不允许每帧内部出现多列变体。
```

建议质量门禁：

```text
alpha_unique <= 4
semi_alpha_pct <= 8%
runtime_size <= 40KB
每帧只有一个主体
禁止黑底残留
禁止多列缩略图
```
