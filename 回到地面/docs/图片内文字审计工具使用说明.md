# 图片内文字审计工具使用说明

> 适用目标：减少人工修改 `textures_audit_manifest.csv` 的工作量  
> 核心思路：工具把疑似有文字的图片做成总览图；你只记录“确实有问题”的编号；脚本自动把其余项批量设为通过。

---

## 1. 你需要做什么

原本需要逐行检查和修改 155 条 CSV。现在改成：

```text
1. 看几张总览图
2. 只把有问题的编号填进 bad_text_items.csv
3. 运行回写脚本
4. 主 CSV 自动更新
```

没有问题的图片不用填，脚本会自动标记为：

```text
has_text=false
text_type=none
text_language=none
text_action=none
status=approved
```

在项目根目录运行：


运行后会生成：


每张总览图里会有多个小格子，每个格子左上角有编号，例如：

这个编号就是你后面要填写的 `review_id`。

## 5. 第三步：填写 bad_text_items.csv

打开：

```text
E:/game/回到地面/art_source/text_review/bad_text_items.csv
```

表头是：

```csv
review_id,text_type,text_language,text_action,text_note
```

只填有问题的编号。

示例：

```csv
review_id,text_type,text_language,text_action,text_note
3,signage,en,paint_over,背景左侧木牌有 SHOP 英文，去掉文字并保留木牌材质
18,english,en,regenerate,图标上有 FIRE 英文，重新生成无文字版本
41,watermark,unknown,regenerate,右下角疑似 AI 水印，必须重做
76,fake_text,unknown,paint_over,背景墙面有伪英文，修成普通墙面纹理
120,logo,zh,keep,游戏标题 Logo，允许保留
```

注意：

```text
不要填写没问题的编号
不要改 text_review_items.csv
只改 bad_text_items.csv
```

---

## 6. 字段怎么填

### 6.1 review_id

填写总览图左上角的编号：

```text
#3  -> review_id=3
#18 -> review_id=18
```

### 6.2 text_type

可选值：

| 值 | 说明 |
|---|---|
| `english` | 明确英文单词 |
| `chinese` | 明确中文 |
| `number` | 数字 |
| `signage` | 背景招牌、海报、横幅 |
| `watermark` | 水印、签名 |
| `fake_text` | AI 伪文字、乱码字母 |
| `logo` | Logo 或标题字 |
| `decorative` | 不可读装饰符号 |
| `unknown` | 不确定 |

### 6.3 text_language

可选值：

| 值 | 说明 |
|---|---|
| `en` | 英文 |
| `zh` | 中文 |
| `unknown` | 无法判断 |
| `multi` | 多语言混合 |
| `none` | 无文字时使用，但坏图表一般不用 |

### 6.4 text_action

可选值：

| 值 | 用途 |
|---|---|
| `regenerate` | 重新生成无文字版本 |
| `paint_over` | 抹除/修补文字 |
| `replace_with_label` | 图片里的文字改为 Cocos Label |
| `keep` | 确认允许保留，比如 Logo |
| `review` | 仍需再次确认 |

推荐选择：

| 情况 | text_action |
|---|---|
| 图标上有英文 | `regenerate` |
| 按钮上有文字 | `replace_with_label` |
| 背景小招牌有字 | `paint_over` |
| AI 水印/签名 | `regenerate` |
| 伪英文 | `paint_over` 或 `regenerate` |
| 游戏 Logo | `keep` |

### 6.5 text_note

写清楚问题和修改要求。

好写法：

```text
背景左侧木牌有 SHOP 英文，去掉文字并保留木牌材质
图标右上角有 FIRE 英文，重新生成无文字版本
按钮中央有 START，改成无文字按钮底图，文字由 Cocos Label 渲染
```

不推荐：

```text
有问题
改一下
文字
```

## 7. 第四步：自动回写主 CSV

填完 `bad_text_items.csv` 后，运行：

```powershell
cd E:\game\回到地面
python tools\art_pipeline\apply_text_review_results.py
```

脚本会做三件事：

1. 读取 `text_review_items.csv`。
2. 读取你填写的 `bad_text_items.csv`。
3. 自动更新 `art_source/textures_audit_manifest.csv`。

脚本会自动备份原 CSV 到：

```text
E:/game/回到地面/art_source/textures_audit_manifest.before_text_review.csv
```

---

## 8. 回写规则

### 8.1 没填进 bad_text_items.csv 的编号

自动改为：

```text
has_text=false
text_type=none
text_language=none
text_action=none
text_note=contact sheet 人工确认无文字/水印/可读字符
status=approved
```

### 8.2 填进 bad_text_items.csv 的编号

如果 `text_action` 是：

```text
regenerate
paint_over
replace_with_label
review
```

自动改为：

```text
has_text=true
status=needs_fix
```

如果 `text_action=keep`：

自动改为：

```text
has_text=true
status=approved
```

## 10. 复查和二次处理

处理完 `needs_fix` 的资源后，可以再次运行：

```powershell
python tools\art_pipeline\generate_text_review_contact_sheet.py
```

如果图片已经修好：

```text
不要再把它填进 bad_text_items.csv
```

再次回写后，它会自动变成 `approved`。

---

## 11. 注意事项

1. 不要手动逐行改 `textures_audit_manifest.csv`，除非你很确定。
2. 不要改 `text_review_items.csv`，它是自动生成的映射表。
3. 只改 `bad_text_items.csv`。
4. `bad_text_items.csv` 只填写有问题的编号。
5. 如果不确定，`text_action=review`，不要放行。
6. 如果是 Logo，必须写清楚为什么允许保留。
7. 如果是按钮文字，优先 `replace_with_label`。
8. 如果是水印，必须 `regenerate`。

---

## 12. 最省事原则

```text
看图时只记坏编号。
没问题的不填。
坏图写清楚怎么改。
脚本自动批准剩下的。
```
