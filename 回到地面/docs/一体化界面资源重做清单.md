# 一体化界面资源重做清单

更新时间：2026-07-06

适用项目：Cocos Creator 3.8.8 微信小游戏《回到地面》

目标：把主要场景和界面从“通用背景 + 小面板贴片 + 通用按钮”的拼装感，升级为“整屏主题背景 + 透明功能层 + 同风格控件”的一体化视觉。所有文字继续走 `assets/resources/config/text.json`，美术图禁止内嵌文字。

## 1. 总原则

一体化界面不等于每个小控件都重画。判断标准如下：

| 类型 | 是否需要重做 | 原因 |
|---|---:|---|
| 整屏界面背景 | 必须 | 决定界面是否一体化，不能靠小面板贴片补救 |
| 当前界面的主操作按钮 | 必须 | 按钮必须和背景材质、边框、光效统一 |
| 当前界面的关键容器/信息底板 | 必须或建议 | 如果用通用 `panel_bg` 会明显像贴片 |
| 图标类资源 | 通常保留 | 图标是内容语义，不需要和每个场景背景绑定 |
| 战斗 HUD | 建议重做 | 战斗中常驻，需和地图、角色风格统一 |
| 地牢房间背景 | 已有但需复核 | 属于战斗场景主体，不按 UI 面板逻辑重做 |

## 2. 通用生成规范

| 项 | 规范 |
|---|---|
| 美术风格 | 明亮卡通动画风、可爱动物冒险、柔和手绘质感、移动游戏 UI |
| 禁止风格 | pixel_style, dark_grunge_style, unsafe_horror_theme, unsafe_injury_theme, hard_sci_fi, low_resolution |
| 禁止内容 | embedded_text, letters_or_numbers, logo_or_watermark, unsafe_horror_theme, unsafe_injury_theme |
| 透明要求 | 整屏背景 RGB/JPG 或 PNG 均可；按钮、图标、面板件必须 PNG RGBA |
| Cocos 要求 | PNG 必须 RGBA，不允许 P 模式索引色 |
| 命名规则 | `snake_case`，路径和 `assetId` 严格一致 |
| 文本规则 | 所有文字由 Label/TextManager 显示，资源中只留空白承载区域 |

## 2.1 高风险词替换规则

AI 生成时不能只依赖负面词。下列词即使出现在文件名中，也不要出现在正向画面描述里，必须替换为安全语义。

| 高风险语义 | 禁止画面联想 | 正向替换描述 |
|---|---|---|
| death / 失败 | unsafe_failure_theme | adventure ended / return to camp / journey summary，表现为温暖营地、花朵、篝火、奖励水晶 |
| revive | unsafe_body_symbol_theme | continue run / green crystal / leaf energy / soft healing glow |
| berserker / frenzy / pact | unsafe_emblem_theme | courage boost / energetic training axe charm / orange power crystal / golden ribbon |
| lifesteal / lifelink / life charm | unsafe_body_symbol_theme | vitality aura / vitality link / green leaf crystal / clover pendant |
| boss | unsafe_major_encounter_marker | major encounter / friendly crown / star gate / large paw mark |
| critical / title / logo | 英文大字、伪文字 | blank decoration frame / starburst symbol / runtime Label area |
| equipment slot | 骨架、人体解剖轮廓 | wooden leafy frame / gem corner / faint tool outline only |

补充要求：资源路径和 assetId 可以保留历史命名，例如 `ui/death/death_bg.png`，但提示词正文必须使用安全替换描述，不能把路径名里的高风险词复制进画面描述。

重要：上表中的“禁止画面联想”和“额外负面词”只用于脚本本地风险扫描，不允许复制进实际生图 prompt。实际传给 Agnes 的 prompt 只保留正向安全画面描述和 `no text/no letters/no words/no numbers`。

## 23. 现有资源替换流程与注意事项

本章用于指导“生成新资源后如何替换当前项目里的旧资源”。替换时优先保证路径、注册表、场景绑定三者一致，避免出现资源存在但运行时加载不到、旧图仍被缓存、或场景节点指向错误的问题。

### 23.1 替换方式分级

| 场景 | 推荐方式 | 说明 |
|---|---|---|
| 新图语义完全等同旧图 | 同路径同名覆盖 | 例如 `btn_default.png` 只换风格，不改用途 |
| 新图语义比旧图更精确 | 新路径 + 新 key | 例如创建角色页应使用 `ui.create.bg`，不要继续复用 `ui.main.bg` |
| 旧图是通用占位图 | 新路径 + 新 key | 例如 `panel_bg` 被多个界面共用，不能直接替换成某个界面专属图 |
| 整屏一体化背景 | 必须新建界面专属资源 | 例如 `ui/create/create_bg.png`、`ui/area/area_bg.png` |
| 小控件只换视觉皮肤 | 可同名覆盖 | 例如 `hud_skillslot.png` 风格升级但用途不变 |

核心原则：

1. 通用资源不要被某个界面专属资源覆盖。
2. 整屏背景必须用专属 key，不要复用 `ui.panel.frame` 或 `ui.main.bg`。
3. 所有新资源必须先进入 `assets/resources/textures/`，再同步配置表。

### 23.2 推荐替换目录

一体化重做资源建议按界面分目录，不继续全部塞到 `ui/common/`。

| 界面 | 推荐目录 |
|---|---|
| Splash | `assets/resources/textures/ui/splash/` |
| MainHub | `assets/resources/textures/ui/main/` |
| Login | `assets/resources/textures/ui/login/` |
| Create | `assets/resources/textures/ui/create/` |
| Character | `assets/resources/textures/ui/character/` |
| AreaSelect | `assets/resources/textures/ui/area/` |
| Shop | `assets/resources/textures/ui/shop/` |
| Log | `assets/resources/textures/ui/log/` |
| Settings | `assets/resources/textures/ui/settings/` |
| Settlement | `assets/resources/textures/ui/settlement/` |
| Death | `assets/resources/textures/ui/death/` |
| HUD | `assets/resources/textures/ui/hud/` |
| Map | `assets/resources/textures/ui/map/` |
| Upgrade | `assets/resources/textures/ui/upgrade/` |
| Equipment | `assets/resources/textures/ui/equipment/` |
| Event | `assets/resources/textures/ui/event/` |

### 23.3 标准替换步骤

每批资源按以下顺序执行：

1. 先确认工作区状态。

```bash
git status --short
```

不要在未确认工作区状态时批量删除旧资源。

2. 放入新资源。

示例：

```text
assets/resources/textures/ui/create/create_bg.png
assets/resources/textures/ui/create/btn_class_default.png
assets/resources/textures/ui/create/btn_class_selected.png
```

3. 确认文件模式。

| 类型 | 要求 |
|---|---|
| 整屏背景 | PNG RGB/RGBA 或 JPG |
| UI 按钮/面板/图标 | PNG RGBA |
| 禁止 | PNG P 模式、带文字、带水印、伪文字 |

4. 在 Cocos Creator 中等待资源导入完成。

如果是新目录或新文件，必须确认 Assets 面板里出现对应资源和 `.meta`。如果资源刚放入但 Cocos 没刷新，先不要修改场景绑定。

5. 更新 `assets/resources/config/assets.json`。

新增资源必须有对应条目：

```json
"textures/ui/create/create_bg": {
  "bundle": "resources",
  "type": "Texture2D",
  "path": "textures/ui/create/create_bg"
}
```

注意：

- `assetId` 不带扩展名。
- `path` 不带扩展名。
- JPG 和 PNG 都用无扩展名路径。
- 如果 Cocos 自动生成的是 SpriteFrame，也仍建议主注册表记录为 `Texture2D`，由 `RenderAssetService.applyTextureAsSprite()` 统一转 SpriteFrame。

6. 更新 `assets/resources/config/ui_assets.json`。

示例：

```json
"ui.create.bg": {
  "assetId": "textures/ui/create/create_bg",
  "type": "background",
  "usage": "create_character_background"
}
```

类型选择：

| type | 使用场景 |
|---|---|
| `background` | 整屏背景，会按父节点铺满 |
| `sprite` | 普通按钮、图标、装饰 |
| `nine_slice` | 可拉伸面板、输入框、长按钮 |
| `icon` | 技能、装备、元素、道具图标 |

7. 更新 `assets/resources/config/ui_skin_bindings.json`。

只有“场景里固定存在的节点”才写入这里。运行时代码动态生成的节点，可以在代码里通过 `UISkinService.instance.applyOptional(node, key)` 应用。

示例：

```json
"Canvas/CreatePanel/PanelRoot/Background": "ui.create.bg",
"Canvas/CreatePanel/PanelRoot/PanelFrame/ContentRoot/ActionZone/ConfirmBtn": "ui.create.confirm_btn"
```

注意：

- 路径必须从 `Canvas` 开始。
- 路径必须和 scene 节点树完全一致。
- 如果节点由代码运行时创建，不要写进 `ui_skin_bindings.json`，否则会出现 `node not found`。

8. 更新代码里的语义 key。

错误示例：

```ts
await UISkinService.instance.applyOptional(btnNode, 'ui.main.character_button');
```

创建角色页应改为：

```ts
await UISkinService.instance.applyOptional(btnNode, 'ui.create.class_btn');
```

选中态应使用：

```ts
await UISkinService.instance.applyOptional(btnNode, 'ui.create.class_btn_selected');
```

9. 运行门禁。

```bash
npm.cmd run validate:all
```

必须解决以下错误：

| 错误 | 原因 | 处理 |
|---|---|---|
| Disk not registered | 磁盘有图，但 `assets.json` 没有 | 补 `assets.json` |
| ui_assets assetId missing from assets.json | `ui_assets.json` 引用了未注册资源 | 补 `assets.json` 或修 assetId |
| Registered file missing | `assets.json` 有记录，但磁盘无图 | 放回资源或删配置 |
| node not found | `ui_skin_bindings.json` 路径错或节点运行时生成 | 修路径或删除绑定 |

10. 在 Cocos 里重新预览。

如果看不到新图，按顺序排查：

1. 资源是否已在 Assets 面板完成导入。
2. `.meta` 是否存在。
3. `assets.json` 的 `assetId/path` 是否不带扩展名。
4. `ui_assets.json` 的 key 是否被代码或绑定实际引用。
5. Cocos Preview 是否还在旧缓存，必要时停止 Preview 后重启编辑器。

### 23.4 同名覆盖注意事项

同名覆盖适合“用途不变”的资源。例如：

```text
assets/resources/textures/ui/hud/hud_skillslot.png
assets/resources/textures/ui/map/map_node_current.png
assets/resources/textures/ui/equipment/inventory_slot.png
```

同名覆盖要求：

| 项 | 要求 |
|---|---|
| 尺寸 | 尽量保持旧尺寸，除非代码/布局已同步调整 |
| 透明 | 旧图透明，新图必须透明 |
| 文件格式 | PNG 继续 PNG，JPG 继续 JPG，避免 meta 类型变化 |
| 语义 | 不能从“通用按钮”变成“某界面专属按钮” |

不建议同名覆盖的资源：

```text
assets/resources/textures/ui/common/panel_bg.png
assets/resources/textures/ui/common/btn_default.png
assets/resources/textures/ui/main/main_bg.png
```

这些被多个界面引用，直接覆盖会造成其它界面一起变样。要做一体化，应该新增界面专属资源和 key。

### 23.5 新建专属资源注意事项

新建专属资源适合“一体化界面”。

示例：创建角色页不要继续使用：

```text
ui.main.bg
ui.panel.frame
ui.main.character_button
```

应使用：

```text
ui.create.bg
ui.create.class_btn
ui.create.class_btn_selected
ui.create.confirm_btn
ui.create.skip_btn
ui.create.name_input
```

这样后续调整创建角色页不会影响主城、登录、角色选择等其它界面。

### 23.6 删除旧资源注意事项

不要在第一轮替换时立即删除旧资源。推荐三阶段：

1. 阶段一：新增新资源，完成配置接入。
2. 阶段二：运行项目，确认所有界面视觉正常。
3. 阶段三：用检查脚本确认旧资源没有引用，再删除。

删除前必须检查：

```bash
rg "textures/ui/common/panel_bg|textures/ui/common/btn_default" assets/resources/config assets/scripts assets/scenes
```

只有确认没有引用后，才可以删除旧资源和 `.meta`。

### 23.7 回滚方案

每批替换建议保持小批量：

| 批次 | 建议范围 |
|---|---|
| P0 | Create 创建角色 |
| P1 | MainHub + Login |
| P2 | Character + AreaSelect |
| P3 | Shop + Log + Settings |
| P4 | Settlement + Death |
| P5 | HUD + Map + Upgrade + Equipment |

每批完成后先跑门禁和预览。若失败，只回滚当前批次，不影响前一批。

回滚优先级：

1. 恢复 `ui_assets.json` 和 `ui_skin_bindings.json`。
2. 恢复被同名覆盖的图片。
3. 删除新加但未接入成功的资源。
4. 重启 Cocos 清理预览缓存。

### 23.8 微信小游戏特别注意

| 项 | 要求 |
|---|---|
| 包体 | 整屏 PNG 容易超包，优先 JPG 或压缩 PNG |
| 首包 | Splash/Main/Create/Login 尽量放首包，其它可按分包策略拆 |
| 安全审核 | 禁止 unsafe_horror_theme / unsafe_injury_theme / unsafe_failure_theme |
| 文本 | 图片内禁止文字，避免英文/伪文字和多语言维护问题 |
| 清晰度 | UI 控件用 RGBA PNG，背景可 JPG；不要为压体积过度模糊 |
| 缓存 | 替换资源后要在真机/微信开发者工具复测，浏览器预览不等于微信表现 |

```text
Label / Icon / Lock / Reward = 独立子节点
禁止使用 Simple Sprite 直接拉伸长条资源
```

建议在 `ui_assets.json` 增加 `slice` 字段：

```json
{
  "ui.area.route_card": {
    "assetId": "textures/ui/area/route_card_default",
    "type": "sliced",
    "slice": { "left": 44, "right": 44, "top": 18, "bottom": 18 },
    "minSize": { "width": 300, "height": 92 },
    "defaultSize": { "width": 720, "height": 92 }
  }
}
```

### 分辨率自适应规则

- 横屏基准仍是 `1280x720`。
- 全屏背景使用 cover 铺满，允许裁切边缘，不允许非等比变形。
- 内容容器按 safe area 布局，左右至少预留 `48px`，窄屏优先缩小列表/卡片宽度。
- 长条卡片、按钮、输入框只改变 `UITransform.width`，底图必须通过 `SLICED` 伸缩。
- 图标和角色模型只能等比缩放，不能 9-slice，也不能非等比拉伸。


## 24. 生成验收清单

每张新资源入库前必须检查：

| 检查项 | 要求 |
