## 1. 结论

所有职业的部件化资源都必须先生成“方便切割的标准母版”，再从同一张母版切出部件。

不要直接生成：

```text
attack sprite sheet
walk sprite sheet
每个部件独立 AI 生成
```

正式链路必须是：

```text
职业标准母版 -> 人工标注切割区 -> 脚本切割 parts -> 自动拼装预览 -> 通过后入库
```

一句话：

```text
母版负责部件干净，动画配置负责动作好看。
```

## 3. 全职业通用母版要求

### 3.1 文件参数

| 项 | 标准 |
|---|---|
| 母版尺寸 | `768x768` |
| 格式 | PNG RGBA |
| 背景 | 真透明 |
| 角色占画布高度 | 约 `560-640px` |
| 脚底位置 | 画布下方中心，约 `y=650-690` |
| 视角 | 正面 3/4 视角 |
| 光源 | 左上方柔光 |
| 风格 | 明亮卡通动物冒险风，圆润、干净、适合移动端 |

### 3.2 与 ART_RESOURCE_RULES.md 对齐的硬性规格

| 项 | 标准 |
|---|---|
| 运行时格式 | PNG |
| 色彩模式 | RGBA |
| 角色部件 Bundle | `characters_basic` |
| 正式资源路径 | `assets/resources/textures/characters/{character}/parts/*.png` |
| 母版审查路径 | `art_source/textures_review/master/characters/{character}/reference/*_rig_master.png` |
| 运行时候选路径 | `art_source/textures_review/runtime_candidates/characters/{character}/parts/*.png` |
| 替换准备路径 | `art_source/textures_export/runtime_replace/characters/{character}/parts/*.png` |
| 配置注册 | `assets.json` + `game_assets.json` + `character_parts.json` + `character_rigs.json` |
| 加载方式 | `AssetBundleService` / Runtime Service |

部件推荐尺寸沿用 `ART_RESOURCE_RULES.md` 与 `docs/角色.txt`：

| 部件 | 推荐尺寸 |
|---|---|
| `body` | `160x160` |
| `head` | `128x128` |
| `arm_l` / `arm_r` / 手武器合并部件 | `96x128` 起，按武器外扩 |
| `leg_l` / `leg_r` | `80x112` |
| `ear_l` / `ear_r` | `64x96` |
| `tail` | `96x128` |
| `cape` | `128x160` |
| 职业装饰件 | `96x96` 起，按实际轮廓外扩；`quiver` 只限 archer |

正式入库必须走规则集的 12 步：

```text
确认规格 -> 确定生成方式 -> 生成母版到 master -> 用户视觉审查
-> 导出 runtime_candidates -> 技术检测 -> 备份旧文件
-> 导入 assets/resources/textures -> 更新注册表
-> npm.cmd run validate:all -> 用户最终确认 -> 标记完结
```

### 3.3 标准绑定姿势

所有职业母版都使用“可切割 idle rig pose”，不是攻击姿势。

标准姿势：

```text
身体直立，微微 3/4 朝向右前方
双脚分开站稳，左右腿之间有清晰间隔
头部和身体接触自然，但脖颈连接处可标注
耳朵、尾巴、披风、背包、武器不要和身体完全粘死
左臂和右臂都要离开躯干一点，保留可切割空隙
武器可以和手合并，但不要和身体大面积重叠
所有部件轮廓完整，边缘干净，透明背景无色边
```

### 3.4 禁止使用的母版姿势

以下姿势不适合切割：

```text
攻击挥砍到极限
弓完全拉满并遮住身体
双手交叉挡住胸口
盾牌完全挡住身体
法杖横穿头部或身体
武器贴住腿、尾巴或耳朵
身体侧身过多导致另一侧手脚不可见
披风、尾巴、职业背部装饰和身体完全连成一团
```

## 4. 标准部件集

### 4.1 候选基础部件

以下是角色部件化的候选基础部件，不代表每个职业必须全部独立存在：

```text
body
head
ear_l
ear_r
arm_l
arm_r
leg_l
leg_r
tail
```

如果某个职业使用“手武器合并部件”，对应的普通手臂部件可以不再单独存在。例如：

```text
archer 使用 arm_r_weapon 后，可以不再单独配置 arm_r。
mage 使用 staff_arm_r 后，可以不再单独配置 arm_r。
warrior 使用 sword_arm_r 后，可以不再单独配置 arm_r。
assassin 使用 dagger_arm_l / dagger_arm_r 后，可以不再单独配置 arm_l / arm_r。
berserker 使用 axe_arm_r 后，可以不再单独配置 arm_r。
```

正式验收不按固定数量判断，而按 `character_parts.json` 的声明判断。

### 4.2 职业扩展部件

| 职业 | 扩展部件 |
|---|---|
| warrior | `shield_arm_l`, `sword_arm_r`, `helmet`, `cape` |
| archer | `arm_r_weapon`, `quiver`, `cape` |
| assassin | `dagger_arm_l`, `dagger_arm_r`, `scarf`, `cape` |
| mage | `staff_arm_r`, `robe_front`, `hat`, `cape` |
| berserker | `axe_arm_r`, `arm_l`, `shoulder_guard`, `belt` |

说明：

```text
第一版优先允许“手和武器合并”的职业部件。
例如 archer 用 arm_r_weapon，warrior 用 sword_arm_r，mage 用 staff_arm_r。
这样比强行拆手和武器更稳定。
```

### 4.3 部件数量按职业配置决定

角色部件数量不是固定 12 个。

正确规则：

```text
基础候选部件 + 职业扩展部件 - 被合并替代的普通部件 = 该职业最终部件集
```

每个职业以配置表为准：

```text
assets/resources/config/character_parts.json
assets/resources/config/character_rigs.json
```

验收标准：

```text
1. character_parts.json 声明的部件全部存在。
2. character_rigs.json 中声明的部件和 character_parts.json 一致。
3. 未声明的多余部件不进入正式 textures。
4. 每个部件来自同一张职业母版。
5. 拼装预览完整，不缺手、脚、耳朵、尾巴、职业武器和关键装饰。
6. idle / attack 两个基础动作预览不穿帮。
```

推荐最终部件集示例：

| 职业 | 推荐最终部件集 | 说明 |
|---|---|---|
| `warrior` | `body`, `head`, `ear_l`, `ear_r`, `leg_l`, `leg_r`, `tail`, `helmet`, `cape`, `shield_arm_l`, `sword_arm_r` | 剑和盾分别与手臂合并，普通 `arm_l/arm_r` 可省略 |
| `archer` | `body`, `head`, `ear_l`, `ear_r`, `leg_l`, `leg_r`, `tail`, `cape`, `quiver`, `arm_l`, `arm_r_weapon` | 右臂和弓合并，`quiver` 只限 archer |
| `assassin` | `body`, `head`, `ear_l`, `ear_r`, `leg_l`, `leg_r`, `tail`, `scarf`, `cape`, `dagger_arm_l`, `dagger_arm_r` | 双匕首分别与左右手臂合并 |
| `mage` | `body`, `head`, `ear_l`, `ear_r`, `leg_l`, `leg_r`, `tail`, `hat`, `robe_front`, `cape`, `arm_l`, `staff_arm_r` | 右臂和法杖合并，左臂保留 |
| `berserker` | `body`, `head`, `ear_l`, `ear_r`, `leg_l`, `leg_r`, `tail`, `shoulder_guard`, `belt`, `arm_l`, `axe_arm_r` | 右臂和大斧合并，左臂保留 |

脚本要求：

```text
切割脚本、预览脚本、校验脚本都必须读取配置表中的 parts。
脚本不能写死 12。
脚本不能因为某职业缺少 quiver、cape、arm_r 等非必需部件就判失败。
脚本只能在配置声明了某部件但文件缺失时判失败。
```

### 4.4 箭袋不是通用部件

`quiver` 只属于 `archer` 鹿弓手。

其他职业不要生成、切割、注册箭袋：

```text
warrior: 箭袋不适用；使用 shield_arm_l / sword_arm_r / helmet / cape
assassin: 箭袋不适用；使用 dagger_arm_l / dagger_arm_r / scarf / cape
mage: 箭袋不适用；使用 staff_arm_r / robe_front / hat / cape
berserker: 箭袋不适用；使用 axe_arm_r / shoulder_guard / belt
```

如果旧配置、旧 prompt 或旧部件目录中给其他职业出现 `quiver`，按无效部件处理：

```text
1. 不写入 character_parts.json。
2. 不写入 character_rigs.json。
3. 不导入 assets/resources/textures/characters/{character}/parts/。
4. 已经存在的旧文件移入 art_source/textures_review/rejected/ 或 backup，不参与正式入库。
```

### 4.5 披风的切割规则

披风是背层部件，不按“完整可见轮廓”硬切。

母版里的披风通常会被头、身体、手臂遮住，这是正常结构。正确切法是：

```text
cape.png = 母版中可见的披风外轮廓 + 肩颈背部连接重叠区
```

切割要求：

```text
1. 母版必须让披风在肩膀两侧、身体下方或一侧边缘露出。
2. cape rect 包含可见披风边缘、披风下摆、肩颈背部连接区。
3. 肩颈背部连接区多保留 16-32px。
4. 被身体遮住的披风中间区域可以是透明，也可以少量补画柔和布料色块。
5. cape 在拼装层级中必须位于 body 后面。
6. 如果母版里披风完全被身体遮住，这张母版需要重做。
```

推荐层级：

```text
shadow       z=0
tail         z=1
cape         z=2
leg_l        z=3
leg_r        z=4
body         z=5
back_prop    z=6
arm_l        z=7
arm_r_weapon z=8
head         z=10
ear_l        z=11
ear_r        z=12
```

如果后续需要披风摆动，第二版可以拆成：

```text
cape_l.png
cape_r.png
```

或者：

```text
cape_top.png
cape_bottom.png
```

第一版推荐仍然使用单个 `cape.png`，先保证角色拼装稳定。

披风母版正向描述可加入：

```text
cape visible behind the shoulders, cape edges visible on both sides of the body, lower cape hem visible below the torso, clear cape silhouette for 2D cutout animation
```

### 4.6 武器的处理规则

武器按“是否需要和手稳定贴合”决定切法。

第一版统一优先使用手武器合并部件：

| 职业 | 第一版推荐部件 | 说明 |
|---|---|---|
| warrior | `shield_arm_l`, `sword_arm_r` | 盾牌和剑都跟手强绑定 |
| archer | `arm_r_weapon` | 右臂、握弓手、弓合并最稳 |
| assassin | `dagger_arm_l`, `dagger_arm_r` | 匕首体积小，和手合并更稳 |
| mage | `staff_arm_r` | 法杖长，先和右手合并 |
| berserker | `axe_arm_r` | 大斧重，先和右手合并 |

第一版不要把手和武器拆成独立图后再拼。独立拆分容易出现：

```text
武器漂浮
手没有握住武器
武器压在错误层级
攻击动画时武器和手错位
不同帧旋转中心不一致
```

第二版需要更精细动画时，再按“后层武器 + 手臂 + 前层武器”拆分：

```text
weapon_back   z=7
arm_with_hand z=8
weapon_front  z=9
```

适用情况：

```text
弓需要穿过手掌
法杖需要被手握住但顶部单独旋转
斧头需要有柄在手后、斧刃在手前
剑盾需要前后遮挡更自然
```

武器母版姿势要求：

```text
1. 武器整体在角色身体外侧，保留清晰轮廓。
2. 武器尖端完整在 768x768 画布内。
3. 武器和脸、胸口、腿部主体保持可切割距离。
4. 握手区域要清晰，手腕和武器接触区多保留 16-32px。
5. 手武器合并部件的 pivot 建议放在肩关节或手腕，根据动画需求二选一。
```

### 6.4 手与武器切割


## 7. archer 鹿弓手母版

```text
鹿弓手，灵巧轻盈，绿色森林装束，弓和箭袋清晰可见。
```

### 7.2 最方便切割的母版姿势

```text

leg_r

arm_r_weapon
```

```text
arm_r_weapon.png = 右臂 + 握弓手 + 弓
```

## 11. 母版切割留白标准

### 11.1 关节重叠区

每个部件不能只沿可见边缘硬切，必须保留关节重叠区。

| 部件 | 推荐重叠 |
|---|---|
| 头部下方 | 多保留 `12-24px` 脖颈连接 |
| 手臂肩部 | 多保留 `16-28px` 肩膀连接 |
| 腿部上方 | 多保留 `16-28px` 髋部连接 |
| 耳朵根部 | 多保留 `8-16px` 头部连接 |
| 尾巴根部 | 多保留 `12-20px` 身体连接 |
| 披风领口 | 多保留 `16-28px` 背部连接 |
| 手武器合并部件 | 多保留 `16-32px` 手腕和武器接触区 |

### 11.2 透明边距

切出的每个部件必须保留透明边距：

```text
最小透明边距：8px
推荐透明边距：12-24px
武器类部件：16-32px
耳朵/尾巴/披风：12-24px
```

不要 tight crop 到刚好贴边，否则旋转时会被裁掉。

## 12. 人工标注规则

### 12.1 标注文件

每个职业必须有：

```text
art_source/characters/{character}/cut_specs/{character}_part_cut_spec.json
```

每个部件至少包含：

```json
{
  "rect": [x, y, width, height],
  "pivot": [px, py],
  "socket": [sx, sy],
  "z": 0,
  "outputSize": [w, h]
}
```

### 12.2 脚本职责

脚本只做：

```text
读取人工标注 rect
绘制 marked 图
切割 PNG
生成 rig json
生成拼装预览
校验边缘和透明度
```

脚本不能做：

```text
自动猜这是手还是腿
自动改 rect 语义
自动删除被认为多余的部件
自动把手和武器拆开
```

```text
[ ] 左右肢体没有交叉成一团

[ ] 没有文字、数字、伪文字、水印
```

## 15. 最小落地顺序

推荐不要五个职业一起推进。按这个顺序：

```text
1. archer：验证手武器合并方案和拼装预览。
2. warrior：验证盾牌/剑这种双手武器方案。
3. mage：验证长法杖和长耳朵。
4. assassin：验证双武器和尾巴。
5. berserker：验证大斧和大体型。
```

每个职业都要先完成：

```text
母版 -> marked 图 -> parts -> rig preview -> idle/attack 测试
```

通过后再做下一个职业。

## 16. 核心注意事项

```text
不要把母版画成攻击动作。
不要让武器横穿身体主体。
不要把盾牌挡住整个身体。
不要让帽子压住耳朵连接点。
不要让披风、尾巴、箭袋和身体完全粘连。
不要让脚被长袍完全遮住。
不要在第一版强行拆手和武器。
```

第一版的目标不是把部件拆到最细，而是稳定拼成一个完整角色，并且能支持基础 idle、walk、attack 动画。
