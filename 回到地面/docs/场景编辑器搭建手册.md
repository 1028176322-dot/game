# 场景编辑器搭建手册

> 正式上线规范路径 · v2.6  
> 覆盖 splash.scene / main.scene / dungeon.scene 三个场景  
> **核心原则：先跑通，再搭完整。** 每个场景都按"最低可运行结构 → 正式上线结构"两档推进。

---

## 约定与术语

- **编辑器布局（默认）**：左侧 = 层级管理器（节点树），中间 = 场景/游戏视图，右侧 = 属性检查器（Inspector），底部 = 资源管理器（Assets）
- **"右键 Canvas → 创建空节点"** = 在层级管理中左键点击选中 `Canvas` → 鼠标右键点击它 → 弹出菜单 → 左键点击"创建空节点"
- **"改名为 Xxx"** = 在层级管理器中双击节点的名称文字（不要点前面的图标，点文字）→ 进入编辑状态 → 输入新名称 → 回车确认
- **"添加组件"** = 选中节点后 → 在属性检查器底部找到蓝色 **"添加组件"** 按钮 → 左键点击 → 弹出搜索框 → 输入脚本名 → 左键点击搜索到的脚本
- **"拖入"** = 在层级管理器中左键按住某个节点不放 → 拖动鼠标到属性检查器中的某个属性字段上 → 看到属性字段边框变蓝/高亮 → 松开鼠标左键
- **"初始隐藏"** = 在面板节点下创建 `PanelRoot` 子节点 → 给 `PanelRoot` 取消勾选（变灰）→ `panelRoot` 绑到 `PanelRoot`，而非面板自身
- **属性检查器排列**：Cocos Creator 中 @property 字段从上到下按代码声明顺序排列

### 文本配置化规则

> 正式文本必须从 `assets/resources/config/text.json` 读取，不能长期写死在 scene 的 `Label.String` 中。

- 编辑器里的 `Label.String` 只作为预览占位，推荐写成 `[ui.xxx]`，不要写正式中文/英文文案。
- 固定 UI 文本节点必须额外添加 `LocalizedLabel` 组件，并填写 `textKey`。
- 动态文本由代码调用 `T(key, params)` 刷新，不要在业务代码里硬编码文案。
- 已经在编辑器里填过中文的 Label 不需要立刻删除节点，只要补上 `LocalizedLabel.textKey`，后续可逐步把 `Label.String` 改成 `[key]` 占位。

**固定 Label 绑定文本 key 的通用步骤：**

1. 选中 Label 节点，确认已有 `Label` 组件。
2. `Label.String` 填写占位文本，例如 `[ui.mainStart]`。
3. 点击 `添加组件`，搜索并添加 `LocalizedLabel`。
4. 在 `LocalizedLabel.textKey` 中填写对应 key，例如 `ui.mainStart`。
5. `fallback` 可填同义预览文本，方便 `text.json` 未加载时排查。

**不需要添加 `LocalizedLabel` 的情况：**

- `GameBootstrap/StatusLabel` 这类由代码实时写入进度的 Label。
- 战斗 HUD、角色数据、资源数量等动态 Label，如果对应代码已经使用 `T(key, params)` 刷新。

**重要判断规则：**

如果 `text.json` 中某个文本值包含 `{xxx}` 参数占位，例如 `魂石: {count}`、`Lv{level}`，这个 key 默认就是动态文本。不要直接给它挂普通 `LocalizedLabel`，否则运行时会显示 `{count}`、`{level}`。这类文本必须由业务代码调用 `T(key, params)` 并传入真实参数。

常见动态 key 风险清单：

| key | 示例文本 | 正确处理 |
|-----|----------|----------|
| `ui.charName` | `{name}` | 不挂 `LocalizedLabel`，由 `MainHubUI` 刷新 |
| `ui.charClass` | `{class}` | 不挂 `LocalizedLabel`，由 `MainHubUI` 刷新 |
| `ui.charLevel` | `Lv{level}` | 不挂 `LocalizedLabel`，由 `MainHubUI` 刷新 |
| `ui.soulStones` | `魂石: {count}` | 不挂 `LocalizedLabel`，由 `MainHubUI` 刷新 |
| `ui.areaPlayerInfo` | `角色: {character} Lv{level} 魂石: {stones}` | 不挂 `LocalizedLabel`，由 `AreaSelectPanel` 刷新 |
| `ui.hp` / `ui.floor` / `ui.defeat` | 带 `{cur}`、`{floor}`、`{count}` | 不挂 `LocalizedLabel`，由战斗 HUD 代码刷新 |
| `ui.settlementFloor` / `ui.settlementKill` / `ui.settlementSoulStone` | 带结算参数 | 不挂 `LocalizedLabel`，由结算面板代码刷新 |
| `ui.settingsAccount` / `ui.loginStatus` | 带账号或状态参数 | 不挂 `LocalizedLabel`，由对应面板代码刷新 |

### 怎么知道应该填哪个 key

key 的唯一来源是 `assets/resources/config/text.json`。编辑器里填的 `LocalizedLabel.textKey` 必须能在这个文件中找到。

常用规则：

- UI 文本统一放在 `ui` 分组下，key 形如 `ui.mainStart`、`ui.areaBack`。
- 角色职业名用 `class.xxx`，例如 `class.bearWarrior`。
- 地区名用 `zone.xxx`，例如 `zone.forest`。
- 技能名用 `skill.xxx`，例如 `skill.dash`。
- 物品名用 `item.xxx`，例如 `item.key`。
- 如果找不到合适 key，先在 `text.json` 增加新 key，再回编辑器填写，不能临时写死文本。

常用 UI key 对照：

| 使用位置 | Label.String 占位 | 处理方式 |
|---------|--------------------|----------|
| 跳过按钮 | `[ui.skip]` | 挂 `LocalizedLabel`，`textKey=ui.skip` |
| 加载提示 | `[ui.loading]` | 挂 `LocalizedLabel`，`textKey=ui.loading` |
| 主城开始按钮 | `[ui.mainStart]` | 挂 `LocalizedLabel`，`textKey=ui.mainStart` |
| 主城角色按钮 | `[ui.mainCharacter]` | 挂 `LocalizedLabel`，`textKey=ui.mainCharacter` |
| 主城商店按钮 | `[ui.mainShop]` | 挂 `LocalizedLabel`，`textKey=ui.mainShop` |
| 主城日志按钮 | `[ui.mainLog]` | 挂 `LocalizedLabel`，`textKey=ui.mainLog` |
| 主城设置按钮 | `[ui.mainSettings]` | 挂 `LocalizedLabel`，`textKey=ui.mainSettings` |
| 角色名显示 | `[ui.charName]` | 动态文本，不挂 `LocalizedLabel`；由 `MainHubUI` 调用 `T(key, params)` 刷新 |
| 职业显示 | `[ui.charClass]` | 动态文本，不挂 `LocalizedLabel`；由 `MainHubUI` 调用 `T(key, params)` 刷新 |
| 等级显示 | `[ui.charLevel]` | 动态文本，不挂 `LocalizedLabel`；由 `MainHubUI` 调用 `T(key, params)` 刷新 |
| 魂石显示 | `[ui.soulStones]` | 动态文本，不挂 `LocalizedLabel`；由 `MainHubUI` 调用 `T(key, params)` 刷新 |
| 版本号 | `[ui.appVersion]` | 挂 `LocalizedLabel`，`textKey=ui.appVersion` |
| 默认角色名 | `[ui.defaultName]` | 挂 `LocalizedLabel`，`textKey=ui.defaultName`（`MainHubUI` 用 `T()` 作为 `getCharacterName()` 的 fallback） |
| 默认职业 | `[ui.defaultClass]` | 不挂 `LocalizedLabel`，`MainHubUI._refreshTopBar()` 用 `T('ui.defaultClass')` 作为职业 fallback |
| 区域选择标题 | `[ui.areaTitle]` | 固定标题时挂 `LocalizedLabel`，`textKey=ui.areaTitle` |
| 区域选择开始 | `[ui.areaStart]` | 挂 `LocalizedLabel`，`textKey=ui.areaStart` |
| 区域选择返回 | `[ui.areaBack]` | 挂 `LocalizedLabel`，`textKey=ui.areaBack` |
| 区域角色信息 | `[ui.areaPlayerInfo]` | 动态文本，不挂 `LocalizedLabel`；由 `AreaSelectPanel` 调用 `T(key, params)` 刷新 |
| 关闭按钮 | `[ui.close]` | 挂 `LocalizedLabel`，`textKey=ui.close` |

快速查找命令：

```bash
rg -n "\"mainStart\"|\"areaBack\"|\"close\"" assets/resources/config/text.json
```

## 推荐搭建策略

| 阶段 | 做什么 | 预期结果 |
|------|--------|---------|
| **阶段 1** (快速) | 结构修复 + AreaSelectPanel 最小结构 | splash → 点 StartButton → AreaSelectPanel → 进 dungeon |
| **阶段 2** (正式) | MainHubUI + 7 个面板 + Shop | 完整主城功能 |
| **阶段 3** (完善) | 面板子节点 + 视觉布局 | UI 完整可交互 |

> ⚠️ **阶段 1 也必须创建 AreaSelectPanel**（至少 PanelRoot + PlayerInfo + RouteContainer + LockedContainer + StartBtn + BackBtn）。因为 `MainSceneController._onFlowState(AREA_SELECT)` 会调 `router.open('area_select')`，没有面板注册就会报 `panel not registered` 而卡住。  
> 阶段 1 **不需要**创建 MainHubUI、CharacterBtn、TopBar 等——StartButton 的点击由 `MainSceneController` 的 fallback 绑定接管。

## 一、splash.scene — 启动加载场景

### 1.1 当前已有内容

打开 `splash.scene`，在层级管理中展开 `Canvas`：

```
Canvas
└── SplashUI                    ← 有 SplashUI 脚本
    ├── SkipButton              [UITransform]  → ⚠️ 只有空节点，没有 Label
    └── SplashImage             [UITransform, Sprite]
```

### 1.2 需要补充

1. 给 SkipButton **创建子节点 Label**（不是直接给 SkipButton 加 Label 组件）
2. 创建 **LoadingLabel** 节点
3. 创建 **GameBootstrap** 节点（可选，代码会兜底创建）

### 1.3 操作步骤

#### 操作 1.1：给 SkipButton 加子 Label（⚠️ 不能直接加在 SkipButton 上）

**错误做法：** 给 SkipButton 自身加 Label 组件 → Cocos 3.x 可能报 `Can't add renderable component`。

**正确做法（请严格按以下步骤）：**

1. 在**层级管理器**中找到 `Canvas → SplashUI → SkipButton`
2. **左键点击** `SkipButton` 选中它
3. **右键点击** `SkipButton` → `"创建空节点"`
4. 立即双击新节点名字 → 输入 **`Label`** → 回车
5. 选中 `SkipButton` 下的 `Label` 子节点
6. 在**属性检查器**底部点 `添加组件` → 输入 `Label` → 左键点击
7. 在 `Label` 组件的 `String` 输入框中输入占位 **`[ui.skip]`**
8. 点击 `添加组件` → 搜索 `LocalizedLabel` → 添加
9. 在 `LocalizedLabel.textKey` 中输入 **`ui.skip`**，`fallback` 可填 `跳过`
10. 按 `回车`

**绑定到 SplashUI.skipLabel：**

11. 在**层级管理器**中**左键点击** `SplashUI`
12. 在**属性检查器**中找到 `SplashUI` 脚本的 `skipLabel` 字段（显示 `None(Label)`）
13. **左键按住**层级管理器中的 `SkipButton/Label` 子节点
14. 拖到属性检查器 `skipLabel` 字段的框上 → **松开**
15. 框中显示 `Label(Label)` 字样

**最终 SkipButton 结构：**

```
SkipButton          ← 空节点，不含 Label 组件
    └── Label           ← Label + LocalizedLabel(textKey=ui.skip)，绑给 skipLabel
```

### 2C. 7 个面板的通用创建模板

> **核心原则：**
> 1. 面板节点自身保持 **active = true**（不取消勾选），确保组件的 `onLoad()` 生命周期稳定
> 2. 在面板下创建 **`PanelRoot`** 子节点 → 对 PanelRoot **取消勾选（初始隐藏）**
> 3. 脚本的 `panelRoot` 字段绑定 PanelRoot，而非面板自身
> 4. `open()` 设置 `panelRoot.active = true`，`close()` 设置 `panelRoot.active = false`

#### 通用创建步骤（以 LoginPanel 为例）

```
Canvas
├── LoginPanel                     ← active=true, 有 LoginPanel 脚本
│   └── PanelRoot                  ← active=false, 绑给 panelRoot (ResponsivePanelRoot)
│       ├── DimMask                Sprite 半透明黑底
│       └── PanelFrame             Sprite 面板底板 (尺寸由 ResponsivePanelRoot 控制)
│           └── ContentRoot        内容容器 — 所有 UI 内容放这里
│               ├── TitleLabel     Label
│               ├── SubtitleLabel  Label
│               ├── WechatBtn      Button + Sprite
│               │   └── Label      Label
│               ├── GuestBtn       Button + Sprite
│               │   └── Label      Label
│               ├── AgreementLabel Label
│               └── StatusLabel    Label (动态)
```

> **ContentRoot 作用**：将 UI 内容与面板背景分离。PanelFrame 负责底板视觉效果，ContentRoot 承载所有标题/按钮/列表。后续自适应、Layout 布局、内容重建时，不会影响背景。
│           └── StatusLabel        Label (动态)
```

**具体操作：**

**步骤 1：创建面板节点**
- 在**层级管理器**中**左键点击** `Canvas` 选中它
- **右键 `Canvas`** → `"创建空节点"` → 改名 **`LoginPanel`**
- 保持选中 → `添加组件` → 搜索 `LoginPanel` → 点击
- ✅ **保持属性检查器顶部的 checkbox 为勾选状态**（节点 active = true）

**步骤 2：创建 PanelRoot 子节点**
- **右键 `LoginPanel`** → `"创建空节点"` → 改名 **`PanelRoot`**
- 选中 `PanelRoot` → **取消勾选属性检查器顶部的 checkbox**（节点变灰）

**步骤 3：绑定 panelRoot**
- 选中 `LoginPanel`
- 在属性检查器找到 `panelRoot` 字段
- **左键按住**层级管理器中的 `LoginPanel/PanelRoot`
- 拖到 `panelRoot` 的框上 → **松开**

**步骤 4：挂 ResponsivePanelRoot + 创建 DimMask / PanelFrame（所有面板必做）**

PanelRoot 需要自适应分辨率，不能写死 1280x720。

**4a. 添加 ResponsivePanelRoot 脚本：**
1. 选中 `PanelRoot`
2. `添加组件` → 搜索 `ResponsivePanelRoot` → 点击
3. 在属性检查器中配置 ResponsivePanelRoot 参数：

| 参数 | 值 | 说明 |
|------|-----|------|
| `dimMask` | 先留空 | 建好 DimMask 后拖入 |
| `panelFrame` | 先留空 | 建好 PanelFrame 后拖入 |
| `FrameWidthRatio` | 见各面板表格 | PanelFrame 宽度占屏幕比例 |
| `FrameHeightRatio` | 见各面板表格 | PanelFrame 高度占屏幕比例 |
| `MaxFrameWidth` | 见各面板表格 | 最大宽度限制 |
| `MaxFrameHeight` | 见各面板表格 | 最大高度限制 |
| `MinFrameWidth` | 520 | 最小宽度 |
| `MinFrameHeight` | 360 | 最小高度 |

**4b. 创建 DimMask（全屏半透明遮罩）：**
1. 右键 `PanelRoot` → `"创建空节点"` → 改名 **`DimMask`**
2. 选中 `DimMask` → `添加组件` → `Sprite`
3. 在 `Sprite.Color` 中设置 `R=0, G=0, B=0, A=150`（半透明黑）
4. 回到 `PanelRoot` → 把层级管理器中的 `DimMask` 拖入 `ResponsivePanelRoot` 的 `dimMask` 字段

**4c. 创建 PanelFrame（面板底板）：**
1. 右键 `PanelRoot` → `"创建空节点"` → 改名 **`PanelFrame`**
2. 选中 `PanelFrame` → `添加组件` → `Sprite`
3. 在 `Sprite.Color` 中设置 `R=245, G=245, B=245, A=255`（浅灰面板底）
4. 回到 `PanelRoot` → 把 `PanelFrame` 拖入 `ResponsivePanelRoot` 的 `panelFrame` 字段

**4d. 调整层级顺序：**
确保层级管理器中的顺序是：
```
PanelRoot (ResponsivePanelRoot)
├── DimMask
└── PanelFrame
    └── ContentRoot     ← 所有内容放这里
        ├── TitleLabel
        ├── Button
        └── ...
```

> ⚠️ DimMask 和 PanelFrame 的 Sprite 都不要绑定 SpriteFrame 资源，纯用 Color 着色即可。

**4e. 创建 ContentRoot（所有面板必做）**

ContentRoot 是内容容器，将 UI 内容与面板背景分离，便于后续自适应和布局。PanelFrame 负责底板视觉，ContentRoot 承载所有标题/按钮/列表。

**创建步骤：**
1. 右键 `PanelFrame` → `"创建空节点"` → 改名 **`ContentRoot`**
2. 选中 `ContentRoot`，添加 `UITransform` 组件（不要加 Sprite、Button、Label）
3. 设置基础属性：`Position` = (0, 0, 0)，`Anchor` = 0.5, 0.5，`Width`/`Height` 见下面表格

**移动子节点：**
把原本在 `PanelRoot` 下或 `PanelFrame` 下的所有内容节点（TitleLabel、Button、Input 等），**逐一拖入** `PanelFrame/ContentRoot`。拖完后结构为：
```
PanelRoot
├── DimMask
└── PanelFrame
    └── ContentRoot
        ├── TitleLabel
        ├── Button
        └── StatusLabel (动态)
```

**移动后重新绑定脚本字段：**
节点路径变了，必须重新拖入。例如 `LoginPanel`：

| 字段 | 拖入路径 |
|------|---------|
| `panelRoot` | `LoginPanel/PanelRoot`（不变） |
| `titleLabel` | `LoginPanel/PanelRoot/PanelFrame/ContentRoot/TitleLabel` |
| `subtitleLabel` | `LoginPanel/PanelRoot/PanelFrame/ContentRoot/SubtitleLabel` |
| `wxLoginBtn` | `LoginPanel/PanelRoot/PanelFrame/ContentRoot/WechatBtn` |
| `guestBtn` | `LoginPanel/PanelRoot/PanelFrame/ContentRoot/GuestBtn` |
| `agreementLabel` | `LoginPanel/PanelRoot/PanelFrame/ContentRoot/AgreementLabel` |
| `statusLabel` | `LoginPanel/PanelRoot/PanelFrame/ContentRoot/StatusLabel` |

> `panelRoot` 始终绑定 `PanelRoot` 节点本身，不是 `ContentRoot`。每个面板的绑定表见 4A-4F 节。

### 2D. AreaSelectPanel 专项结构

> 与普通面板不同，`AreaSelectPanel` 的代码依赖多个子节点来显示内容和处理点击。只绑 `panelRoot` 会导致面板打开后一片空白、无法点击。

#### 完整结构

```
Canvas/AreaSelectPanel         ← active=true, 有 AreaSelectPanel 脚本
├── PanelRoot                  ← active=false, 绑给 panelRoot (ResponsivePanelRoot)
│   ├── DimMask                Sprite 半透明黑底
│   ├── PanelFrame             Sprite 面板底板 (尺寸由 ResponsivePanelRoot 控制)
│   │   └── ContentRoot        内容容器
│   │       ├── PlayerInfo     Label (动态显示角色信息)
│   │       ├── RouteContainer Node (内容容器)
│   │       ├── LockedContainer Node (锁定路线容器)
│   │       ├── StartBtn       Button + Sprite (开始按钮)
│   │       │   └── Label      Label + LocalizedLabel(textKey=ui.areaStart)
│   │       └── BackBtn        Button + Sprite (返回按钮)
│   │           └── Label      Label + LocalizedLabel(textKey=ui.areaBack)
```

#### 操作步骤

**步骤 1：创建 AreaSelectPanel 和 PanelRoot**

按 2C 模板完成：创建 `AreaSelectPanel` → 挂 `AreaSelectPanel` 脚本 → 创建 `PanelRoot` 子节点 → 隐藏 PanelRoot → 绑 `panelRoot` 到 PanelRoot。

**步骤 2：创建 RouteContainer（右键 PanelRoot → 创建空节点 → 改名 RouteContainer，已自带 UITransform）**

## 五、常见问题

**Q：SkipButton 加了 Label 之后报错？**
→ SkipButton 不能自己带 Label 组件。按操作 1.1 改：把 Label 做在 SkipButton 的子节点上。

**Q：`panelRoot` 绑了但面板打不开？**
→ 检查两点：
  1. `panelRoot` 绑的是面板节点自身还是 `PanelRoot` 子节点？推荐绑 `PanelRoot`。
  2. 面板主节点是否是 active=true 的？不能整体隐藏。

**Q：Shop 打不开，控制台报 `[UiRouter] panel not registered: shop`？**
→ ShopUI 不是 UIPanel。代码已修复：Shop 按钮走 `eventBus.emit('ui:open_shop')`，MainSceneController 接管。确认 `shopUI` 属性已绑定到 `Canvas/Shop`。

**Q：AreaSelectPanel 打开了但是一片空白？**
→ 代码需要 `routeContainer`、`startBtn` 等子节点。按 2D 专项说明补完整结构。

**Q：地牢加载后黑屏？**
→ 检查 `DungeonSceneController` 的 `gridManager`、`player`、`battleManager`、`dungeonManager` 四个必绑字段是否绑定正确。

**Q：控制台报 `[DungeonSceneInstaller] render layers not initialized`？**
→ `DungeonSceneController` 不在 `Canvas` 的直接子节点下。回 3.2 修复嵌套结构。

**Q：编辑器里之前已经填了中文/英文 Label.String，要怎么处理？**
→ 不需要删除节点。给这个 Label 补 `LocalizedLabel`，填正确 `textKey`；`Label.String` 先保留也能跑。整理阶段再把它替换成 `[key]` 占位，避免误以为 scene 文本是正式配置。

**Q：哪些文本必须进 `text.json`？**
→ 所有玩家可见文本都必须进 `assets/resources/config/text.json`，包括按钮、标题、面板提示、结算文案、错误提示。图片资源上也不能烘焙文字，文字统一由 Label 渲染。

**Q：动态 Label 还要不要加 `LocalizedLabel`？**
→ 如果代码每次都会用 `T(key, params)` 刷新，可以不加。比如进度、数值、角色信息这类动态内容，关键是代码不能硬编码最终文案。

**Q：浏览器里显示 `{name}`、`{class}`、`Lv{level}`、`魂石: {count}` 是什么原因？**
→ 这是把带参数模板的动态 Label 错挂了 `LocalizedLabel`。删除这些节点上的 `LocalizedLabel`，让 `MainHubUI._refreshTopBar()` 传入真实参数刷新：
  - `Canvas/MainUI/CharNameLabel`
  - `Canvas/MainUI/CharClassLabel`
  - `Canvas/MainUI/LevelLabel`
  - `Canvas/MainUI/SoulStoneLabel`

如果删除后仍不刷新，再检查 `MainHubUI` 组件里的 `charNameLabel`、`charClassLabel`、`levelLabel`、`soulStoneLabel` 是否分别绑定到了对应节点。
