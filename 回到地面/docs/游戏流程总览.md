> **版本**: v1.1 | **日期**: 2026-07-01 | **设计者**: GameDesigner

## 架构总览

### 核心原则

| 原则 | 说明 |
|------|------|
| **3 场景原则** | 只保留 `splash` / `main` / `dungeon` 三个场景。登录、创建角色、角色选择、区域选择、商店等全部作为 main 场景下的全屏 Panel。 |
| **Flow 驱动** | 场景切换和流程推进由 `AppFlowController` 状态机统一管理，不分散在各 UI 脚本中。 |
| **Panel 体系** | main 场景下的所有功能入口统一走 Panel 栈管理（UiRouter），禁止独立场景。 |
| **Run 统一入口** | 进入地牢必须通过 `RunCoordinator.startRun(config)`，携带完整的 `RunStartConfig`，禁止直接 `loadScene('dungeon')`。 |
| **渲染层标准化** | 地牢战斗的渲染层固定为 Background→Tile→Actor→Effect→Door 五层，Systems 只放逻辑组件，不可渲染。 |
| **动画配置化** | 所有角色/怪物动画通过 `SpriteAnimationService` 读取 JSON 配置播放，不写死帧索引。 |
| **UI 文本 100% 配置化** | 所有玩家可见文本走 `text.json`，图片内零文字。 |

### 四条硬约束（违反即视为架构违规）

#### 约束 1: AppFlowController 必须是唯一流程入口

**规则**: UI 层不允许直接调用 `director.loadScene()`。所有场景/流程切换必须通过以下三个接口之一：

```typescript
// ✅ 允许的三条入口
appFlow.goTo(AppFlowState.AreaSelect);      // 面板/流程切换
runCoordinator.startRun(config);             // 进入地牢
appFlow.returnToMainHub();                   // 返回主城
```

**禁止**:
```typescript
// ❌ 直接 loadScene
director.loadScene('dungeon');
director.loadScene('main');

// ❌ 绕过 Flow 的事件驱动
eventBus.emit(GameEvent.DUNGEON_ENTER);
GameManager.instance.setPhase(GamePhase.Dungeon);
```

**实现方式**: `SceneFlowService` 是唯一持有 `director.loadScene` 引用的类，其他代码通过 `AppFlowController` 间接调用。

---

#### 约束 2: Panel 必须走统一生命周期

**规则**: main 场景下的每个 Panel 必须实现 `UIPanel` 接口：

```typescript
interface UIPanel {
    id: string;                          // 面板唯一标识
    open(params?: unknown): void;         // 打开（由 UiRouter 调用）
    close(): void;                        // 关闭
    refresh?(): void;                     // 可选：刷新数据
}
```

**禁止**:
```typescript
// ❌ 各自 show/hide
panelRoot.active = true;
this.node.active = false;

// ❌ 没有注册到 UiRouter
```

**实现方式**: `UiRouter` 维护 Panel 栈，管理打开/关闭/遮罩层级。所有 Panel 通过 `UiRouter.open('panel_id')` 调用，通过 `UiRouter.close('panel_id')` 关闭。

**Panel 注册**:
```typescript
// UiRouter.registerPanel({ id: 'character', ctor: CharacterPanel });
// router.open('character', { filter: 'warrior' });
```

---

#### 约束 3: RunStartConfig 是进入地牢的唯一数据结构

**规则**: 启动地牢必须且只能通过 `RunStartConfig` 传递参数。

```typescript
export interface RunStartConfig {
    characterId: string;      // 角色类型
    characterName: string;    // 角色名
    zoneRoute: string[];      // 区域路线
    seed: number;             // 随机种子
    difficulty: number;       // 难度系数
    startedAt: number;        // 启动时间
    isContinue: boolean;      // 是否继续
}
```

**禁止**:
```typescript
// ❌ 全局变量传递参数
GameManager.instance.currentFloor = 1;
GameManager.instance.setPhase(GamePhase.Dungeon);
eventBus.emit(GameEvent.DUNGEON_ENTER);
```

**流程**: `AreaSelectPanel` → 组装 `RunStartConfig` → `RunCoordinator.startRun(config)` → `RunStateService` 存储运行期状态 → `SceneFlowService.loadDungeon()`。

---

#### 约束 4: RuntimeLayerService 和 SpriteAnimationService 必须优先落地

**这两个不是视觉优化，而是正式渲染底座。** 在开始任何新 UI 或战斗功能之前，必须先完成这两个服务。

**优先级**: P0，排在所有业务功能之前。

**如果不做**:
- 角色序列帧继续被当作单图显示
- 怪物层级/玩家排序继续硬编码 siblingIndex
- 特效没有专属层，闪白/缩放直接改颜色
- 每次加新角色都要写一遍帧动画代码

**验收标准**:

```typescript
// RuntimeLayerService 验收
layerService.ensureLayers(rootNode);
layerService.addToLayer(monsterNode, LayerType.Actor, gridY);
layerService.addToLayer(effectNode, LayerType.Effect, 0);
const order = layerService.getSortOrder(gridY);
// → 自动 siblingIndex = 100 + Math.floor(y / 64)

// SpriteAnimationService 验收
await animService.loadAnim('warrior_idle');
animService.play(playerNode, 'warrior_idle');
animService.play(playerNode, 'warrior_attack', { onComplete: () => {
    animService.play(playerNode, 'warrior_idle');
}});
```

## Run 启动参数

```typescript
// run/RunStartConfig.ts

export interface RunStartConfig {
    characterId: string;      // 角色类型: warrior/archer/...
    characterName: string;    // 玩家自定义名称
    zoneRoute: string[];      // 区域路线: ['forest', 'catacombs', 'volcano']
    seed: number;             // 随机种子（可复现）
    difficulty: number;       // 难度系数 (1-5)
    startedAt: number;        // 启动时间戳
    isContinue: boolean;      // 是否为继续上次冒险
}

// RunCoordinator 统一入口
// coordinator.startRun(config) 负责:
//   1. 验证 config 合法
//   2. 写入 RunStateService
//   3. 通知 GameManager
//   4. director.loadScene('dungeon')
```

## 动画配置规范

所有角色/怪物动画统一走 JSON 配置，不写死帧索引。

```json
// assets/resources/config/animations.json
{
    "warrior_idle": {
        "resource": "textures/characters/warrior/warrior_idle",
        "frameWidth": 96,
        "frameHeight": 96,
        "frames": 4,
        "fps": 6,
        "loop": true,
        "layout": "vertical"
    },
    "warrior_attack": {
        "resource": "textures/characters/warrior/warrior_attack",
        "frameWidth": 96,
        "frameHeight": 96,
        "frames": 4,
        "fps": 8,
        "loop": false,
        "layout": "vertical"
    }
}
```

## UI 文本规则（强制）

- 所有玩家可见文本必须通过 `T('key')` 读取 `text.json`
- 图片内不得包含任何文字（水印、标题、按钮文字等）
- 按钮图、标题框、结算面板、主城背景只能是**空容器/装饰**，中文由 Label 渲染
- 禁用词汇: 死亡、击杀、骷髅、血、墓碑、骨头 → 替换为 结束、探索、击败、返回营地
