# 数据埋点方案

> **版本**: v1.0 | **日期**: 2026-06-24
> **目标平台**: 微信小游戏 (使用 wx.reportAnalytics)
> **数据限制**: 单条日志 < 1KB | **上报频率**: 事件驱动，非轮询

---

## 1. 埋点设计原则

| 原则 | 说明 |
|------|------|
| **少而精** | 核心事件不超过 15 个，不埋"什么都统计"的无用点 |
| **事件驱动** | 只在关键行为发生时上报，不轮询 |
| **1KB 红线** | 每条日志 payload < 1KB，字段名用缩写 |
| **本地缓存** | 上报失败时缓存，下次启动重试，最多保留 20 条 |
| **用户隐私** | 不上报任何个人可识别信息（PII）|

---

## 2. 核心事件表

### 2.1 全局事件

| 事件 ID | 触发时机 | 字段 | 说明 |
|---------|----------|------|------|
| `game_start` | 游戏启动（启动屏结束后）| `day`(日活标记) | 日活统计 |
| `game_end` | 游戏关闭/切后台 | `sec`(在线时长,秒) | 留存辅助 |
| `version_check` | 启动时版本比对 | `ver`(当前), `remote`(远程) | 版本分布 |

### 2.2 对局事件

| 事件 ID | 触发时机 | 字段 | 说明 |
|---------|----------|------|------|
| `run_start` | 新一局游戏开始 | `seed`(种子码), `char`(角色) | 开局统计 |
| `run_end` | 本局结束（觉悟结算）| `depth`(层数), `kills`(击杀), `time`(秒), `result`(dead/quit), `seed` | **核心留存指标** |
| `floor_enter` | 进入新楼层 | `floor`(层号), `mutations`(变异,逗号分隔) | 楼层分布 |
| `room_clear` | 战斗房清空 | `sec`(耗时), `hp_lost`(受伤), `reactions`(元素反应次数) | 战斗平衡 |
| `build_pick` | 强化房 3 选 1 | `pick`(所选), `tier`(A/B/C), `floor` | **Build 选择分布** |

### 2.3 变现事件

| 事件 ID | 触发时机 | 字段 | 说明 |
|---------|----------|------|------|
| `ad_impression` | 广告展示 | `pos`(广告位, revive/chest/upgrade/shop/soul/daily/marquee), `type`(reward/interstitial/banner) | **核心变现指标** |
| `ad_click` | 用户点击广告 | `pos` | 点击率 |
| `ad_fail` | 广告加载失败 | `pos`, `err`(错误码) | 网络质量监控 |

### 2.4 进程事件

| 事件 ID | 触发时机 | 字段 | 说明 |
|---------|----------|------|------|
| `unlock` | 魂石商店解锁 | `item`(解锁项), `cost`(魂石) | 长线留存 |
| `tutorial_step` | 教学步骤完成 | `step`(1~6) | 新手引导流失 |
| `seed_share` | 分享种子码 | `seed` | 社交传播 |

## 4. 单条日志大小验证

### 对局结束事件（最长的日志）

```
字段:  depth=3&kills=12&time=485&result=dead&seed=AB3K7Z
大小:  ≈ 55 字节 ✓
```

### 完整日志示例（含所有字段）

```
run_end&depth=3&kills=12&time=485&result=dead&seed=AB3K7Z
```
→ **约 65 字节**，远低于 1KB 红线。

### 广告事件（最短的日志）

```
ad_impression&pos=revive&type=reward
```
→ **约 35 字节**

## 5. 上报实现方案

### 5.1 微信小游戏 API

```javascript
// 使用 wx.reportAnalytics 上报
function reportEvent(eventId, params) {
  try {
    wx.reportAnalytics(eventId, params);
  } catch (e) {
    // 上报失败不阻塞游戏
    cacheEvent(eventId, params); // 写入本地缓存
  }
}

// 示例: 对局结束
reportEvent('run_end', {
  depth: 3,
  kills: 12,
  time: 485,
  result: 'dead',
  seed: 'AB3K7Z'
});
```

### 5.2 缓存策略

```javascript
// 本地缓存失败事件
const CACHE_KEY = 'analytics_cache';
const MAX_CACHE = 20;

function cacheEvent(eventId, params) {
  let cache = wx.getStorageSync(CACHE_KEY) || [];
  cache.push({ eventId, params, ts: Date.now() });
  if (cache.length > MAX_CACHE) cache.shift();
  wx.setStorageSync(CACHE_KEY, cache);
}

// 下次启动时重试
function flushCache() {
  let cache = wx.getStorageSync(CACHE_KEY) || [];
  cache.forEach(item => {
    wx.reportAnalytics(item.eventId, item.params);
  });
  wx.setStorageSync(CACHE_KEY, []);
}
```

### 5.3 上报触发点

```javascript
// 在游戏代码的关键位置插入上报

// 进入战斗房时
function onRoomEnter(room) {
  if (room.type === 'combat') {
    // 战斗开始 - 由战斗系统的 onCombatStart 事件触发
  }
}

// 房间清空时
function onRoomClear(stats) {
  reportEvent('room_clear', {
    sec: stats.duration,
    hp: stats.hpLost,
    reactions: stats.reactionsTriggered
  });
}

// 觉悟战时
function onPlayerDeath(stats) {
  reportEvent('run_end', {
    depth: stats.floor,
    kills: stats.kills,
    time: stats.playTime,
    result: 'dead',
    seed: stats.seed
  });
}
```

## 6. 数据分析用途

| 事件 | 分析用途 |
|------|----------|
| `run_end` + `depth` | **核心留存曲线**: 玩家能打到第几层？|
| `build_pick` | **Build 平衡**: 哪些能力没人选？哪些太强？|
| `ad_impression` + `pos` | **广告收入**: 哪个广告位变现效率最高？|
| `ad_fail` | **网络质量**: 哪个区域网络差导致广告加载失败？|
| `tutorial_step` | **新手流失**: 哪一步教学流失最多？|
| `floor_enter` + `mutations` | **变异平衡**: 哪个变异最受欢迎？哪个导致暴毙？|
