## 2. 渲染层标准化 (RuntimeLayerService)

**目标**: 所有可渲染对象归入固定层级, Systems 只放逻辑组件。

| 层 | siblingIndex | 内容 |
|----|-------------|------|
| BackgroundLayer | 0 | 战斗背景图 |
| TileLayer | 1 | 6x6 网格瓦片 |
| ActorLayer | 2 | 玩家 + 怪物（动态 Y 轴排序） |
| EffectLayer | 3 | 战斗特效（新建） |
| DoorLayer | 4 | 出口门 |
| UIRoot | — | HUD/技能/地图 |

**RuntimeLayerService** 职责:
- 创建/管理五层节点
- 提供 `addToLayer(entity, layer, yPos)` 统一接口
- 自动处理 Y 轴排序: `sibIdx = 100 + Math.floor(yPos / 64)`

## 3. 动画系统 (SpriteAnimationService)

**目标**: 所有角色/怪物动画由 JSON 配置驱动, 不写死帧索引。

```json
// assets/resources/config/animations.json
{
    "warrior_idle": {
        "resource": "textures/characters/warrior/warrior_idle",
        "frameWidth": 96, "frameHeight": 96,
        "frames": 4, "fps": 6, "loop": true, "layout": "vertical"
    },
    "warrior_attack": {
        "resource": "textures/characters/warrior/warrior_attack",
        "frameWidth": 96, "frameHeight": 96,
        "frames": 4, "fps": 8, "loop": false, "layout": "vertical"
    }
}
```

**SpriteAnimationService** 职责:
- `loadAnimation(id): AnimationConfig` — 加载动画配置
- `play(node, animId): void` — 在目标节点上播放动画
- `stop(node): void` — 停止
- 自动分帧、定时器切换、loop 处理
