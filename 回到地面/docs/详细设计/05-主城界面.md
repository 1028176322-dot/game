# Step 5: 主城界面 (MainHub) — 功能规格

> **版本**: v1.1 | **日期**: 2026-07-01 | **设计者**: GameDesigner  
> **编码**: UTF-8 | **实现方式**: **MainHubUI** — main 场景核心入口

---

## 1. 职责

整个游戏的功能入口中枢。MainHub 不负责具体业务逻辑，只做三件事：
1. **展示** — 玩家信息、货币、跑马灯
2. **路由** — 各功能按钮打开对应 Panel
3. **入口** — "开始冒险"创建 Run 并进入地牢

---

## 2. 架构关系

```
MainHubUI (挂 main.scene 持久节点)
  │
  ├── TopBar (头像/昵称/等级/魂石/金币/设置⚙)
  ├── MainArea (角色站姿 + 装饰背景, 居中)
  ├── ActionBar (按钮行)
  │   ├── [开始冒险] → AreaSelectPanel → RunCoordinator → dungeon
  │   ├── [角色]     → CharacterPanel
  │   ├── [商店]     → ShopPanel (已有)
  │   └── [冒险日志] → AdventureLogPanel
  ├── BottomBar (跑马灯进度 + 钥匙数 + [看广告])
  │
  └── UiRouter (面板栈)
        ├── LoginPanel      ← AppFlowController 触发
        ├── CreatePanel     ← AppFlowController 触发
        ├── CharacterPanel  ← 角色按钮
        ├── AreaSelectPanel ← 开始冒险
        ├── ShopPanel       ← 商店按钮
        ├── SettingsPanel   ← 设置按钮
        ├── AdventureLogPanel ← 日志按钮
        └── SettlementPanel ← 从 dungeon 返回时
```

## 5. 数据刷新时机

| 时机 | 刷新内容 |
|------|----------|
| MainHub.show() | 全部 TopBar + 跑马灯 |
| 从 dungeon 返回 | 全部刷新(魂石/层数/击杀可能变了) |
| Panel 关闭后 | 对应数据刷新 (角色→TopBar, 商店→魂石) |

---

## 6. 跑马灯管理 (MarqueeManager)

跨局数据, 保存在 StorageService:

| 字段 | key | 说明 |
|------|-----|------|
| 已点亮格数 | `marquee_segments` | 0-3, 满3格重置为0, 钥匙+1 |
| 钥匙数 | `marquee_keys` | 用于宝箱房开箱 |
