# 数值系统 · 完整模型

> **版本**: v1.0 | **日期**: 2026-06-25 | **设计者**: GameDesigner
> **目标平台**: 微信小游戏 (实时自动攻击 · 无限循环)
> **状态**: 所有数值标注 `[H]` (Hypothesis)，需 playtest 验证

---

## 0. 数值设计原则

| 原则 | 含义 | 验证标准 |
|------|------|----------|
| **2 秒杀小怪** | 普通战斗房的小怪应在约 2 次攻击内死亡 | 玩家感到"流畅"而非"刮痧" |
| **3 次容错** | 玩家在正常血量下能承受 Boss 至少 3 次攻击 | 不出现"秒杀"体验 |
| **30-90 秒** | 一场战斗的时长 | 符合碎片化场景 |
| **金币掐脖子** | 金币永远差一点——玩家必须做取舍 | 每次商店都有纠结感 |
| **build 拉开 2 倍差** | 最优 build vs 最差 build 的输出差 ≤ 2 倍 | build 有影响但不致命 |

## 1. 玩家基础属性

### 1.1 一局内的属性增长

```
玩家每局起始属性固定，通过以下方式增长:
  ① 强化房 B 层属性取舍 (每层 1 次, 共 3 次)
  ② 核心能力 A 层效果 (改变攻击行为而非数值)
  ③ 遗物 C 层 (全局修正)
  ④ 魂石商店永久升级 (跨局)
  
无等级概念 —— 局内属性 = 基础值 + 强化房选择
```

### 1.2 基础值表

| 属性 | 初始值 | 最小值 | 最大值 | 增长方式 |
|------|--------|--------|--------|----------|
| HP | 25 | 10 | 80 | B层+生命护符(+8) / 魂石永久(+5) |
| ATK | 4 | 1 | 20 | B层+狂战士契约(+4) / 魂石永久(+1) |
| DEF | 2 | 0 | 10 | B层+铁壁(+3) / 核心被动 |

| Range | 1.5 | 1 | 4 | B层+长臂(+1) |
| 暴击率 | 16.7% | 5% | 40% | d6=6 (+幸运币→d6=5~6 = 33%) |
| 闪避率 | 5% | 0% | 20% | 仅装备提供 |
| 移速 | 100% | 70% | 150% | B层+疾风(+10%) / 核心+疾跑(+30%) |

### 1.3 属性对战斗的影响

| 属性 | 影响 | 优先级 (对玩家) |
|------|------|----------------|
| ATK | 决定 TTK (击杀速度) | ★★★ 核心输出 |
| 攻击频率 | 决定 DPS, 对元素反应频率有加成 | ★★★ 核心输出 |
| 暴击率 | 期望 DPS 提升约 8%~20% | ★★☆ 稳定提升 |
| 吸血 | 有效 HP = HP / (1 - 吸血率) | ★★☆ 续航保障 |
| DEF | 有效 HP 提升约 10%~30% | ★★☆ 生存保障 |
| HP | 硬上限, 决定能抗几下 | ★★☆ 生存保障 |
| 移速 | 走位操作空间 | ★☆☆ 手感影响 |
| 翻滚 CD | 容错率, 越短越安全 | ★☆☆ 手感影响 |

## 2. 伤害模型

### 2.1 公式

```
// 基础伤害
原始伤害 = ATK(攻击者) + 骰子(1~6)
减伤值  = floor(DEF(目标) × 0.5)
实际伤害 = max(原始伤害 - 减伤值, 1)

// 暴击
if (骰子结果 == 6):
    实际伤害 = floor(实际伤害 × 1.5)

// 距离衰减 (仅玩家对怪物)
if (攻击距离 > Range):
    命中率 = 100% - (超出格数 × 20%)
    if (未命中): 伤害 = 0, 不触发元素附着
```

### 2.2 伤害期望值速查

```
玩家 ATK=4 → 原始伤害期望 = 4 + 3.5 = 7.5
玩家 ATK=4 + 暴击 → 期望 = 7.5 × (1 + 16.7% × 0.5) = 8.13  (约+8%)

各 ATK 下的期望伤害:
ATK=4 → 7.5   (1层起始)
ATK=6 → 9.5   (2层+强化1次)
ATK=8 → 11.5  (3层+强化2次)
ATK=10→ 13.5  (4层+强化3次)
ATK=12→ 15.5  (5层+全攻build)

减伤效果:
DEF=2 → 减伤 1  (约 10-15% 减伤)
DEF=5 → 减伤 2.5 向上取整? floor(5×0.5)=2  (约 20-30% 减伤)
DEF=8 → 减伤 4  (约 40-50% 减伤)
DEF=10→ 减伤 5  (约 50% 减伤)
```

**怪物属性增长公式**:
```

|------|------|------|

| 精英 | ×1.8 HP, ×1.5 ATK | 约 2 倍强度 |

## 4. 战斗平衡模型

### 4.1 TTK (击杀时间) 模型

```
玩家每秒伤害 (DPS) = (ATK + 3.5) / 攻击间隔 × (1 + 暴击率 × 0.5)

示例: 1 层开场
  ATK=4, 攻击间隔=1.0秒, 暴击率=16.7%
  DPS = (4 + 3.5) / 1.0 × (1 + 0.167 × 0.5) = 7.5 × 1.083 = 8.13

各层 TTK 预计:
                玩家DPS   普通怪HP   TTK(秒)  精英怪HP  TTK(秒)  Boss HP  TTK(秒)
1层(ATK=4)        8.1       8        1.0       16      2.0       20      2.5
2层(ATK=6)       10.3       12       1.2       22      2.1       30      2.9  
3层(ATK=8)       12.5       16       1.3       28      2.2       40      3.2
4层(ATK=10)      14.6       20       1.4       34      2.3       52      3.6
5层(ATK=12)      16.8       24       1.4       40      2.4       64      3.8
```

**验证**: TTK 在 1.0~3.8 秒之间, 符合"2 秒杀小怪, Boss 战约 4 秒"的设计目标。

### 4.2 TTD (死亡时间) 模型

```
玩家承受伤害 = 怪物DPS × (1 - DEF×0.5 / (ATK+3.5))

各层 TTD 预计 (玩家不操作、站桩):
               怪物DPS(1怪)  玩家减伤   有效DPS   玩家HP   TTD(秒)
1层(1怪ATK=3)     3.3         30%       2.3       25      10.9
2层(1怪ATK=5)     5.3         25%       4.0       25      6.3
3层(2怪ATK=7)     15.1        20%       12.1      30      2.5
4层(2怪ATK=9)     19.3        18%       15.8      30      1.9
5层(3怪ATK=11)    33.8        15%       28.7      30      1.0
```

**验证**: 
- 站着不动约 2~10 秒死亡 → 走位和翻滚是必须的, 不操作就会死
- 3 层开始 TTD 急剧下降 → 玩家必须利用地形和翻滚
- 目标: 有操作的玩家 TTD 约为站桩的 2~3 倍

### 4.3 战斗时长模型

```
一场战斗的时长 = 怪物数量 × TTK + 走位时间

示例 1 层战斗房 (3 只普通怪):
  击杀 3 怪 × 1.0 秒 = 3 秒输出时间
  走位/翻滚/调整: +10 秒
  总时长 ≈ 13 秒 ← 比目标 30~90 秒短!

→ 结论: 需要调整房间怪物数或增加战斗房密度

调整后 (战斗房改 4~6 只怪):
  1 层战斗房 (5 只普通怪): 5×1.0 + 12 = 17 秒 (仍然偏短)
  
→ 修正: 需要更长 TTK 或更多怪物

修正方案: 每层基础怪物数调整为:
  1层: 4~5 只普通怪 → 击杀时间 ≈ 4×1.0 + 15 = 19 秒
  3层: 4~5 只混合 → 击杀时间 ≈ 4×1.3 + 18 = 23 秒
  
→ 仍然偏短。

根因: 实时自动攻击的战斗节奏比回合制快得多。
     30-90 秒的战斗需要更多怪物或更慢的击杀。
     
修正 TTK 目标: 从"2 次攻击击杀"改为"3~4 次攻击击杀"
→ 降低玩家 ATK 或提高怪物 HP

修正后参数:
  1 层小怪 HP 从 8 → 12 (提高 50%)
  玩家 ATK 维持 4 → 伤害期望 7.5 → TTK ≈ 12/7.5 = 1.6 秒/怪
  5 只怪 + 走位 ≈ 5×1.6 + 15 = 23 秒 (仍然偏短)
  
  还需要更多怪物:
  1 层: 6~8 只小怪 (分 2 批出) → 6×1.6 + 20 = 30 秒 ✅
  3 层: 4~6 只混合 + 精英 → (4×1.3 + 1×2.2)+20 = 28 秒 ✅
  Boss 房: 1 Boss + 2~3 小怪 → (1×3.2 + 3×1.3)+10 = 17 秒
```

### 4.4 修正后怪物数量

| 层数 | 每战斗房怪物数 | 生成方式 |
|------|---------------|----------|
| 1 | 5~7 | 一次性全部出现 |
| 2 | 5~7 | 一次性 |
| 3 | 4~6 + 1 精英 | 或分 2 批 (第 1 批清完出第 2 批) |
| 4 | 5~7 + 1 精英 | 分 2 批 (精英最后出) |
| 5 | 4~6 + 1~2 精英 | 分 2~3 批 |

**验证后的战斗时长**:
```
1 层战斗房: 6 怪 × 1.6 秒 + 15 秒走位 = 25 秒 (在 30-90 秒目标内)
3 层战斗房: 5 怪 × 1.8 秒 + 1 精英 × 3.0 秒 + 18 秒 = 30 秒
Boss 战斗: 1 Boss × 4.0 秒 + 3 小怪 × 1.8 秒 + 12 秒 = 21 秒
```

✅ 调整后全部在 20~30 秒范围内, 加上回复/走位/探索, 整层约 3~4 分钟, 3 层约 10~12 分钟。

## 5. HP 经济模型 (每层消耗)

### 5.1 预期 HP 消耗

```
假设玩家每场战斗消耗 HP 约 20~40% (有操作):

层1: 3 战斗房 × 每次消耗 6~10 HP + Boss 消耗 8~12 HP
     总消耗: 26~42 HP
     提供回复: 回复房1次(8~15 HP) + 道具药水(10 HP)
     净消耗: 8~17 HP
     玩家 HP: 25 → 剩余: 8~17 → 可以接受
     
层3: 4 战斗房 × 每次消耗 10~18 HP + Boss 消耗 15~25 HP
     总消耗: 55~97 HP
     提供回复: 回复房1次(15~20 HP) + 药水若干
     但玩家 HP 此时约 30~40
     需要靠吸血、道具补给 → 否则会死
```

### 5.2 HP 回复来源

| 来源 | 回复量 | 出现频率 |
|------|--------|----------|
| 治疗符文站桩 | 2 HP/秒 | 战斗房中约 10% |
| 回复房 | 8~20 HP | 每层 0~1 个 |
| 药水 (商店买) | 10 HP | 商店必卖 |
| 吸血光环 (核心能力) | 15% 伤害量 | 玩家选才生效 |
| 回响之珠 (遗物) | 3 HP/技能使用 | 玩家选才生效 |
| Boss 前篝火 (DDA) | 5 HP | HP<30% 时保底出 |

**关键结论**: 不选任何回复类 Build 的玩家, 最多撑到第 3 层就会死。这是故意的——你需要通过 Build 来弥补续航。

```

## 7. 魂石经济模型 (长线)

### 7.1 魂石收入

```
结算魂石 = layerBase × floorMultiplier  +  bossBonus  +  talentBonus  +  adBonus

layerBase: 每层基础 = 5 + 层数 × 5    (1层=10, 3层=20, 5层=30)
floorMultiplier: 玩家到达的层数 (到达=结算)
bossBonus: 击杀 Boss +10 (每层 Boss)
talentBonus: 天赋"贪婪" ×1.15
adBonus: 看广告翻倍 ×2 (玩家选择)

示例:
  玩家死在第 3 层, 未看广告, 无天赋
  = 20 × 3 + 0 + 1.0 + 1.0 = 60 魂石
  
  同上 + 看广告翻倍
  = 60 × 2 = 120 魂石
  
  同上 + 贪婪天赋
  = 60 × 1.15 = 69 魂石
  (+看广告 = 138 魂石)
```

### 7.2 魂石消耗

| 解锁项 | 魂石价格 | 累计魂石(按顺序解) |
|--------|----------|-------------------|
| 初始携带道具 | 400 | 400 |
| 铁胃 (回复翻倍) | 250 | 650 |
| 贪婪 (魂石+15%) | 300 | 950 |
| 战士 (盾反) | 500 | 1450 |
| 狂战斧 (遗物) | 600 | 2050 |
| 弓手 (弹射箭) | 500 | 2550 |
| 回响之珠 (遗物) | 600 | 3150 |
| 初始 HP+5 ×3 | 600 | 3750 |
| 刺客 (穿影) | 800 | 4550 |
| 急速手套 (遗物) | 600 | 5150 |
| 初始 ATK+1 ×3 | 450 | 5600 |
| 法师 (元素共鸣) | 1000 | 6600 |
| 铁甲 (遗物) | 600 | 7200 |
| 探索者 (多1房间) | 400 | 7600 |
| 狂战士 (怒吼) | 1200 | 8800 |
| **全解锁总计** | **8800** | **—** |

### 7.3 解锁节奏

```
假设玩家平均死在第 3 层:
  每次基础魂石 = 10 × 3 = 30
  每局 30 秒广告翻倍后 = 60
  约 70% 的局会看翻倍广告

  → 期望每次收益 = 30 × 0.3 + 60 × 0.7 = 51 魂石/局
  → 全解锁需 8800 / 51 ≈ 173 局
  → 每局 10 分钟 → 约 29 小时游戏时间

如果玩家平均死在第 4 层:
  → 基础魂石 = 15 × 4 = 60
  → 期望每次收益 = 60 × 0.3 + 120 × 0.7 = 102 魂石/局
  → 全解锁需 8800 / 102 ≈ 86 局
  → 约 14 小时

[HYPOTHESIS] 需要决定目标全解锁时间:
  - 核心内容解锁 (角色+关键遗物) 应在 5~10 小时内完成
  - 全解锁约 15~20 小时 (对 IAA 游戏来说合理)
  
调整方案 (缩短解锁周期):
  - 增加基础魂石: 层数系数从 5 改为 8 → 1层=13, 3层=29, 5层=45
  - 改后: 平均 3 层 → 期望 70 魂石/局 → 全解锁约 60 局 → 10 小时
```

## 9. 修正总结

## 附录: 数值调整参数速查

```
// 核心调参 (adjust 这四个就能改变整体难度)
HP_SCALE:       25        // 玩家初始 HP
ATK_BASE:       4         // 玩家基础攻击
MONSTER_HP:     12+层×4  // 怪物基础 HP 公式
MONSTER_COUNT:  5         // 每房怪物基数
SOULSTONE_RATE: 8         // 魂石层数系数

// 派生值 (不用手动调)
DAMAGE_AVG = ATK + 3.5            // 伤害期望
DPS = DAMAGE_AVG / ATTACK_INTERVAL  // 秒伤
TTK = MONSTER_HP / DPS              // 击杀时间
```
