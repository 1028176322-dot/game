# 种子系统 · 详细规格

> **版本**: v1.0 | **日期**: 2026-06-25 | **设计者**: GameDesigner
> **系统层**: 元层 P2 | **依赖**: 全局 (所有随机系统的种子来源)

---

## 1. 种子生成

```
种子码: 6 位大写字母 + 数字 (36^6 = 21.7 亿种可能)

格式: [0-9A-Z]{6} (不含 I, O 等易混淆字符)

生成方式:
  玩家开始新游戏时:
    1. 取当前时间戳 (毫秒)
    2. 取玩家微信 OpenID 后 4 位 (作为盐值)
    3. hash(时间戳 + 盐值) → 映射到种子码
    4. 如果种子码与最近 10 局重复 → 重新生成

  或者: 玩家手动输入种子码 (分享复制)
```

## 2. 消费链 (谁能种子决定了什么)

```
种子码 (如 "AB3K7Z")
    ↓
主种子: createSeededRNG("AB3K7Z")

主种子按固定顺序消费, 生成各子种子:

消费 1 → 地牢生成子种子
  └→ 每层用子种子 hash("seed_floor_N") → 确保层与层独立
    └→ 房间布局 (DAG 结构可视化数据)
    └→ 房间类型分配 (战斗/宝箱/回复/商店/事件/强化)
    └→ 地形模板 (每房间的 6×6 矩阵)

消费 2 → 怪物装配子种子  
  └→ 按房间进入顺序, 依次消费
    └→ 身体模板 (6种)
    └→ 元素属性 (6种)
    └→ 核心技能 (12种)
    └→ 规模等级 (4种)
    └→ 变异因子 (10种)

消费 3 → 强化选项种子
  └→ 进入强化房时消费
    └→ 3 选 1 的具体选项 (从 A/B/C 三层池中抽)

消费 4 → 事件池种子
  └→ 进入事件房时消费
    └→ 场景选择
    └→ 事件后果分配

消费 5 → 商店库存种子
  └→ 进入商店时消费
    └→ 本次商店的在售商品

**注意**: 
  词缀系统不使用种子 (全随机) → 保证每次掉落不同
  即使同种子, 不同玩家打出来的装备词缀不同
```

## 3. 确定性保证

```
function createSeededRNG(seed: string) {
  // 使用 mulberry32 算法, 简单且确定性
  let h = hashString(seed);
  return function() {
    h |= 0; h = h + 0x6D2B79F5 | 0;
    let t = Math.imul(h ^ h >>> 15, 1 | h);
    t = t + Math.imul(t ^ t >>> 7, 61 | t) ^ t;
    return ((t ^ t >>> 14) >>> 0) / 4294967296;
  };
}

function hashString(str: string): number {
  let hash = 5381;
  for (let i = 0; i < str.length; i++) {
    hash = ((hash << 5) + hash) + str.charCodeAt(i);
    hash |= 0;
  }
  return hash;
}

// 子种子生成
function getSubSeed(mainSeed: string, systemName: string, key?: number): number {
  const input = key !== undefined 
    ? `${mainSeed}_${systemName}_${key}`
    : `${mainSeed}_${systemName}`;
  return hashString(input);
}
```
