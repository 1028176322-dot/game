## 2. 为什么不能继续沿用旧 prompt

旧 `prompts.json` 虽然资源数量已经补齐到 499，但正文中大量出现负向词和危险词，例如：

```text
No skull
do not
avoid
blood
heart
organ
skeleton
horror
English
letters
watermark
pseudo-writing
```

这类写法的问题是：即使本意是禁止，AI 生成模型仍可能被这些词激活，继续生成骷髅、心脏、英文、伪文字等问题。

因此本次采用“重建式统一”，不在旧 prompt 上做替换，而是按资源路径、类别、尺寸和用途重新生成正向提示词。

## 3. 新 prompt 的统一结构

每条 prompt 都由以下模块组成：

```text
1. Unified visual direction
2. Release quality target
3. Runtime copy rule
4. Safe world motif
5. Target canvas
6. Runtime format intent
7. Asset role
8. Category-specific composition
9. Production note
```

### 3.1 统一风格

所有资源都统一到：

```text
bright cheerful cartoon animal adventure
warm hand-painted animation look
rounded friendly shapes
soft forest light
clean toy-like materials
saturated natural colors
gentle highlights
integrated forest-and-gem interface style
```

### 3.2 正向文本规则

旧写法：

```text
No text, no letters, no English, no watermark.
```

新写法：

```text
Runtime copy rule: all readable copy is provided by Cocos labels and game configuration; the artwork uses blank calm label areas, decorative leaf curls, paw motifs, stars, flowers, gems, ribbons, ropes, wood trims, soft cloth shapes, and simple geometric ornaments only.
```

这样不再把危险词或禁止词发送给生图模型。

## 5. 生成脚本

新增脚本：

```text
E:/game/tools/normalize_prompts_positive_style.py
```

脚本职责：

```text
1. 扫描正式资源目录。
2. 为全部正式图片生成 prompt。
3. 按路径识别类别、模块、角色、动作、区域。
4. 自动替换高风险路径词为安全画面语义。
5. 读取图片尺寸并写入 Target canvas。
6. 覆盖 prompts.json。
7. 自动备份旧文件。
8. 输出风险扫描报告。
```

运行命令：

```powershell
python E:/game/tools/normalize_prompts_positive_style.py
```

## 9. 注意事项

1. `prompts.json` 的 key 必须和正式资源路径一致。
2. 路径可以保留历史名称，例如 `death`、`boss`，但 prompt 正文必须使用安全替换语义。
3. 体积限制不要写进 prompt，继续由 `art_quality_budget.json` 管。
4. 生图失败时不要往 prompt 里追加危险负向词。
5. 所有 UI 图继续保持“图中无字，文字走 Label/TextManager”。
6. 如果未来要更改整体风格，只改 `normalize_prompts_positive_style.py` 的统一合同，再重新生成 `prompts.json`。
