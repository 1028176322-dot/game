# 3D 预览基础设施改造方案（根源修复，非补丁）

> 目标：在 2D UI 场景（main / splash）里正确、可复用地内嵌 3D 模型，
> 消除"每个面板各自动态造相机 / 挂场景根 / 找层"的绕过式补丁，
> 把它收敛为集中的场景基础设施 + 统一服务。

---

## 1. 问题根因（已确认）

| 场景 | 主相机 `_projection` | 能否渲染 3D |
|---|---|---|
| `dungeon.scene` | `1`（透视） | ✅ |
| `main.scene` | `0`（正交 2D） | ❌ |
| `splash.scene` | `0`（正交 2D） | ❌ |

- UI 场景用正交 2D 相机，**架构上从未为"2D UI 内嵌 3D"预留任何设施**。
- 现状补丁：`CreatePanel._updateModelDisplay` 里**每个面板自己** `new Node` 造透视相机、挂 `director.getScene()` 根、调内部 `_resolvePreviewLayer()` 找独立层、再递归设子树 layer。
- 后果：相机/层/根逻辑散落、重复，多面板共存时易串渲染，且 CreatePanel 承载大量 `[DIAG]` / `console.warn` 调试残留。

**结论**：根因不是"哪个面板没显示"，而是**缺少统一的 3D 预览能力层**。

---

## 2. 设计原则（根源 vs 补丁的分界）

| 维度 | 现状（补丁） | 本方案（根源） |
|---|---|---|
| 预览相机 | 每面板运行时各自 `new` | 全局**单例**透视预览相机，集中管理 |
| 预览层 | 每面板自己解析 `DEFAULT` 层 | 统一定义的专用层 `UI_3D_PREVIEW` |
| 预览根 | 直接挂 `director.getScene()` 根 | 统一 `UIPreviewRoot` 容器节点 |
| 面板代码 | 几十行相机/层/挂接逻辑 | 一行 `SceneModelPreview.instance.showCharacter(...)` |
| 诊断残留 | 大量 `[DIAG]` / `console.warn` | 删除，回归生产级 |

---

## 3. 新增基础设施：`SceneModelPreview` 服务

**文件**：`assets/scripts/render/SceneModelPreview.ts`（新建）

### 3.1 内部设施（单例、集中）

- **预览层**：专用层 `UI_3D_PREVIEW`。
  - 取值：`1 << 22`（Cocos 用户层 22，bitmask = `4194304`）。所有 3D 预览模型挂此层。
  - 理由：与 `UI_2D`（bitmask `33554432`，即层 25）严格隔离，透视相机只看预览层，绝不重绘 2D UI。
- **预览相机**（单例）：
  - 查找策略：先找场景中名为 `UIPreviewCamera` 的节点（未来可在编辑器预置，彻底固化）；找不到则**单例运行时创建**并挂在场景根。
  - 配置：`projection = PERSPECTIVE`，`fov = 45`，`near = 1`，`far = 2000`，`clearFlags = DEPTH_ONLY`（保留 UI 颜色缓冲），`visibility = UI_3D_PREVIEW`，`priority = 1`（在 UI 正交相机之后绘制）。
  - 位置：相机固定在 `(anchor.x, anchor.y, 700)` 看向 -z；模型置于 z=500，二者相距 200，稳定落在透视视锥内。
- **预览根**（单例）：`UIPreviewRoot` 节点（场景预置或运行时创建），承载所有预览模型子树，避免污染场景根。

### 3.2 公开 API

```ts
interface PreviewHandle {
  node: Node;
  destroy(): void;          // 销毁模型节点，归还句柄（不销毁单例相机/根）
}

interface ShowOptions {
  action?: string;          // 默认 'idle'
  yOffset?: number;         // 相对锚点的 y 偏移，默认 0
  zOffset?: number;         // 相对锚点的 z 偏移，默认 500
  targetHeight?: number;    // 自动缩放目标高度（像素空间），默认 120
  forceUnlit?: boolean;     // UI 预览默认 true（不依赖 PBR 光照）
  rotate?: boolean;         // 是否持续旋转（背景用），默认 false
}

class SceneModelPreview {
  static get instance(): SceneModelPreview;

  // 按角色 id 显示 3D 角色（anchorWorldPos 来自 2D 锚点的 worldPosition）
  showCharacter(anchorWorldPos: Vec3, characterId: string, opts?: ShowOptions): Promise<PreviewHandle>;

  // 按模型资源 id 直接显示 3D 模型（背景用）
  showModel(anchorWorldPos: Vec3, modelAssetId: string, opts?: ShowOptions): Promise<PreviewHandle>;
}
```

### 3.3 实现要点

- `showCharacter` 内部：在 `UIPreviewRoot` 下建 `CharacterPreview` 节点 → 设 layer = `UI_3D_PREVIEW` → `setWorldPosition(anchor.x, anchor.y + yOffset, anchor.z + zOffset)` → 调 `CharacterVisualService.instance.play(node, 'character.${id}.${action}', 8, forceUnlit, UI_3D_PREVIEW)`。
- `showModel` 内部：同上，但调 `CharacterModelAssembler.instance.mount(node, modelAssetId, undefined, 'Weapon', action, forceUnlit, UI_3D_PREVIEW)`。
- `rotate` 选项：用 `schedule` 每帧绕 Y 轴旋转模型节点（仅背景用）。
- 相机/根节点在多次 show 之间复用（单例），不随每个 handle 销毁；进程级常驻，符合"基础设施"语义。
- 复用 `CharacterVisualService` / `CharacterModelAssembler` 已有的 `targetLayer` 参数（已具备），不自造渲染逻辑。

## 4. 各面板改造清单

### 4.1 CreatePanel（删补丁，走服务）
**文件**：`assets/scripts/ui/main/CreatePanel.ts`

- 删除：`_resolveUiLayer()` / `_resolvePreviewLayer()` / `_setSubtreeLayer()` / `_previewCamera` 字段 / `_previewNode` 字段里的全部动态造相机逻辑、`import { Camera }` 中与预览相关的部分。
- 新增字段：`private _handle: PreviewHandle | null = null;`
- `_updateModelDisplay(id)` 改为：
  ```ts
  if (this._handle) { this._handle.destroy(); this._handle = null; }
  const anchor = this._modelDisplay.worldPosition;
  this._handle = await SceneModelPreview.instance.showCharacter(
    anchor, id, { action: 'attack', yOffset: -60, zOffset: 500, targetHeight: 120, forceUnlit: true }
  );
  ```
- `close()` 改为 `this._handle?.destroy(); this._handle = null;`（替代原 `_previewNode` / `_previewCamera` 移除逻辑）。
- 保留 `_prepareIntegratedBackground()` 的"仅禁用 Sprite、不 active=false"修复（这是正确修复，不是补丁）。

### 4.2 CharacterPanel（审计最高优先级缺口：当前完全没有角色形象）
**文件**：`assets/scripts/ui/main/CharacterPanel.ts`

- 面板内新增 3D 预览锚点（运行时 `new Node('ModelAnchor')` + `UITransform`，布局到面板合适位置，如右侧）。
- `_refresh()` 选中角色时：`this._handle = await SceneModelPreview.instance.showCharacter(anchor, selectedId, { action: 'idle', targetHeight: 160, forceUnlit: true });`
- `close()`：`this._handle?.destroy();`

### 4.3 MainMenuBackdrop（已写好但未接入 + 直接挂 UI 节点无效）
**文件**：`assets/scripts/ui/main/MainMenuBackdrop.ts` + `MainSceneController.ts`

- `MainMenuBackdrop` 改造：移除"直接把 `ModelComponent` 挂 UI 节点"的无效做法，改用 `SceneModelPreview.instance.showModel(centerAnchor, backdropAssetId, { rotate: true, targetHeight: 400, forceUnlit: true })` 生成可旋转 3D 背景。
- `MainSceneController._ensurePersistentMainBackground`：把 `MainMenuBackdrop` 组件挂到 `MainBackground` 节点并设置 `backdropAssetId`（配置驱动，缺失则静默 no-op，不报错、不崩）。

### 4.4 SplashUI（启动界面 3D 背景）
**文件**：`assets/scripts/ui/SplashUI.ts`

- 配置驱动：若配置存在 `splash.backdropAssetId`，用 `SceneModelPreview.instance.showModel(centerAnchor, id, { rotate: true, targetHeight: 500, forceUnlit: true })` 在屏幕中心生成 3D 背景；缺失则保持原 2D 背景（`splashImage` + `ui.splash.bg`），降级无副作用。

## 5. 诊断代码清理（贯穿，回归生产级）

以下调试残留应删除或降级为 `console.debug`：

- `CreatePanel.ts`：所有 `[DIAG]` 注释、`console.warn('[CreatePanel] preview camera created...')`。
- `CharacterModelAssembler.ts`：`mount` / `_autoFitScale` / `_forceUnlitMaterials` / `_ensureVisibility` / `_makeMaterialsEmissive` 内的 `console.warn` 调试输出（保留关键错误 `console.warn` 如 prefab 加载失败）。
- `CharacterVisualService.ts`：`applyStatic` / `play` 内的 `console.warn('[CharacterVisualService] try mount 3D...')` 等调试输出。

## 8. 风险与回滚

- **风险 A（改 .scene）**：本方案**不修改** `main.scene` / `splash.scene` 的相机 JSON，预览相机走运行时单例回退，避免编辑器资源风险。后续若要彻底固化，可在编辑器预置 `UIPreviewCamera` + `UIPreviewRoot` 节点，服务会自动复用（无需改代码）。
- **风险 B（层冲突）**：`UI_3D_PREVIEW = 1<<22` 与现有 `UI_2D`（层 25）不重叠；若项目已占用层 22，改用其他空闲用户层（22–30 间选一）。
- **回滚**：任一面板回归只需将其 `showCharacter` 调用替换为原 2D `RenderAssetService.applySpriteById` 路径；服务本身无副作用，可整体保留。
