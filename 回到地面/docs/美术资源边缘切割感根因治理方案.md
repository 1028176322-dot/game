# 美术资源边缘切割感根因治理方案

日期：2026-07-08

适用项目：Cocos Creator 3.8.8 项目《回到地面》

适用目录：

```text
E:/game/回到地面/assets/resources/textures
E:/game/回到地面/art_source/textures_review/master
E:/game/回到地面/art_source/textures_export/runtime_replace
E:/game/assets/resources/config/prompts.json
```

---

## 1. 问题结论

当前部分资源出现“四周明显切割感”，不是单纯压缩或尺寸问题，而是资源类型定义错误导致的。

典型问题资源：

```text
ui/create/btn_create_confirm.png
ui/character/character_card_selected.png
ui/character/character_card_default.png
ui/area/area_badge_forest.png
icons/items/icon_item_key.png
```

表现：

```text
1. 资源四周带完整矩形背景。
2. 阴影或光效被画布边缘截断。
3. 图标带了花草/场景底图。
4. UI 卡片像一张小插画贴在界面上。
5. 透明 UI 资源实际不是透明组件，而是完整小画框。
```

根因：

```text
AI 生成时把资源理解为“完整插画/完整面板/完整小场景”，而游戏实际需要的是“可叠加、可拉伸、可复用的透明 UI 组件或单体对象”。
```

所以不能靠单张图片修边解决，必须从资源分类、prompt、后处理、门禁和 Cocos 配置一起治理。

---

## 2. 总体原则

所有美术资源必须先归类，再按类别生成。

```text
整屏背景类：
  允许完整矩形画面，不需要透明。

透明 UI 组件类：
  外部必须透明，主体不能贴边，阴影不能截断。

9-slice UI 类：
  外部透明，四角保留装饰，边线可拉伸，中心可读。

单体图标类：
  只保留单个物品/符号，外部透明。

角色/怪物/Boss 类：
  只保留角色主体，外部透明，逐帧一致。

特效类：
  只保留抽象能量形状，外部透明，帧格清晰。

Tile 类：
  可拼贴地表，不要求透明，不允许中心图标和边框。
```

核心判定：

```text
如果资源会叠在界面、按钮、卡片、角色、图标上：
  必须是透明 PNG 组件。

如果资源本身就是页面背景或战斗背景：
  可以是完整 JPG/PNG 矩形。
```

## 4. 后处理流程

### 4.1 透明资源标准流程

适用：

```text
透明按钮
透明卡片/面板
徽章
图标
角色
怪物
Boss
特效
```

流程：

```text
1. 生成高清母版
2. 判断资源类型
3. 从画布边缘 flood-fill 外部背景
4. 删除外部连通背景
5. 保留主体内部浅色/空洞区域
6. 自动扩透明 padding
7. 检查外圈 alpha
8. 检查主体是否贴边
9. 检查阴影/光效是否截断
10. 输出 RGBA PNG
```

不要使用：

```text
全图颜色阈值硬抠
全局 chroma key 误删主体
强行 pngquant 到 P 模式
```

### 4.2 背景资源标准流程

适用：

```text
整屏背景
战斗背景
一体化界面背景
```

流程：

```text
1. 生成高清母版
2. 检查无文字/无违规
3. 检查预留 UI 区域
4. 派生 runtime JPG
5. JPG quality 88-92
6. 检查无明显压缩糊
```

## 5. 自动门禁规则

### 5.1 透明边缘检查

```python
def check_outer_alpha(image):
    # transparent resources only
    # outer 2px alpha should be 0 or near 0
    pass
```

判定：

```text
最外 2px 非透明像素比例 <= 1%
四角 8x8 区域非透明比例 <= 1%
```

不通过：

```text
fail_edge_alpha
```

### 5.2 主体留边检查

判定：

```text
主体 alpha bbox 距画布边缘 >= 6px
按钮/卡片 >= 8px
角色/Boss >= 8%-12%
```

不通过：

```text
fail_subject_padding
```

### 5.3 矩形背景检查

判定：

```text
透明类资源如果大面积连续不透明背景铺满画布，判失败
```

不通过：

```text
fail_rect_background
```

### 5.4 阴影截断检查

判定：

```text
边缘 alpha 有连续半透明阴影，说明阴影碰边
```

不通过：

```text
fail_shadow_clipped
```

### 5.5 9-slice 检查

判定：

```text
四角细节集中
边线无复杂局部图案
中心低对比
拉伸区域连续
```

不通过：

```text
fail_nine_slice
```

## 6. prompts.json 修改方向

当前 `prompts.json` 已经统一为正向风格，但还需要进一步细化“透明组件”和“整屏背景”的差异。

必须避免透明组件 prompt 出现：

```text
background
scene
full canvas
matte background
full rectangular
complete illustration
```

透明组件必须出现：

```text
Transparent PNG
reusable overlay asset
clean alpha transparency around it
centered with transparent padding
soft shadow fades before reaching canvas edge
```

整屏背景必须出现：

```text
full-screen integrated background
one continuous painting
full opaque rectangular image
runtime zones naturally embedded
```

图标必须出现：

```text
standalone object
single centered object
transparent padding
only the object is visible
```

## 13. 给执行 AI 的要求

执行时必须遵守：

```text
1. 先分类，再生成。
2. 不要用同一套 prompt 生成所有资源。
3. 透明 UI 不允许生成完整背景图。
4. 图标只生成单体对象。
5. 整屏背景才允许完整矩形场景。
6. 后处理必须做外圈 alpha 检查。
7. 失败资源不要靠裁剪硬修，优先重做 prompt 和生成规格。
8. 体积不是第一失败条件，边缘透明和视觉正确性优先。
9. 所有 UI 图不含文字，文字走 Label/TextManager。
10. 更新 prompts.json 时必须保持正向描述，不回到 no/do not/avoid 写法。
```
