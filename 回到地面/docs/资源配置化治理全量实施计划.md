## Phase 1 — game_assets.json + GameAssetService

**目标**：非 UI 资源语义注册表建好，服务能查询。

### 1.1 game_assets.json

覆盖 7 类非 UI 资源：

| 分类 | key 示例 | assets.json 条目数 |
|------|---------|:------------------:|
| characters | `character.warrior.idle` | 35（5角色×7动作） |
| monsters | `monster.forest.slime.idle` | 36（6区×6怪） |
| bosses | `boss.forest.guardian.idle` | 60（10Boss×6动作） |
| backgrounds | `background.forest.combat` | 17 |

| effects | `effect.reaction.burn` | 27（combat+reactions+relics+ui） |
| icons | `icon.skill.dash` | 67（buffs+elements+items+relics+sets+skills） |

## Phase 3 — 业务配置表迁移

**目标**：业务配置不再写 `assetId`，改写 `game_assets.json` 的语义 key。

### 3.1 characters.json → 加 `visual` 字段

```json
{
  "id": "warrior",
  "visual": {
    "idle": "character.warrior.idle",
    "attack": "character.warrior.attack",
    "death": "character.warrior.death"
  }
}
```

### 3.2 monsters.json → 加 `visual` 字段

类似，按 zone+monster。

### 3.3 bosses.json → 加 `visual` + `scale` + `effect`

### 3.4 zones.json → 加 `backgrounds` + `tileset`

```json
{
  "id": "forest",
  "backgrounds": {
    "combat": "background.forest.combat",
    "shop": "background.forest.shop"
  },
  "tileset": {
    "floor": "tile.forest.floor",
    "wall": "tile.forest.wall"
  }
}
```

### 3.5 skills.json / items.json / equipment.json → 加 `icon` / `effect`

**验收**：所有业务配置引用 key 均通过 `check_game_assets_registry.py`。

## Phase 4 — 运行时系统迁移

**目标**：游戏实际渲染走新管线。

### 4.1 DungeonSceneInstaller

| 改什么 | 现状 | 改成 |
|--------|------|------|
| `loadInitialArt()` | 硬编码 `ArtResourceResolver.backgroundCombat(zoneId)` | `BackgroundService.apply(node, \`background.${zoneId}.combat\`)` |
| 角色加载 | 硬编码 `ArtResourceResolver.character(characterId, 'idle')` | `CharacterVisualService.play(playerNode, \`character.${characterId}.idle\`)` |
| 怪物加载 | 硬编码 `ArtResourceResolver.monster(zoneId, monsterId, 'idle')` | `CharacterVisualService.play(monsterNode, \`monster.${zoneId}.${monsterId}.idle\`)` |

### 4.2 GridManager

| 改什么 | 现状 | 改成 |
|--------|------|------|
| tile 加载 | `RenderAssetService.applyTileSprite(node, zoneId, terrain)` | TileAssetService.getTileAssetId → `RenderAssetService.applySpriteById` |

### 4.3 BattleManager / PlayerController / EffectSystem

| 改什么 | 现状 | 改成 |
|--------|------|------|
| 特效 | 硬编码 create/play | `EffectService.play()` |
| 图标 | 硬编码路径 | `IconService.apply()` |

**验收**：三场景进入后所有视觉效果与迁移前一致，无加载失败 warning。

扫描所有 `.ts` 文件中的 `textures/` 字符串引用，确认无残留硬编码：

```bash
grep -rn "'textures/" assets/scripts/ --include="*.ts"
grep -rn '"textures/' assets/scripts/ --include="*.ts"
grep -rn '`textures/' assets/scripts/ --include="*.ts"
```

`check_config_registry.py` 的结果里可能扫描到 `.ts` 文件中的中文注释，按 ASCII_SOURCE_POLICY 转为英文。

## 执行建议

1. **Phase 0 优先级最高** — UI 侧底座已经建了一半，先打通链路让 UI 能实际跑起来
2. **Phase 1（game_assets.json）最耗时** — 266+ 条资源都要写 metadata（帧数/尺寸等），建议脚本辅助生成
3. **Phase 2（服务实现）与 Phase 3（配置迁移）可并行** — 服务接口定了之后，两边可以同时改
4. **Phase 4 风险最高** — 改动 DungeonSceneInstaller 和 GridManager 可能影响战斗画面，必须逐个场景验证
5. **每个 phase 完成后跑 `npm.cmd run validate:all` + 浏览器预览**
