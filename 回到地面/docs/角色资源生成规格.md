# 角色资源生成规格

> 关联文档: `game_assets.json`（35 条角色定义）、`ART_RESOURCE_RULES.md`（规格标准）
> 管线: `art_pipeline.py generate --category characters`

---

## 一、格式规范

| 参数 | 值 | 说明 |
|------|:---:|------|
| 单帧尺寸 | **192×192** | 竖排叠加 4 帧 |
| 总尺寸 | **192×768** | 192px宽 × (192×4)px高 |
| 布局 | **竖排 vertical** | 帧 0 在顶部，帧 3 在底部 |
| 格式 | PNG RGBA | 带透明通道 |
| 体积预算 | warning=900KB / hard=1229KB | 当前文件 ~130-185KB |
| 帧数 | **4 帧** | 与游戏所有角色统一 |

### 为什么是竖排 4 帧

```
SpriteAnimationService 只支持 vertical/horizontal
BUILTIN_SHEETS 全部定义 frameCount=4, vertical
game_assets.json 35 条全部 layout=vertical, fc=4
所以必须竖排 4 帧，不可换其他布局
```

## 三、5 个职业的角色描述

| 职业 | 动物 | 主色 | 武器 | 体型 |
|:----:|:----:|:----:|:----:|:----:|
| **warrior** | 🦌 鹿 | 蓝银 | 剑盾 | 健壮 |
| **archer** | 🦌 鹿 | 绿 | 弓 | 灵活 |
| **assassin** | 🐱 猫 | 紫黑 | 匕首×2 | 纤细 |
| **berserker** | 🦁 狮 | 红金 | 大斧 | 魁梧 |
| **mage** | 🐰 兔 | 蓝紫 | 法杖 | 娇小 |

### 通用视觉要求（所有动作）

- 卡通动物冒险风格，明亮饱和色彩
- 全身入镜，站立在画面中心
- 每个角色每帧位置一致（对齐轴心）
- 浅色背景，边缘透明
- 无文字、无水印、无签名
- 简单干净、低细节、高可读性

## 五、若要重新生成

```bash
# 先 audit 确认哪些需要重做
python tools/art_pipeline.py audit --category characters

# 全量生成（注意 prompt 须包含 "192x768" 以指示尺寸）
python tools/art_pipeline.py generate --category characters

# 验证门禁
python tools/art_pipeline.py validate --category characters
```

**关键**：prompt 中必须包含 `192x768` 或 `4 frames vertical`，否则管线会退到 128×128 或网格布局。当前已有 archer_idle 生成了 256×256 3×3 网格的问题，就是因为 prompt 里没写清楚尺寸和布局。
