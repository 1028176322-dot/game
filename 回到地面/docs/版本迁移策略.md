## 2. 合并门禁（合回 master 前必须全绿）

每个 feature 分支在发起 MR/PR 前，本地执行：

```bash
# 1. 单元测试（纯 TS，node 运行，不依赖编辑器）
npm.cmd run test            # vitest run，当前 167 passed

# 2. 编码审计（UTF-8 完整性）
python tools/encoding_audit.py --ci   # 必须 issues=0, p0=0

# 3. 全量门禁（9 道）
npm.cmd run validate:all    # 必须 9/9

# 4. 分支门禁（CI 用，等同上面三项的封装）
python tools/branch_gate.py
```

**合入硬性条件**（决策 D / 红线 1–6）：
- 所有新增 System 必须 `implements ILifecycle`，否则 `check_architecture.py` 拦截；
- 禁止 `new` 引擎底层（`director` / `PhysicsSystem` / `instantiate` 等），仅 `SceneFlowService` 可调 `director.loadScene()`；
- 禁止 `switch(skillId)`（技能走 `SkillGraph` 数据驱动）；
- 禁止 `Math.random`（用 `Rng` xorshift32 确定性）；
- 业务代码禁止 `new` 服务（全部 `ctx.get` 注入）。

## 3. 冻结 2D 时机

触发条件（满足全部才冻结）：
1. 垂直切片（§7.3）通过——`tests/core/vertical_slice.test.ts` 端到端全绿；
2. 全量内容开始从 2D 复制/重写为 3D（美术轨推进到 Characters/Bosses 阶段）。

冻结动作：
- 在 2D 实现文件头标注 `@deprecated`，写明替代的 3D System；
- 冻结后不再接受 2D 逻辑改动，仅允许紧急 hotfix（需 maintainer 批准）。

待冻结清单：

- `battle/MonsterController.ts` / `battle/entity/MonsterAgent.ts` → 由 `IAIController` + ECS 替代
- `SpriteAnimationService` → 由 `AnimationStateMachine` 替代

## 5. 具体 git 操作示例

```bash
# 从 master 拉出 feature 分支
git checkout master && git pull
git checkout -b feature/combat

# 开发完成、门禁全绿后推送
git push -u origin feature/combat

# 发起 MR，CI 跑 branch_gate.py，通过后合回 master（禁止 --no-verify / 禁止 -f 强推）

# 冻结 2D：在文件头加 @deprecated 注释（不删）
# 删除 2D：Phase 6 末，确认无引用后整文件删除并单独 commit
git rm assets/scripts/battle/BattleManager.ts
git commit -m "[chore] remove legacy 2D BattleManager (decision F, Phase 6)"
```
