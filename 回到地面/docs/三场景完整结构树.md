# 三场景完整结构树（2D → 3D 升级目标）

> 适用范围：splash.scene / main.scene / dungeon.scene  
> 本文档记录每个场景在 2D→3D 全面升级（依据 `../docs/2D转3D全面升级方案.md` v3）**目标完成后的节点层级、组件绑定、脚本字段映射**，方便查改。  
> **当前状态（2026-07-11）**：本文为「3D 目标结构」蓝图。真实 `.scene` 文件按 `feature/3d-*` 分支分阶段落地（垂直切片 + Phase 0–6，决策 F），落地后回填「现状」标注。迁移前 2D 基线见 **附录 A**。  
> 注：UI 类（splash/main 面板、icons、HUD）按方案 §4.1 **保持 2D**，故 main.scene 结构基本不变；真正 3D 化集中在 dungeon.scene 与角色/怪物/Boss 表现层。

## 约定说明

| 标记 | 含义 |
|------|------|
| `[组件名]` | 节点上挂载的 Cocos 组件 |
| `(Script: XxxPanel)` | 节点上挂载的自定义脚本 |
| `field=节点名` | 脚本字段绑定的子节点 |
| `LocalizedLabel(key)` | 挂载了 `LocalizedLabel` 组件，textKey 为 key |
| `*` | 运行时由代码动态创建的子节点 |
| `!` | 编辑器绑定的必填项 |
| `✅` | 已在场景文件中确认存在 |
| `❌` | 场景文件中缺失，需手动创建 |
| `?` | 需在编辑器中手动确认是否存在 |
| `← 3D 新增` | 相对 2D 基线新增/替换的节点或组件（详见附录 A 旧结构） |
| `@deprecated` | 兼容壳，Phase 6 末删除（决策 F） |

# 1. splash.scene — 启动场景（3D 目标）

UI 全部保留 2D（方案 §4.1：splash/login 保持 2D）。本场景相对 2D 基线**几乎不变**，唯一实质变化是 `GameBootstrap` 现额外引导 3D 基础设施。

```
splash
├── LightingRoot              [LightingService]            ← 3D 新增：方向光+阴影+雾（§3.14），接管原 Main Light
├── Main Camera               [CameraBrain]               ← 3D 新增：启动场景用基础 Follow 模式（§3.4）
├── GameBootstrap             [Component: GameBootstrap]  ← 现额外注册 GameContext / LifecycleManager / ConfigDatabase / Logger / AssetCache / EventBus（§5）
└── Canvas                   [UITransform]
    └── SplashUI             [UITransform, SplashUI, SplashLayout]
        ├── SkipButton
        │   └── Label        [Label] LocalizedLabel(ui.skip)
        ├── SplashImage      [UITransform, Sprite]        （2D UI 保留）
        ├── LoadingLabel     [Label] LocalizedLabel(ui.loading)
        └── LoadingBar       [UITransform]               （空容器, 代码填充 Graphic 进度条）
```

## 1.1 SplashUI 字段绑定

| SplashUI 字段 | 类型 | 绑定到 |
|--------------|------|--------|
| `skipLabel` | Label | `SplashUI/SkipButton/Label` |
| `splashImage` | Node | `SplashUI/SplashImage` |
| `loadingLabel` | Label | `SplashUI/LoadingLabel` |
| `loadingBar` | Node | `SplashUI/LoadingBar` |

## 1.2 SplashLayout 字段绑定

| SplashLayout 字段 | 类型 | 绑定到 |
|------------------|------|--------|
| `splashImage` | Node | `SplashUI/SplashImage` |
| `loadingBar` | Node | `SplashUI/LoadingBar` |
| `loadingLabel` | Node | `SplashUI/LoadingLabel` |
| `skipButton` | Node | `SplashUI/SkipButton` |

## 1.3 关键说明

- `GameBootstrap` 放在场景根节点下（不在 Canvas 内），作为全局启动服务；3D 化后它额外创建 `GameContext` 并注册全部服务（`ICollisionService`/`IAssetCache`/`IAnimationController`/`IAIController`/`ICameraBrain`/`IEventBus`/`IConfigDatabase`/`ILogger`/`ISaveManager`/`IDebugService`/`IAudioService`/`IRuntimeState`/`IReplayRecorder`，§5.2）。
- `LoadingBar` 是空容器节点，代码在 `SplashUI._createProgressBar()` 中向其添加 `Graphics` 子节点绘制进度条。
- `LoadingLabel` 初始文本由 `LocalizedLabel(ui.loading)` 提供，加载过程中被代码动态消息覆盖。
- 场景 UI 节点层级与附录 A §1 完全一致，无结构改动。

# 3. dungeon.scene — 地牢战斗场景（3D 目标，重头戏）

本场景是 2D→3D 升级的核心。相对 2D 基线（附录 A §3）的主要变化：
- 光照/相机升级为 `LightingService` + `CameraBrain`（§3.4 / §3.14）。
- `Player` 由 `Sprite` 改为 `ModelComponent`/`SkinnedMeshRenderer`，`PlayerController` 拆为 6 个组件（§3.12）。
- `World` 渲染层由 2D 贴图升级为 3D 模块件（§3.7 RoomBuilder）。
- 战斗由胖 `BattleManager` 拆为 `CombatSystem`/`TargetSelector`/`HitResolver`/`DamageResolver`/`EffectExecutor`/`ProjectileSystem`/`LockOnManager`（§3.8）。
- 碰撞统一走 `ICollisionService`（§3.3），业务禁直连 `PhysicsSystem`。
- 新增**世界空间 UI**（§2.4）：敌人血条 / 伤害飘字 / 锁定 reticle / 技能范围圈 / Boss 血条。

├── Canvas

## 3.3 运行时节点生命周期（3D）

```
DungeonSceneInstaller.install()
  ├── LightingService.init()        (方向光/阴影/雾/天空盒)
  ├── CameraBrain.setMode("Follow") (透视俯角 35°–50°)
  ├── Canvas/World (3D 渲染层: Background → Tile → Actor → Effect → Door, 由 RoomBuilder 拼装)
  ├── Canvas/Systems (CombatSystem/TargetSelector/.../ICollisionService, 由 GameContext 注入)
  ├── Canvas/UIRoot (UI 容器 + WorldSpaceUI billboard 投影)
  └── LifecycleManager.enterAll()
        └── RoomRuntime.enter() / BattleRuntime.enter() / 各 System.enter()
```

## 3.4 关键说明

- **Player 表现层**：`Sprite` → `ModelComponent`/`SkinnedMeshRenderer`；`PlayerController` 拆为 6 组件（Movement/Animation/Combat/Stat/Target/Interaction），由 `LifecycleManager` 统一管理（§5.1）。
- **碰撞**：实时碰撞走 `ICollisionService`（`overlapSphere`/`raycast`/`checkGround`），业务禁止 `import PhysicsSystem`（§3.3）；`GridManager.isWalkable` 仍用于逻辑/生成。
- **战斗分层**：`BattleManager` 仅作 `@deprecated` 兼容壳，最终删除（决策 F / §6.3 门禁）；实际调度由 `CombatSystem` 等 7 子系统承担（§3.8）。
- **地牢拆分**：`GridManager`/`DungeonManager` 拆为 `DungeonGenerator`→`RoomBuilder`→`TileMap`→`NavigationGrid`→`RoomRuntime`（§3.7）；第一阶段继续 Grid 寻路（决策 C）。
- **世界空间 UI**：新增 `WorldSpaceUI` 节点，承载 billboard 血条、伤害飘字、锁定 reticle、技能范围圈、Boss 血条（§2.4）。
- **AI**：怪物挂 `AIComponent` + `IAIController`，策略可配置 BT/FSM/GOAP/Utility（§3.10）；AI 只产出 `MoveCommand`/`SkillRequest`，由战斗子系统执行。

# 4. 共用底座组件（保留）

## 4.1 ResponsivePanelRoot

| 属性 | 默认值 | 说明 |
|------|--------|------|
| `dimMask` | (Node) | 绑到 `PanelRoot/DimMask` |
| `panelFrame` | (Node) | 绑到 `PanelRoot/PanelFrame` |
| `frameWidthRatio` | 0.72 | PanelFrame 宽度 = canvasW × 此比例 |
| `frameHeightRatio` | 0.78 | PanelFrame 高度 = canvasH × 此比例 |
| `maxFrameWidth` | 900 | PanelFrame 最大宽度 |
| `maxFrameHeight` | 620 | PanelFrame 最大高度 |
| `minFrameWidth` | 520 | PanelFrame 最小宽度 |
| `minFrameHeight` | 360 | PanelFrame 最小高度 |

生命周期：
1. `onLoad` → `applyLayout()` + 监听 `canvas-resize` / `design-resolution-changed`
2. `onEnable` → `applyLayout()` + `scheduleOnce(applyLayout, 0)`（延迟重刷）
3. 尺寸计算顺序：`view.getVisibleSize()` → 父节点 UITransform（>100 才用）→ 兜底 1280×720（<100 时）

## 4.2 ResponsivePanelContent

| 属性 | 默认值 | 说明 |
|------|--------|------|
| `panelFrame` | (Node) | 绑到 `PanelFrame` |
| `paddingX` | 48 | 水平内边距 |
| `paddingY` | 56 | 垂直内边距 |

`applyLayout()` 执行后自动触发同节点上的其他 `applyLayout()`（如 `*PanelLayout`）。

## 4.3 ResponsiveUIRoot

地牢场景专用。挂载在 `UIRoot` 上，使 UIRoot 自动铺满可视区域。

## 4.4 *PanelLayout 通用规则

所有 `*PanelLayout` 组件：
- 挂载在 `ContentRoot` 上
- 在 `onEnable` 和 `applyLayout` 时执行
- 使用 `_setSize(node, w, h)` 设置每个子节点的 UITransform 尺寸
- 按 ContentRoot.width/height 计算子节点位置

### A.3.1 运行时节点生命周期（2D）

```
DungeonSceneInstaller.install()
  ├── Canvas/World (渲染层: Background → Tile → Actor → Effect → Door)
  ├── Canvas/Systems (全局系统)
  ├── Canvas/UIRoot (UI 容器)
  │   ├── ResponsiveUIRoot (自适应铺满)
  │   ├── BattleHUD / VirtualJoystick / SkillUI / DungeonMapUI
  │   │   └── 预置 → 从 DungeonSceneController 下 reparent 到 UIRoot
  │   ├── UpgradeUI / DeathUI / EquipmentUI / InventoryUI
  │   └── DungeonHudLayout (四角定位)
  └── Canvas/UIRoot: ResponsiveUIRoot.applyLayout() → DungeonHudLayout.applyLayout()
```
