## 约定说明

| 标记 | 含义 |
|------|------|
| `[组件名]` | 节点上挂载的 Cocos 组件 |
| `(Script: XxxPanel)` | 节点上挂载的自定义脚本 |
| `field=节点名` | 脚本字段绑定的子节点 |
| `LocalizedLabel(key)` | 挂载了 `LocalizedLabel` 组件，textKey 为 key |
| `*` | 运行时由代码动态创建的子节点 |
| `!` | 编辑器绑定的必填项 |
| `✅` | 已在场景文件中确认存在 |
| `❌` | 场景文件中缺失，需手动创建 |
| `?` | 需在编辑器中手动确认是否存在 |

# 4. 共用底座组件

## 4.1 ResponsivePanelRoot

| 属性 | 默认值 | 说明 |
|------|--------|------|
| `dimMask` | (Node) | 绑到 `PanelRoot/DimMask` |
| `panelFrame` | (Node) | 绑到 `PanelRoot/PanelFrame` |
| `frameWidthRatio` | 0.72 | PanelFrame 宽度 = canvasW × 此比例 |
| `frameHeightRatio` | 0.78 | PanelFrame 高度 = canvasH × 此比例 |
| `maxFrameWidth` | 900 | PanelFrame 最大宽度 |
| `maxFrameHeight` | 620 | PanelFrame 最大高度 |
| `minFrameWidth` | 520 | PanelFrame 最小宽度 |
| `minFrameHeight` | 360 | PanelFrame 最小高度 |

生命周期：
1. `onLoad` → `applyLayout()` + 监听 `canvas-resize` / `design-resolution-changed`
2. `onEnable` → `applyLayout()` + `scheduleOnce(applyLayout, 0)`（延迟重刷）
3. 尺寸计算顺序：`view.getVisibleSize()` → 父节点 UITransform（>100 才用）→ 兜底 1280×720（<100 时）

## 4.2 ResponsivePanelContent

| 属性 | 默认值 | 说明 |
|------|--------|------|
| `panelFrame` | (Node) | 绑到 `PanelFrame` |
| `paddingX` | 48 | 水平内边距 |
| `paddingY` | 56 | 垂直内边距 |

`applyLayout()` 执行后自动触发同节点上的其他 `applyLayout()`（如 `*PanelLayout`）。

## 4.3 ResponsiveUIRoot

地牢场景专用。挂载在 `UIRoot` 上，使 UIRoot 自动铺满可视区域。

## 4.4 *PanelLayout 通用规则

所有 `*PanelLayout` 组件：
- 挂载在 `ContentRoot` 上
- 在 `onEnable` 和 `applyLayout` 时执行
- 使用 `_setSize(node, w, h)` 设置每个子节点的 UITransform 尺寸
- 按 ContentRoot.width/height 计算子节点位置
