# 编辑器节点绑定通用规则

## 目标

避免 Cocos Creator 编辑器字段拖拽错误导致运行时空引用、组件错绑、界面只显示默认白图、按钮不响应、Label 不刷新等问题。

本规则适用于：

- main/splash/dungeon 场景中的所有 UI 面板
- HUD、按钮、Label、Sprite、EditBox、动态列表
- 后续新增的任何可在 Inspector 中拖拽绑定的脚本字段

## 核心规则

1. Inspector 中暴露的字段默认使用 `@property(Node)`。
2. 不推荐在 UI/Panel/HUD 脚本中暴露 `@property(Label)`、`@property(Button)`、`@property(Sprite)`、`@property(EditBox)`。
3. 运行时通过 `NodeRef.component(nodeRef, Label/Button/Sprite/EditBox)` 获取组件。
4. 关键字段必须有稳定节点路径兜底，例如 `ActionZone/ConfirmBtn`。
5. 面板控制脚本只挂在面板根节点，例如 `CreatePanel.ts` 只挂 `CreatePanel`，不要挂到 `ContentRoot`。
6. `ContentRoot` 只挂布局类和内容自适应类，例如 `ResponsivePanelContent`、`CreatePanelLayout`。
7. 动态创建的节点也要走同一套工具，不要直接假设某个子节点一定存在。

## 正确写法

```ts
import { _decorator, Component, Node, Label, Button } from 'cc';
import { NodeRef } from '../utils/NodeRef';

const { ccclass, property } = _decorator;

@ccclass('ExamplePanel')
export class ExamplePanel extends Component {
    @property(Node) titleLabel: Node | null = null;
    @property(Node) confirmBtn: Node | null = null;
    @property(Node) contentRoot: Node | null = null;

    onLoad(): void {
        const button = NodeRef.component(this.confirmBtn, Button, this.contentRoot, 'ActionZone/ConfirmBtn');
        button?.node.on(Button.EventType.CLICK, this._onConfirm, this);
    }

    refreshTitle(): void {
        const label = NodeRef.component(this.titleLabel, Label, this.contentRoot, 'HeaderZone/TitleLabel');
        if (label) label.string = '...';
    }

    private _onConfirm(): void {
        // ...
    }
}
```

## 错误写法

```ts
@property(Label) titleLabel: Label | null = null;
@property(Button) confirmBtn: Button | null = null;
```

原因：

- 编辑器里可能拖入的是节点而不是组件。
- 场景结构调整后旧组件引用可能残留。
- 同一节点上可能被误加多个组件。
- 脚本字段类型变化后，scene 中旧序列化数据不会自动完全清理。

## 布局脚本规则

布局脚本也使用 `Node` 字段：

```ts
@property(Node) titleLabel: Node | null = null;
@property(Node) confirmBtn: Node | null = null;
```

布局内部只处理节点尺寸和位置：

```ts
const titleNode = NodeRef.node(this.titleLabel) ?? NodeRef.find(this.node, 'HeaderZone/TitleLabel');
titleNode?.setPosition(0, 0);
```

不要在布局脚本里写业务文本、资源加载、流程跳转。

## 面板结构规则

标准面板结构：

```text
PanelNode
└── PanelRoot
    ├── DimMask
    └── PanelFrame
        └── ContentRoot
            ├── HeaderZone
            ├── BodyZone
            └── ActionZone
```

挂载规则：

| 节点 | 允许组件 |
| --- | --- |
| `PanelNode` | 面板控制脚本，例如 `CreatePanel.ts` |
| `PanelRoot` | `ResponsivePanelRoot` |
| `PanelFrame` | `Sprite` 或皮肤绑定 |
| `ContentRoot` | `ResponsivePanelContent`、具体布局脚本 |
| `HeaderZone/BodyZone/ActionZone` | `UITransform`，必要时挂局部布局脚本 |

## 验证规则

运行：

```bash
npm.cmd run validate:all
```

架构门禁会提示：

```text
[WARN] Risky editor Component fields (prefer Node + NodeRef)
```

这些 warning 不会阻塞当前运行，但新增代码必须逐步清掉。

## 迁移步骤

1. 把 `@property(Label/Button/Sprite/EditBox)` 改成 `@property(Node)`。
2. 在 Inspector 中重新拖入对应节点。
3. 使用 `NodeRef.component()` 获取真实组件。
4. 为关键节点补路径兜底。
5. 运行 `npm.cmd run validate:all`。
6. 打开 Cocos Preview 验证界面显示、按钮点击、面板开关。
