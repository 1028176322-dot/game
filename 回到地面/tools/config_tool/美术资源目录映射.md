# 美术资源目录映射与旧资源清理方案 v3

> 版本：v3.0  
> 日期：2026-06-27  
> 适用项目：`E:/game/回到地面`  
> 运行资源根目录：`E:/game/回到地面/assets/resources/textures`  
> 权威审计清单：`E:/game/回到地面/art_source/textures_audit_manifest.csv`

---

## 1. 当前结论

这份文档替代旧版“416 个文件绝对路径映射”。旧版文档的定位是“AI 生成后直接写入 `assets/resources/textures`”，现在已经不适合当前阶段。

当前项目已经进入新阶段：

```text
S 级战斗资源已经高清替换完成
角色目标帧宽：192px
Boss 目标帧宽：256px
miniboss 目标帧宽：192px
精英怪目标尺寸：192px
旧资源需要隔离清理
后续资源生成必须先进入 art_source，再经过审计清单确认
```

新的原则是：

```text
textures_audit_manifest.csv 是资源执行清单
assets/resources/textures 是运行时目录
art_source 是母版、导出、旧资源隔离目录
AI 不直接覆盖 resources
旧脚本不直接覆盖高清资源
```

## 3. 目录定位

### 3.1 美术资源根目录

```text
assets/resources/
├─ arts/
├─ audio/
├─ fonts/
├─ prefabs/
├─ spine/
└─ textures/
```

目录职责：

| 目录 | 是否运行时 | 当前策略 |
|---|---|---|
| `textures` | 是 | 核心图片资源，按 CSV 分级、分包、替换、清理 |
| `arts` | 不应长期作为正式运行目录 | 当前主要是 placeholder，应逐步移出或替换 |
| `audio` | 是 | 后续 BGM/SFX 需单独审计体积和格式 |
| `fonts` | 是 | 后续字体需审计授权、体积、语言覆盖 |
| `prefabs` | 是 | 美术 prefab 需审计依赖和引用 |
| `spine` | 是 | Spine 需按 skeleton/atlas/png 成组审计 |

原则：

```text
正式运行图片优先进入 textures 或后续 bundles
placeholder 不应长期留在 resources
源文件不进入 assets/resources
prefab/spine/audio/font 也必须进入全量审计
```

### 3.2 核心纹理运行时目录

```text
assets/resources/textures/
├─ backgrounds/
├─ bosses/
├─ characters/
├─ effects/
├─ icons/
├─ monsters/
├─ tiles/
└─ ui/
```

用途：

```text
Cocos Creator 导入
运行时加载
场景、Prefab、脚本引用
构建进入包体或 Asset Bundle
```

禁止放入：

```text
AI 母版
PSD / Aseprite / 源工程
raw 原图
bak 备份
before/after 对比图
测试图
未审核导出图
旧版低清资源副本
```

### 3.3 Placeholder 目录

当前目录：

```text
assets/resources/arts/placeholders/
```

当前状态：

```text
约 74 张 placeholder PNG
体积很小，但仍属于 resources 下的可打包资源
```

处理策略：

```text
1. 如果 placeholder 已被正式 textures 资源替代：action=move_source 或 delete
2. 如果仍被代码/配置作为 fallback 使用：action=keep，但必须标明 used_by
3. 如果只是历史占位图：移到 art_source/legacy_textures_YYYYMMDD/placeholders
4. 不允许 placeholder 和正式资源长期并行存在且用途不清
```

建议新增清单：

```text
art_source/placeholders_audit_manifest.csv
```

字段：

```csv
path,width,height,size_kb,mode,used_by,reference_count,replaced_by,action,status,note
```

### 3.4 Prefab / Spine / Font / Audio

Prefab：

```text
assets/resources/prefabs/
assets/prefabs/
```

需要审计：

```text
依赖哪些 SpriteFrame
是否引用旧低清资源
是否仍引用 placeholder
是否应该进入首包或分包
```

Spine：

```text
assets/resources/spine/
```

Spine 必须成组管理：

```text
xxx.json
xxx.atlas
xxx.png
xxx.meta
```

不能只移动其中一个文件。

字体：

```text
assets/resources/fonts/
```

需要记录：

```text
字体名
用途
体积
授权
是否首包必需
是否可用系统字体替代
```

音频：

```text
assets/resources/audio/
```

虽然音频不是美术图片，但属于表现资源，也要纳入包体审计：

```text
BGM：按场景/区域分包
SFX：通用音效进 common，区域音效进 zone bundle
source 目录不放未压缩母版
```

```text
art_source/

```

### 4.1 角色资源

目录：

```text
assets/resources/textures/characters/{character}/
```

角色列表：

```text
warrior
archer
assassin
mage
berserker
```

动作：

```text
idle
walk
attack
skill
dodge
hit
death
```

规格：

| 项 | 当前目标 |
|---|---:|
| 目标帧宽 | 192px |
| 布局 | vertical |
| 运行目录 | `assets/resources/textures/characters` |
| Bundle | `bundle_core_battle` |
| 清理策略 | 旧 48/64px 副本不得留在同目录 |

示例：

```text
assets/resources/textures/characters/warrior/warrior_idle.png
assets/resources/textures/characters/warrior/warrior_walk.png
assets/resources/textures/characters/warrior/warrior_attack.png
```

注意：

```text
当前 CSV 已包含 source_frame_w/source_frame_h。
当前 CSV 中 frame_w/frame_h 视为目标帧尺寸。
因此序列帧应满足：
target_w = frame_w
target_h = frame_h * frame_count
```

### 4.2 Boss 资源

目录：

```text
assets/resources/textures/bosses/finalboss/
assets/resources/textures/bosses/finalboss/{zone}/
```

规格：

| 类型 | 目标帧宽 | Bundle |
|---|---:|---|
| final boss 单帧版 | 256px | `bundle_boss_{zone}` |
| final boss 序列帧版 | 256px | `bundle_boss_{zone}` |
| miniboss | 192px | `bundle_boss_{zone}` |

区域 bundle：

```text
bundle_boss_forest
bundle_boss_catacombs
bundle_boss_volcano
bundle_boss_tundra
bundle_boss_swamp
bundle_boss_abyss
```

序列帧规则：

```text
layout=vertical
target_w = frame_w
target_h = frame_h * frame_count
```

示例：

| 字段 | 示例 |
|---|---:|
| `width` / `height` | 64 / 256 |
| `source_frame_w` / `source_frame_h` | 64 / 64 |
| `frame_w` / `frame_h` | 256 / 256 |
| `frame_count` | 4 |
| `target_w` / `target_h` | 256 / 1024 |

低频动作：

```text
death
phasechange
```

可以保留为 `defer`，不强制第一批高清替换。

### 4.3 怪物资源

目录：

```text
assets/resources/textures/monsters/{zone}/
```

区域：


规格：

| 类型 | 目标 |
|---|---|
| 精英怪 | 192px |
| 普通怪 | 按 CSV，当前多数 `defer` |

```text
精英怪优先保持高清

```

### 4.4 UI 资源

```text
assets/resources/textures/ui/
```

子目录：

```text
splash
main
map

hud

equipment
shop
```

策略：

| 子类 | 当前策略 |
|---|---|
| splash/main | 首屏关键资源，需控体积 |
| hud | 高频显示，优先稳定 |
| upgrade 1024 图标 | `resize_export` 到 256px |
| common/equipment 未确认资源 | `defer`，等引用扫描 |
| shop_bg | `compress_review` |
| 小按钮/框体 | `merge_atlas` 或九宫格 |


注意：

```text
UI 资源不能因为旧脚本存在就重新生成 48/64px。
升级图标的 1024 图只应作为母版，不应作为普通运行图。
```

## 6. 旧资源清理规则

### 6.1 绝对禁止留在 resources 的文件

```text
*.bak
*.bak.*
*_raw.png
raw_*.png
*_backup.png
backup_*.png
*_legacy.png
legacy_*.png
*_test.png
test_*.png
*_tmp.png
tmp_*.png
before_*.png
after_*.png
*.psd
*.aseprite
*.kra
*.clip
*.ai
*.sketch
*.blend
*.wav.source
*.mp3.source
```

注意：不要简单用 `*old*` 扫描，因为正常文件名可能包含 `soldier`，例如：

```text
monster_tundra_penguinsoldier_idle.png
```

这种会被 `*old*` 误伤。

适用范围：

```text
assets/resources/textures
assets/resources/arts
assets/resources/audio
assets/resources/fonts
assets/resources/prefabs
assets/resources/spine
assets/prefabs
```

### 6.2 清理动作定义

| 动作 | 说明 |
|---|---|
| `move_source` | 移到 `art_source/legacy_textures_YYYYMMDD` |
| `delete` | 确认无引用、无保留价值后删除 |
| `defer` | 引用不明确或低频，暂缓 |
| `keep` | 保留运行时资源 |
| `verify_usage` | 需要引用扫描后再决定 |

### 6.3 各类资源清理策略

| 类型 | 位置 | 清理策略 |
|---|---|---|
| 正式纹理 | `assets/resources/textures` | 按 `textures_audit_manifest.csv` 执行 |
| placeholder | `assets/resources/arts/placeholders` | 有替代则移出，无引用则删除，有 fallback 用途则登记 |
| prefab | `assets/resources/prefabs`, `assets/prefabs` | 不直接删除，先查依赖和场景引用 |
| spine | `assets/resources/spine` | 成组移动 `.json/.atlas/.png/.meta` |
| font | `assets/resources/fonts` | 保留运行字体，源字体/未授权字体移出 |
| audio | `assets/resources/audio` | 保留压缩运行音频，未压缩源文件移出 |
| config 引用图 | `assets/resources/config` 中路径指向 | 必须先改配置再清理资源 |

### 6.4 旧资源隔离目录

建议创建：

```text
art_source/legacy_art_assets_20260627/
├─ moved_from_resources/
│  ├─ png/
│  └─ meta/
├─ old_lowres/
├─ placeholders/
├─ prefabs/
├─ spine/
├─ fonts/
├─ audio/
├─ raw_source/
└─ manifest_legacy_cleanup.csv
```

隔离原则：

```text
移动 PNG 时，如果对应 .meta 已不参与运行，也一起移动
如果 .meta 还被 Cocos 引用，不直接删除
隔离后必须打开 Cocos 重新导入并检查场景
隔离目录不进入 assets，不参与构建
```

## 7. 清理流程

### 第 1 步：生成旧资源候选清单

输出：

```text
art_source/legacy_textures_YYYYMMDD/manifest_legacy_cleanup.csv
```

字段：

```csv
path,size_kb,reason,action,has_meta,reference_count,status,note
```

候选来源：

```text
命名规则命中 raw/bak/backup/legacy/tmp/test
CSV 中 action=move_source
CSV 中 used_by=unknown 且长期未引用
高清同名替换后残留的旧副本
不在 textures_audit_manifest.csv 的 PNG
```

### 第 2 步：引用扫描

扫描范围：

```text
assets/**/*.scene
assets/**/*.prefab
assets/**/*.anim
assets/**/*.ts
assets/**/*.meta
```

检查内容：

```text
路径引用
文件名引用
UUID 引用
SpriteFrame 子资源引用
resources.load 动态路径
```

结果：

```text
reference_count = 0：可隔离
reference_count > 0：不能移动，先改引用
动态路径无法确认：verify_usage
```

### 第 3 步：移动到隔离目录

不要直接删除。先移动到：

```text
art_source/legacy_textures_YYYYMMDD/moved_from_resources/
```

需要保留相对路径，例如：

```text
assets/resources/textures/ui/main/main_bg_raw.png
→ art_source/legacy_art_assets_20260627/moved_from_resources/textures/ui/main/main_bg_raw.png
```

### 第 4 步：Cocos 重新导入

操作：

```text
打开 Cocos Creator
等待资源导入完成
检查 Console
检查 Asset 面板是否有丢失 SpriteFrame
检查 splash/main/dungeon 三个场景
```

### 第 5 步：运行验证

必须验证：

```text
splash 不黑屏
```

## 9. manifest 字段规范

当前 `textures_audit_manifest.csv` 字段：

```csv
path,category,width,height,size_kb,mode,has_alpha,grade,action,used_by,reference_count,scene_or_ui,visible_w,visible_h,target_w,target_h,target_size_kb,bundle,atlas_group,source_frame_w,source_frame_h,frame_w,frame_h,frame_count,layout,display_mode,status,note
```

当前字段口径：

```text
source_frame_w/source_frame_h：源文件单帧尺寸
frame_w/frame_h：目标运行单帧尺寸
width/height：当前文件整图尺寸
target_w/target_h：目标运行整图尺寸
frame_count：帧数
layout：single / vertical / horizontal / grid
```

因此，当前 CSV 已经基本对齐第 9 章的帧尺寸要求，不强制再新增 `target_frame_w/target_frame_h`。如果后续为了避免语义误解，可以把 `frame_w/frame_h` 重命名为 `target_frame_w/target_frame_h`。

建议下一版只额外增加清理追踪字段：

```csv
legacy_path,cleanup_status
```

用途：

```text
legacy_path：旧资源被移到哪里
cleanup_status：pending / moved / verified / deleted
```

### 9.1 全美术资源 manifest

为了适配所有美术资源，需要新增：

```text
art_source/art_assets_manifest.csv
```

它不替代 `textures_audit_manifest.csv`，而是覆盖所有美术/表现资源根目录：

```text
assets/resources/textures
assets/resources/arts
assets/resources/prefabs
assets/resources/spine
assets/resources/fonts
assets/resources/audio
assets/prefabs
```

推荐字段：

```csv
path,root,asset_type,category,size_kb,extension,grade,action,used_by,reference_count,bundle,status,note
```

字段说明：

| 字段 | 说明 |
|---|---|
| `path` | 相对项目路径 |
| `root` | `textures` / `arts` / `prefabs` / `spine` / `fonts` / `audio` |
| `asset_type` | `texture` / `placeholder` / `prefab` / `spine` / `font` / `audio` / `source` |
| `category` | UI、角色、怪物、Boss、音效、字体等 |
| `grade` | S/A/B/C |
| `action` | replace/keep/defer/move_source/delete/merge_atlas/compress_review |
| `used_by` | 使用场景或系统 |
| `reference_count` | 引用数 |
| `bundle` | 分包归属 |
| `status` | pending/approved/moved/deleted/verified |

`textures_audit_manifest.csv` 继续记录图片尺寸、帧尺寸、PNG mode 等纹理细节；`art_assets_manifest.csv` 记录全资源级别的存在、用途、清理和分包。

## 10. 执行顺序

### 阶段 A：锁定当前高清资源

```text
1. 确认 S 级角色/Boss/miniboss/精英怪已经为高清版
2. 禁止旧 generate 脚本直接写入 resources
3. 备份当前 textures 状态
4. 固化 textures_audit_manifest.csv
```

### 阶段 B：清理明显旧资源

```text
1. 清理 raw/bak/backup/tmp/test 命名资源
2. 处理 CSV 中 action=move_source 的资源
3. 移动到 legacy_textures_YYYYMMDD
4. 不直接删除
```

### 阶段 C：引用扫描

```text
1. 扫 .scene/.prefab/.anim/.ts/.meta
2. 回填 reference_count
3. used_by=unknown 的资源保持 defer
4. 只移动 reference_count=0 的资源
```

### 阶段 D：构建与真机验证

```text
1. Cocos 重新导入
2. 浏览器预览
3. 微信开发者工具构建
4. 真机检查
5. 包体报告
```

### 阶段 E：归档

```text
1. 隔离目录保留一个版本周期
2. 无问题后归档到外部备份或删除
3. 更新 manifest status
```

## 11. 验收标准

清理完成后必须满足：

```text
assets/resources 不包含 raw/bak/backup/tmp/test/legacy 源文件
S 级战斗资源保持高清规格
CSV 中所有 action=move_source 已处理
used_by=unknown 不直接删除
旧生成脚本不会覆盖高清资源
浏览器预览无丢图
微信开发者工具无资源缺失
真机无黑屏、无白块、无 SpriteFrame 丢失
构建包体比清理前下降或保持可控
```

全美术资源额外验收：

```text
assets/resources/arts/placeholders 已全部登记到 art_assets_manifest.csv
正式资源已替代的 placeholder 已移动或删除
prefab 引用没有丢失 SpriteFrame
spine 资源成组存在或成组移出
fonts 只有运行必需字体，不包含源文件或未授权字体
audio 只有运行压缩音频，不包含未压缩源文件
所有美术相关资源都有 bundle 或 none 归属
```
