# 组合性 × 随机性系统设计

> **版本**: v0.5 (系统详设)
> **日期**: 2026-06-24
> **设计者**: GameDesigner
> **核心目标**: 通过**元素反应、词缀组合、怪物装配、房间变异**四层随机系统，让每局游戏产生本质差异
> **编码**: UTF-8
> **版权声明**: 本文件中的系统名称、怪物名称、能力名称均为通用游戏术语，不使用任何 IP 专属名词。详见`项目约定与版权声明.md`

---

## 0. 设计哲学

**核心理念**：组合性的价值不在于"有多少种东西"，而在于**"东西之间有多少种互动方式"**。

- 10 种元素 × 每对 1 种反应 = **45 种组合**
- 12 个前缀 × 12 个后缀 = **144 种词缀组合**
- 6 种怪物身体 × 6 个元素 × 8 个技能 × 3 个头目倍率 = **864 种怪物变体**
- 8 种房间变异 × 5 层 = **40 种楼层状态**

**这不是加法，这是乘法。** 玩家每次开局的"谜题"都不一样——他们不是在重复同一种战斗，而是在**发现新的化学方程式**。

## 1. 元素反应系统 (Element Reaction System) —— 核心组合机制

### 1.1 六元素体系

每个技能、附魔、怪物攻击、道具都有**元素属性**：

| 元素 | 基础效果 | 代表来源 |
|------|----------|----------|
| 烈焰 Fire | 附加灼烧 (2回合, 每回合2点伤害) | 火焰附魔、火球术、火龙卷 |
| 冰霜 Frost | 缓速 (SPD -1, 2回合) | 冰霜附魔、冰锥术 |
| 闪电 Lightning | 概率眩晕 (25%, 1回合) | 闪电附魔、雷击 |
| 毒素 Poison | 持续毒伤 (3回合, 每回合1点 + 可叠加) | 毒附魔、毒雾 |
| 暗影 Shadow | 吸血 (造成伤害的20%回血) | 暗影附魔、暗影打击 |
| 神圣 Holy | 净化 + 对亡灵额外伤害 + 1.5倍 | 神圣附魔、审判 |

### 1.2 元素反应表 (触发时消耗两种元素状态)

当目标同时受到两种元素影响时，触发**元素反应**：

| 组合 | 反应名 | 效果 | 发现价值 |
|------|--------|------|----------|
| Fire + Poison | 爆燃 Explosion | 立即额外 5 点 AoE 伤害，灼烧扩散至相邻敌人 | **清场 combo** |
| Fire + Frost | 蒸汽 Steam | 目标失去锁定(下一回合内无法被 AI 瞄准) | **生存 escape** |
| Frost + Lightning | 超导 Superconduct | 目标 DEF 归零 2 回合 | **破甲增伤** |
| Lightning + Poison | 神经毒素 Neurotoxin | 目标麻痹 1 回合 + 毒伤翻倍 | **控杀 combo** |
| Shadow + Holy | 虚空 Void | 两种效果互相抵消，纯伤害 8 点 | **简单粗暴** |
| Fire + Lightning | 等离子 Plasma | 无视 50% DEF | **坦克杀手** |
| Frost + Shadow | 灵魂霜冻 Soul Chill | 目标 SPD 归零 2 回合 + 每回合扣 2 HP | **控制伤害** |
| Holy + Fire | 圣焰 Radiance | 2 格范围内所有友方回复 3 HP + 亡灵敌人受伤×2 | **攻守兼备** |
| Poison + Shadow | 腐朽 Decay | 毒伤扩散至相邻敌人 + 持续时间重置 | **连锁感染** |
| Holy + Lightning | 裁决 Judgment | 目标当前 HP 越低伤害越高(每缺 10% HP +1) | **斩杀** |
| Fire + Holy + Lightning | 三重奏 Tri-Element | 触发所有两两反应 + 额外 10 点伤害 | **终极爆发** |

**反应触发设计意图**：玩家需要思考"我先后使用哪两个技能/附魔"——这比"哪个伤害数字大"需要更多的战略判断。

### 1.3 反应链 (Chain Reaction)

当一次触发反应产生的效果**本身带有元素属性**时，可能引发二次反应：

```
例：敌人中了 毒(Poison) + 灼烧(Fire) → 爆燃(Explosion)
    → 爆燃的 AoE 是 火焰(Fire) 属性
    → 如果相邻敌人身上已有 冰霜(Frost) → 触发 蒸汽(Steam)
    → 系统继续检查是否有新反应...
```

**链式反应没有预设上限**，但实际游戏中受限于棋盘大小和怪物数量，通常 2~3 层。这就是玩家在社交媒体上能传播的"爽图"素材。

## 2. 词缀系统 (Affix System) —— 每件物品都独一无二

### 2.1 词缀结构

每个可拾取的**武器/护甲/道具**随机生成 **1~2 个词缀**——一个前缀+一个后缀：

| 前缀 (Prefix) | 类型 | 效果 |
|---------------|------|------|
| 锋利的 Sharp | 武器 | ATK +2 |
| 沉重的 Heavy | 武器 | ATK +3, Move -1 |
| 迅捷的 Swift | 武器/护甲 | SPD +1 |
| 坚固的 Sturdy | 护甲 | DEF +2 |
| 吸血 Vampiric | 武器 | 攻击回复 20% 伤害 |
| 带电的 Shocking | 武器 | 攻击附加闪电伤害 |
| 燃烧的 Blazing | 武器 | 攻击附加火焰伤害 |
| 冰封的 Icy | 武器 | 攻击附加冰霜伤害 |
| 剧毒的 Toxic | 武器 | 攻击附加毒素伤害 |
| 暗影的 Shadowy | 武器 | 攻击附加暗影伤害 |
| 神圣的 Hallowed | 武器 | 攻击附加神圣伤害 |
| 幸运的 Lucky | 通用 | 暴击率 +5% |

| 后缀 (Suffix) | 类型 | 效果 |
|--------------|------|------|
| 之火焰 of Flames | 武器 | 普攻变为火属性 |
| 之寒冬 of Winter | 武器 | 普攻变为冰属性 |
| 之雷霆 of Thunder | 武器 | 普攻变为雷属性 |
| 之亡者 of the Dead | 武器 | 击杀后概率召唤骷髅 |
| 之狂战士 of the Berserker | 武器 | HP < 50% 时 ATK 翻倍 |
| 之幻影 of the Phantom | 护甲 | 进入暗区后永久隐身 |
| 之守护 of Protection | 护甲 | 每回合自动回复 1 HP |
| 之荆棘 of Thorns | 护甲 | 受击反弹 2 点伤害 |
| 之力量 of Power | 通用 | ATK +1, DEF +1 |
| 之贪婪 of Greed | 通用 | 金币掉落 +20% |
| 之不朽 of Immortality | 通用 | 每层第一次死亡以 1 HP 复活 |
| 之回响 of Echo | 通用 | 使用技能时回复 1 HP |

### 2.2 组合示例

| 完整名称 | 效果 | 适用流派 |
|----------|------|----------|
| 锋利的火焰之剑 Sharp Sword of Flames | ATK+2 + 普攻火属性 | **火系爆燃流** |
| 吸血的幻影之刃 Vampiric Blade of the Phantom | 20%吸血 + 暗区隐身 | **暗影游走流** |
| 坚固的不朽铠甲 Sturdy Armor of Immortality | DEF+2 + 每层1次免死 | **不动明王流** |
| 带电的雷霆之锤 Shocking Hammer of Thunder | 闪电附加 + 雷属性普攻 | **超导破甲流** |
| 剧毒的荆棘之盾 Toxic Shield of Thorns | 毒伤 + 反伤 2 点 | **反伤毒瘤流** |

**设计意图**：词缀让**每件物品都有名字**，而不是一堆"铁剑+3"的抽象数值。玩家会因为捡到一把"吸血幻影之刃"而兴奋——因为这个名字暗示了一个玩法方向。

### 2.3 词缀稀有度

| 稀有度 | 词缀数量 | 出现概率 | 说明 |
|--------|----------|----------|------|
| 普通 (白色) | 0 词缀 | 60% | 基础 |
| 魔法 (蓝色) | 1 词缀 | 30% | 前缀或后缀 |
| 稀有 (金色) | 2 词缀 | 9% | 前缀+后缀 |
| 传奇 (橙色) | 2 词缀 + 特殊效果 | 1% | 额外特效（如"此武器可同时装备两个附魔"）|

## 3. 怪物装配系统 (Monster Assembly) —— 每场战斗都不一样

### 3.1 怪物生成流程

```
生成一个怪物 = 随机选择:
  1. 身体模板 (决定基础属性) 
  2. 元素属性 (决定攻击附加效果)
  3. 核心技能 (AI 行为模式)
  4. 规模等级 (决定属性倍率)
  5. 变异因子 (额外特性)
```

### 3.2 身体模板 (6 种)

| 模板 | HP | ATK | DEF | SPD | 特点 |
|------|----|-----|-----|-----|------|
| 骷髅 Skeleton | 低 | 中 | 低 | 中 | 被神圣克制，物理抗性 -1 |
| 魔像 Golem | 极高 | 高 | 极高 | 极低 | 免疫眩晕，移速慢 |
| 恶魔 Demon | 高 | 高 | 中 | 中 | 火抗+2，神圣额外伤害 |
| 野兽 Beast | 中 | 高 | 低 | 高 | 冲锋技能，无元素抗性 |
| 史莱姆 Slime | 中 | 低 | 中 | 低 | 受物理攻击时分裂(低概率) |
| 幽灵 Ghost | 低 | 中 | 低 | 高 | 物理伤害 -50%，可穿墙 |

### 3.3 核心技能池 (12 种)

| 技能 | 效果 | 出现概率 |
|------|------|----------|
| 冲锋 Charge | 向玩家移动 3 格并攻击，伤害 ×1.5 | 20% |
| 骨盾 Bone Shield | 本回合 DEF +4 | 15% |
| 自爆 Self-Destruct | 3 回合后自爆，范围 2 格 AoE | 10% |
| 召唤 Summon | 召唤 1 个小怪（上限 3 只同时在场，总怪 > 8 禁召）| 15% |
| 治愈 Heal | 回复 20% HP | 10% |
| 毒雾 Poison Cloud | 在自身 1 格范围内释放毒雾(持续 2 回合) | 10% |
| 瞬移 | 瞬间移动到玩家相邻格 | 10% |
| 咆哮 Roar | 降低玩家 ATK 1 点(本场战斗) | 10% |
| 吞噬 Devour | 击杀后回复 50% HP | 5% (仅精英/Boss) |
| 分裂 Split | HP < 30% 时分裂成 2 个小怪 | 5% (仅史莱姆) |
| 时间锁 Time Lock | 玩家下回合 AP -1 | 5% (仅精英) |
| 复仇 Revenge | 每损失 10% HP，ATK +1 | 5% (仅 Boss) |

### 3.4 变异因子 (10 种，每个怪物带 0~1 个)

| 变异 | 效果 | 玩家应对 |
|------|------|----------|
| 爆炸性 Explosive | 死亡时造成 1 格 AoE 伤害 | 别站旁边 |
| 再生 Regenerating | 每回合回复 2 HP | 集中火力别拖 |
| 相位偏移 Phase-Shifting | 每 2 回合隐身 1 回合 | 记住位置，预测行动 |
| 吸血 Vampiric | 攻击回复伤害的 30% | 优先击杀 |
| 荆棘 Thorns | 受击反伤 2 点 | 用技能而不是普攻 |
| 巨大化 Giant | 体型 ×2, HP ×1.5 但每次 2 格 | 卡位受限 |
| 迷你化 Tiny | 体型缩小, 闪避 +25% | 用 AoE 打 |
| 元素亲和 Elemental Affinity | 同属性伤害免疫, 对属性伤害 ×1.5 | 换元素打 |
| 狂暴 Enraged | HP < 30% 时 ATK ×2, DEF 归零 | 低血量时最后一击秒杀 |
| 集群 Swarm | 被击杀时在相邻格生成 2 个小怪 | 拉到空地打 |

### 3.5 战斗变体示例

```
战斗 A:  骷髅 + 火焰 + 冲锋 + 精英 + 再生
         → 一只高速冲锋、边打边回血的火骷髅
         → 玩家策略：用冰霜缓速 + 神圣克制，阻止它冲锋路径

战斗 B:  魔像 + 冰霜 + 自爆 + 普通 + 荆棘
         → 一个移动缓慢但带反伤的冰魔像，3 回合后自爆
         → 玩家策略：拉开距离，3 回合内远程解决，或在自爆前位移

战斗 C:  幽灵 + 暗影 + 闪烁 + 精英 + 相位偏移
         → 一个闪烁穿墙、隐身、带吸血的相位幽灵
         → 玩家策略：用神圣攻击克制，预判闪烁位置，AoE 覆盖
```

**每场战斗的"解法"都不同**——玩家需要快速判断"这个怪怎么打"，而不是无脑平 A。

## 4. 房间变异系统 (Room Mutation) —— 每层都不同

### 4.1 每层战场变异

进入新楼层时，随机选择 **1~2 个全局变异**，持续到该层结束：

| 变异 | 效果 | 玩家适应策略 |
|------|------|--------------|
| 黑暗降临 Darkness | 所有房间视野缩减 50%，只能看到身边 2 格 | 谨慎探索，利用暗区隐身 |
| 血月 Blood Moon | 治疗效果 -50%，吸血效果 ×2 | 吸血 build 的黄金层 |
| 奥术风暴 Arcane Storm | 每 3 回合全屏随机元素伤害(3 点) | 提前找回复房或带抗性 |
| 时空扭曲 Time Distortion | 怪物 SPD 减半，玩家 AP +1 | 输出流的天堂 |
| 地震 Earthquake | 每回合地形随机变化(障碍物移动) | 无法卡位，灵活应战 |
| 淘金热 Gold Rush | 所有怪物金币掉落 ×2，怪物 HP ×1.3 | 贪金 vs. 生存的权衡 |
| 虚弱诅咒 Weakness | 玩家 ATK -2，但每击杀一只怪物永久 +1 ATK | 滚雪球层 |
| 狂乱 Frenzy | 所有怪物 SPD +1, ATK +2, DEF -2 | 速战速决，不能拖 |
| 回声 Echos | 每击杀一个怪物，下个技能 CD -1 回合 | 技能流爆发层 |
| 薄雾 Mist | 玩家移动时在身后留下 1 格毒雾 | 风筝流 yyds |
| 倒计时 Countdown | 该层限时 15 回合，超时后每秒扣 1 HP | 速通挑战 |
| 不稳定的空间 Unstable | 每进入一个新房间，随机一个元素布满全场 | 元素反应乱局 |

### 4.2 变异与流派的交互

| 变异 | 利好流派 | 利空流派 |
|------|----------|----------|
| 血月 | 暗影吸血流、圣骑士 | 盾反流（必须挨打）|
| 奥术风暴 | 任意元素反应流 | 纯物理流 |
| 时空扭曲 | 时间法师、疯狗流 | 不动明王（需要 AP 架盾）|
| 淘金热 | 地主流（打金）| 刺客（血量换输出，不敢贪）|
| 回声 | 技能爆发流 | 普攻流 |
| 狂乱 | 不动明王（反伤）| 穿影（脆皮）|

**设计意图**：变异不是单纯的"增加难度"，而是**改变游戏规则**。玩家需要根据变异调整策略——这本身就是一个 meta-level 的决策。

## 5. 种子系统 (Seed System) —— 可复现的随机

### 5.1 种子构成

每次游戏开始时生成一个**6 位种子码**（如 `AB3K7Z`），决定一切随机事件：

| 种子决定项 | 说明 | 是否对玩家可见 |
|-----------|------|--------------|
| 地牢房间布局 | 每层 DAG 结构、房间类型分布 | 否（但可在官网查看）|
| 怪物装配 | 每个房间的怪物组合 | 否 |
| 强化选项池 | 3 选 1 的具体选项 | 开局不可见，到了才知 |
| 词缀生成 | 掉落物词缀 | 不可预见 |
| 事件房选项 | 2 选 1 的选项池 | 不可预见 |
| 商店库存 | 在售商品 | 不可预见 |

### 5.2 种子分享

玩家完成一局后，可在结算界面看到本局种子码，并选择"分享种子"：

```
种子: AB3K7Z — 第 3 层出现 淘金热 + 奥术风暴，推荐火毒爆燃流
```

- 其他玩家输入同种子 → **完全相同的房间布局和怪物配置**
- 但他们的**词缀和商店库存**会有差异（不同玩家行为影响掉落）
- **用途**：社交媒体分享"神局"、攻略党研究最优路线、竞速

## 6. 事件房重设计：随机事件生成器

将原来固定的 5 种事件改为**参数化随机事件生成**：

### 6.1 事件模板

```
事件 = 场景(Scene) + 条件(Condition) + 选项 A + 选项 B + 后果(Consequence)
```

### 6.2 场景池 (8 种)

| 场景 | 描述 |
|------|------|
| 破碎祭坛 | 地上一座破损的献祭台 |
| 发光水晶 | 墙壁上镶嵌的能量水晶 |
| 古代雕像 | 一尊持剑的战士雕像 |
| 扭曲传送门 | 旋转的次元裂隙 |
| 魔法井 | 深不见底的蓝色井口 |
| 神秘商人 | 兜帽遮面的流浪商人 |
| 陷阱走廊 | 两侧墙壁有箭孔 |
| 休眠火山 | 地面裂开，岩浆流动 |

### 6.3 条件池 (6 种)

| 条件 | 说明 |
|------|------|
| 当前 HP 比例 | 高于/低于 50% 时选项变化 |
| 持有金币数量 | 大于/小于 15 时选项变化 |
| 是否持有钥匙 | 有钥匙 vs 无钥匙 |
| 已击杀怪物数 | 该层击杀数影响 |
| 当前层数 | 越深选项越激进 |
| 持有元素类型 | 身上有/无特定元素附魔 |

### 6.4 后果池 (12 种)

| 后果 | 类型 | 参数范围 |
|------|------|----------|
| 回复 HP | 正 | 3~15 |
| 失去 HP | 负 | 2~10 |
| 获得金币 | 正 | 3~20 |
| 失去金币 | 负 | 3~15 |
| 获得钥匙 | 正 | 1 |
| 失去钥匙 | 负 | 1 |
| 获得随机遗物 | 正 | 1 (稀有) |
| 失去随机附魔 | 负 | 1 |
| 下个房间强化 | 正 | 怪物 HP -50% |
| 下个房间恶化 | 负 | 怪物 ATK +2 |
| 获得临时 Buff | 正 | 3 回合 |
| 施加永久 Debuff | 负 | 本局有效 |

### 6.5 事件示例 (动态生成)

```
场景: 发光水晶
条件: HP > 50% + 持有金币 > 15
选项 A: 触碰水晶 → 消耗 5 金币，ATK +2 (3回合)
选项 B: 砸碎水晶 → 获得 8 金币，但该层怪物 ATK +1

---
场景: 扭曲传送门
条件: 当前层数 > 2 + 持有钥匙
选项 A: 使用钥匙进入 → 跳过本层直接到下一层 (保留全部状态)
选项 B: 忽略 → 无事发生

---
场景: 神秘商人
条件: 持有火焰附魔
选项 A: 展示火焰 → 商人惊喜，免费送你一瓶属性药水
选项 B: 不展示 → 用 10 金币买一瓶属性药水
```

**设计意图**：事件不再是固定脚本，而是**条件触发的动态叙事**。玩家的当前状态（装备、资源、元素）直接影响事件走向——这鼓励玩家"带着特定资源去冒险"。

## 7. 组合性验证：从系统到体验

### 7.1 假设一局游戏的典型流程

```
种子: X9M4K2

第 1 层 — 变异: 血月
  → 治疗减半、吸血翻倍
  → 玩家的策略：倾向于选择吸血类装备和技能
  
  战斗房: 遇到一只 "暗影 + 幽灵 + 闪烁"
    → 暗影吸血 + 幽灵穿墙 + 闪烁 = 极其难缠的游击怪
    → 玩家应对：神圣武器克制幽灵，或使用 AoE 覆盖
    
  宝箱: 开出 "吸血幻影之刃" (蓝色)
    → 完美血月层武器
    
  强化房 3 选 1: 穿影 / ATK+2(HP-1) / 荆棘甲
    → 结合血月 + 吸血武器，选穿影 → 准备走暗影游走路线

第 2 层 — 变异: 淘金热
  → 金币 ×2，怪物变强
  → 策略调整：尽可能多打钱，商店期大采购
  
  事件房: 神秘商人(有暗影附魔 → 免费药水)
  
  商店: 买 "回响之珠" (遗物, 技能回血)
    → 穿影 + 回响之珠 = 无消耗吸血穿梭

第 3 层 — 变异: 奥术风暴
  → 全场随机元素伤害
  → 策略调整：利用元素反应打 combo
  
  战斗房: 火 + 毒双重附魔 → 触发 爆燃 链式反应
    → 清场大屏幕 → "爽"
  
  Boss: 暗影领主(暗影+召唤+复仇)
    → 吸血对吸血, 看谁吸得过谁
    → 利用奥术风暴的随机元素触发虚空反应
```

**这局游戏和上一局的体验完全不同。** 不是"数值不同"，而是**策略目标完全不同**——血月层你追求吸血，淘金热你贪金，奥术风暴你追求元素反应。

### 7.2 组合性数据

| 维度 | 可能组合数 | 备注 |
|------|-----------|------|
| 元素反应 | 10 对 + 1 三重 + 链式 | 实际游戏体验为无限链 |
| 词缀组合 | 12 前缀 × 12 后缀 = 144 | 加上无词缀基础 ≈ 150 |
| 怪物装配 | 6 × 6 × 12 × 4 × 10 = 17,280 | 身体×元素×技能×规模×变异 |
| 房间变异 | 12 种选 1~2 | 适用 5 层 |
| 动态事件 | 8 × 6 × 6 × 12 ≈ 3,456 | 场景×条件×状态×后果 |
| 种子空间 | 36^6 ≈ 21.7 亿 | 6 位字母数字 |
| **理论总组合空间** | **> 10^14** | 乘数级扩散 |

**注意**：不是所有组合都"有意义"——设计目标是让其中的约 10% 成为玩家可感知、可谈论的体验。这已经比传统肉鸽游戏高出一个数量级。

## 8. 难度平衡：智能随机

### 8.1 动态难度调整 (DDA)

| 触发条件 | 调整 | 玩家感知 |
|----------|------|----------|
| 连续 2 层选择"高难度变异" | 下一层强制出"利好"变异 | "这层运气不错"|
| 玩家当前 HP < 30% 且无回复房 | 下个房间必定是回复房 | 不会被憋死 |
| 玩家连续 3 次战斗中未触发元素反应 | 掉落一枚"元素融合石"(必带双元素) | 引导发现 combo |
| Boss 战斗前玩家 ATK 低于层数×2 | Boss 房前置一个强化房保底 | 防止"完全打不过"|
| 玩家同一怪物前死亡 2 次 | 该怪物不再出现在当前种子 | 不反复受挫 |

### 8.2 软锁保护 (Softlock Protection)

| 情况 | 保护机制 |
|------|----------|
| 金币吃紧买不起药 | 每个商店必定有一个特价药水瓶(3 金币) |
| 0 钥匙进入宝箱房 | 木箱无需钥匙可开一次 |
| 该层所有房间已探索但 HP 极低 | Boss 房前出现一个免费篝火(回复 5 HP) |
| 玩家已连续死亡 5 次 | 下次开局自动获得"不朽石"(每层免死一次)|

### 8.3 难度的"波峰波谷"设计

```
难度曲线不是单调递增的，而是:
  
  第1层 → 简单 (上手)
  第1层Boss → 小波峰 (第一次考验)
  第2层 → 略升
  第2层特殊房间 → 波谷 (喘息+回复)
  第2层Boss → 波峰
  第3层 → 结合变异
  第3层Boss → 最大波峰
```

玩家不会感到"一直变难"，而是**张弛有度的节奏**——这更接近好游戏的体验曲线。
