# 美术风格指南

> ⚠️ **内容已过时**：本文档为早期微信小游戏时代的「像素风·暗黑地牢」风格方案，与当前 TapTap 高清卡通动物风严重不符，仅保留作历史参考；以 `design/美术资源需求清单.md` / `docs/美术资源生成与入库规范.md` 为准。

> **版本**: v1.0 | **日期**: 2026-06-25
> **风格定位**: 像素风 · 低面数矢量 · 暗黑地牢探险
> **画布基准**: 横屏设计基准（TapTap 安卓；原微信小游戏竖屏 750×1334 已不适用）
> **色板约束**: 每区域 3 色主调 + 2 色点缀

---

## 1. 风格总述

```
回到地面的美术风格 = 像素角色 + 矢量场景 + 光效渲染

核心原则:
  "一眼认得出, 两眼看得出区别"

  ① 像素角色 (48×48~96×96) → 清晰辨识, 携带武器特征
  ② 低面数场景 Tile (32×32) → 可平铺, 每区域不同
  ③ 光效渲染 (粒子/闪白/发光) → 战斗反馈全靠特效
```

### 与市场竞品的视觉差异

```
提灯与地下城:   Q 版日系, 圆润角色, 暖色调
贪婪洞窟:       写实暗黑, 高细节, 冷暗色调
回到地面:       像素角色 + 矢量场景, 每区域色调鲜明不同
```

## 2. 角色设计规范

### 2.1 角色比例

```
像素角色统一比例:

        ┌──────────┐
        │  头部   │  16×16px
        ├──────────┤
        │  身体   │  24×32px
        ├──────────┤
        │  腿脚   │  8×16px
        └──────────┘
  总尺寸: 48×64px  (角色在网格中占 1.5 格)

  碰撞盒: 32×48px  (比视觉尺寸略小, 手感更好)
  武器挂点: (身体右侧, 距顶 24px)
  特效挂点: (头顶上方 8px)
```

### 2.3 怪物配色规则

```
每个区域的怪物遵循该区的色调定位:

遗忘森林: 主色系 #4A~8A 绿系, 偶尔紫色点缀(毒)
幽暗墓穴: 主色系 #5A~7A 灰紫, 幽灵发光 #AAFFAA
熔岩火山: 主色系 #AA~ 红橙, 阴影 #3A 深黑
冰封雪原: 主色系 #7A~DD 蓝白, 冰晶高光 #DDFFFF
剧毒沼泽: 主色系 #3A~5A 墨绿, 毒泡 #AA77AA 紫
暗影深渊: 主色系 #2A~3A 紫黑, 虚空 #6644AA 发光

同区域精英怪 = 普通怪 × 1.2 倍体型 + 发光特效
同区域 Boss   = 普通怪 × 1.5~2 倍体型 + 专属特效
```

## 3. UI 风格规范

### 3.1 UI 基础视觉

```
背景色:    #1A1A2E (深蓝黑, 全 App 基调)
面板底色:  #2A2A3E (稍亮, 弹窗/列表用)
卡片底色:  #3A3A4E (卡牌/物品格)
边框色:    #5A5A7E (默认边框)
高亮色:    #D4AF37 (金色, 选中/激活)
文字主色:  #F0E8D8 (米白)
文字辅色:  #8A8A9E (灰色, 辅助信息)
危险色:    #FF4444 (红色, 扣血/危险)
安全色:    #44FF88 (绿色, 回血/安全)
```

### 3.2 按钮规范

```
按钮尺寸:  200×60px (大) / 140×44px (中) / 80×32px (小)
圆角:      8px
三态差异:
  默认:   面板底色 + 边框色
  悬停:   边框色变亮 + 加外发光
  按下:   整体下移 2px + 亮度降低 20%
```

### 3.3 面板规范

```
九宫格切图 (9-slice):
  边角: 8×8px (圆角, 不缩放)
  边:   8×1px (水平/垂直拉伸)
  中心: 1×1px 透明 (内容区域)

面板边框粗细: 2px
面板阴影偏移: (0, 4px) 透明度 30%
```

### 3.4 字体规范

```
标题:  24px 粗体, #D4AF37 金色
正文:  16px 常规, #F0E8D8 米白
提示:  12px 常规, #8A8A9E 灰色
数值:  等宽数字, 带 #44FF88(正) #FF4444(负) 着色

行距:  1.6 倍
```

## 4. 地图 / Tile 风格

### 4.1 网格风格

```
6×6 网格, 每格 32×32px
玩家在网格中间行走

Tile 设计要求:
  ① 每区域 4 种核心 Tile (floor/wall/thorn/highground)
  ② 平铺流畅, 边缘无缝
  ③ 颜色饱和度低 (看起来像旧地牢, 而非新鲜油漆)
```

### 4.2 6 区域 Tile 设计方向

```
森林:  地板=苔藓砖块 / 墙=藤蔓石 / 荆棘=树根 / 高台=树桩
墓穴:  地板=破损石板 / 墙=灰石+骨 / 荆棘=骨刺 / 高台=石棺
火山:  地板=焦黑岩 / 墙=熔岩石 / 荆棘=裂缝 / 高台=熔岩柱
雪原:  地板=冰晶板 / 墙=冰墙 / 荆棘=冰刺 / 高台=冰柱
沼泽:  地板=腐木 / 墙=枯木 / 荆棘=毒藤 / 高台=树根台
深渊:  地板=虚空板 / 墙=暗影壁 / 荆棘=裂隙 / 高台=悬浮石
```

## 5. 特效风格

### 5.1 特效生成规则

```
战斗特效: 像素粒子 + 半透明叠加, 不遮挡角色轮廓
  命中:    白色闪白 (2 帧) + 1~2 个像素碎片
  暴击:    红色爆裂 (4 帧) + 屏幕震动 0.1 秒
  元素反应: 2 色调和 + 发光辉光 + 粒子扩散
  Boss 技能: 全屏半透明遮罩类特效, 提示圈+爆发

特效尺寸: 不超过 96×96px
特效帧数: 4~8 帧 (60FPS 下约 0.07~0.13 秒)
```

### 5.2 11 种元素反应视觉指引

```
爆燃 (Fire+Poison):    红绿混合爆炸 + 火花四溅
蒸汽 (Fire+Frost):      白色蒸汽扩散 + 水珠
超载 (Fire+Lightning):  金色爆裂 + 电弧四射
融化 (Fire+Shadow):     紫红火焰 + 暗影拖尾
冻结 (Frost+Lightning): 蓝白冰晶 + 电光冻结
碎裂 (Frost+Poison):    青绿碎裂 + 冰晶散落
腐朽 (Frost+Shadow):    灰紫雾气 + 缓慢扩散
传导 (Lightning+Poison): 黄绿电弧弹射
虚空 (Lightning+Shadow): 紫黑漩涡 + 吸引粒子
腐蚀 (Poison+Shadow):    暗绿液体滴落
光辉 (Shadow+Holy):      白金光柱 + 辉光粒子

颜色公式: 反应色 = (元素 A 色 + 元素 B 色) / 2
```

## 6. 动画帧规范

### 6.1 各类型动画帧数

```
角色动作:
  Idle:    4 帧 (呼吸浮动, 约 0.5 秒循环)
  Walk:    6 帧 (迈步交替, 约 0.4 秒循环)
  Attack:  4 帧 (前摆+命中+收招, 约 0.3 秒)
  Skill:   6 帧 (蓄力+释放+收势, 约 0.5 秒)
  Dodge:   4 帧 (前滚翻, 约 0.3 秒)
  Hit:     2 帧 (后仰+恢复, 约 0.15 秒)
  Death:   4 帧 (倒下+消失, 约 0.5 秒)

怪物动作:
  Idle:    4 帧 (轻微浮动/呼吸)
  Attack:  4 帧
  Hit:     2 帧 (闪白)
  Death:   4 帧 (碎裂/消散)

Boss 动作:
  Idle:    4 帧
  Attack:  6 帧
  Skill:   8 帧 (Boss 技能, 带预警提示)
  Phase:   6 帧 (转阶段特效)
  Death:   8 帧 (消亡动画)
```

### 6.2 动画过渡规则

```
任何动作之间不强制过渡帧 (像素风格允许瞬间切换)
但禁止: 从 Idle→Death 跳帧 (必须先 Hit)

状态机允许的跳转:
  Idle ↔ Walk
  Idle ↔ Attack
  Idle → Hit → Idle
  Idle → Death (不可逆)
  Attack → Dodge (允许攻击中翻滚打断)
```

## 7. 提交文件标准

| 检查项 | 标准 | 不合格情况 |
|--------|------|-----------|
| 色调一致性 | 符合 0.2 节色值 ±10% | 森林怪物用了红色 |
| 尺寸合规 | 符合每类资源指定尺寸 | 怪物图 64×64 但角色当 32×32 出 |
| 命名规范 | 符合第 10 节命名规则 | `xxx.png` / `未命名.png` |
| 文件大小 | 单文件小于分类上限 | BGM > 200KB |
| 透明通道 | UI/角色/怪物/特效必须含 Alpha | 白色背景未抠图 |
| 平铺性 | Tile 四边无缝 | 地板 Tile 有明显接缝 |
| 帧数 | 动画帧数等于指定值 | Attack 做了 10 帧 (超限) |

## 8. 风格参考

```
视觉参考关键词 (给 AI 参考):
  pixel art, low-poly vector, dark dungeon,
  16-bit color palette, tileable textures,
  isometric grid, 4-directional sprites,
  particle effects, alpha glow, sprite sheet

游戏色调气质:
  "dark but readable" —— 暗黑但不糊
  "colorful gloom"   —— 整体暗淡但每个区域有鲜艳点缀
  "pixel but smooth" —— 像素风格但动画流畅
```
