# 事件系统 · 详细规格

> **版本**: v1.0 | **日期**: 2026-06-25 | **设计者**: GameDesigner
> **系统层**: 内容层 P1 | **依赖**: 地牢生成系统 (事件房是房间类型之一)
> **核心机制**: 参数化随机生成 — 场景(8)×条件(6)×状态(6)×后果(12) = 3,456+ 种变体

---

## 1. 设计目的

**玩家体验**: 在前行的路上遇到一个"出乎意料"的选择——不是在战斗中做操作决策，而是在十字路口做价值判断。

**与战斗房的区别**:
```
战斗房: "怎么打?" → 操作 + 走位
事件房: "怎么选?" → 价值判断 + 风险评估
```

**核心感受**: 每次进入事件房都是一次"2 选 1"的谜题——玩家的当前状态（HP/金币/装备）直接影响最优解。

## 2. 生成引擎

### 2.1 生成流程

```
每个事件房的生成流程:

步骤 1: 随机选择一个场景 (Scene)
  从 8 种场景中按权重选取

步骤 2: 检测玩家当前状态 (State)
  HP 比例 / 金币数量 / 是否持有钥匙 / 已击杀数 / 当前层数 / 持有元素

步骤 3: 根据状态匹配条件 (Condition)
  条件命中 → 生成相应的 A/B 选项
  条件未命中 → 使用默认选项

步骤 4: 从后果池中分配后果 (Consequence)
  后果与场景/条件匹配，确保叙事合理性

步骤 5: 呈现给玩家
  显示场景描述 + 选项 A + 选项 B + 风险提示
```

### 2.2 场景池 (8 种)

| 场景 | 权重 | 描述文本 (示例) |
|------|------|----------------|
| 破碎祭坛 | 20% | 一座破损的献祭台，台面上刻着古老的符文，周围散落着骨骸和金币 |
| 发光水晶 | 18% | 墙壁上镶嵌着一颗脉动的能量水晶，散发出不稳定的光芒 |
| 古代雕像 | 15% | 一尊持剑的战士雕像，剑刃上刻着一段褪色的铭文 |
| 扭曲传送门 | 14% | 空气扭曲着形成一个旋转的次元裂隙，发出低沉的嗡鸣声 |
| 魔法井 | 13% | 一口深不见底的蓝色井口，水面泛着微弱的荧光 |
| 神秘商人 | 10% | 一个兜帽遮面的流浪商人蹲在角落，面前摆着几个发光的瓶子 |
| 陷阱走廊 | 5% | 走廊两侧墙壁布满箭孔，地板上能看到压力板的痕迹 |
| 休眠火山 | 5% | 地面裂开一道缝隙，岩浆在下方缓慢流动，散发着灼热的空气 |

### 2.3 状态检测 (6 项)

```
引擎在生成事件时检测的玩家状态:

S1: HP_RATIO     →  高(>70%) / 中(30-70%) / 低(<30%)
S2: GOLD         →  富裕(>20) / 中等(10-20) / 贫穷(<10)
S3: HAS_KEY      →  有钥匙 / 无钥匙
S4: KILLS_THIS_FLOOR → 多(>10) / 少(<5)
S5: FLOOR_NUM    →  浅(1-2) / 中(3-4) / 深(5)
S6: HAS_ELEMENT  →  持有特定元素附魔 (Fire/Frost 等)
```

### 2.4 条件匹配表

```
条件决定选项 A 和选项 B 分别是什么。

通用条件 (任何场景, 60% 概率):
  A: 消耗 3~5 HP 换 6~10 金币
  B: 消耗 5~8 金币换回复 5~10 HP

特定条件 (依场景不同, 40% 概率):
  例: 发光水晶 + HP_RATIO=低
    A: 触碰水晶 → 回复 8 HP (免费)
    B: 砸碎水晶 → 获得 10 金币, 但怪物 ATK+1 (本层)
  
  例: 神秘商人 + GOLD=富裕
    A: 20 金币换一瓶高级药水 (回复全部 HP)
    B: 10 金币换一个临时 Buff (ATK+3 本场)
  
  例: 古代雕像 + HAS_KEY
    A: 用钥匙开启雕像底座 → 获得随机遗物
    B: 无视 → 无事发生
```

### 2.5 后果池 (12 种)

| 后果 | 类型 | 数值范围 | 说明 |
|------|------|----------|------|
| 回复 HP | 正 | 5~15 | 立即回血 |
| 失去 HP | 负 | 3~10 | 立即扣血 |
| 获得金币 | 正 | 5~20 | 立即加钱 |
| 失去金币 | 负 | 5~15 | 立即扣钱 |
| 获得钥匙 | 正 | 1 | 白嫖一把钥匙 |
| 失去钥匙 | 负 | 1 | 钥匙被拿走 |
| 获得临时 Buff | 正 | 3 场战斗 | ATK/DEF/SPD 临时提升 |
| 施加 Debuff | 负 | 本层有效 | 怪物变强或玩家变弱 |
| 下房怪物削弱 | 正 | 1 场战斗 | 下个战斗房怪物 HP-50% |
| 下房怪物增强 | 负 | 1 场战斗 | 下个战斗房怪物 ATK+2 |
| 获得随机遗物 | 正 | 稀有 | 从遗物池中随机抽取 1 个 |
| 无事发生 | 中立 | — | 浪费一个房间 |

## 3. Mechanic: 事件房交互

```
Mechanic: 事件房交互

Purpose: 在战斗之外提供"2 选 1"的价值判断时刻
Player Fantasy: "选左边还是选右边?"
Input: 
  1. 进入事件房 → 展示场景描述 (2 秒渐入)
  2. 展示选项 A (左按钮) + 选项 B (右按钮)
  3. 选项上标注后果 (绿色=正, 红色=负, 灰色=中立)
  4. 玩家点击选择

Output: 对应后果立即执行 (HP/金币/钥匙变更, 或获得 Buff/Debuff)

Success Condition: 后果正常执行, 玩家继续前进
Failure State: 不适用 (事件房不造成死亡)

Edge Cases:
  - 选择"失去 HP"但玩家 HP=1 → 后果改为"失去 1 HP 但保留 1 HP"
  - 选择"失去金币"但金币=0 → 选项置灰不可选
  - 选择"获得遗物"但遗物池为空 → 改为获得 15 金币
  - 玩家长时间不做选择 (15 秒) → 自动执行 A

Tuning Levers:
  - 各场景权重
  - 后果数值范围
  - 时间限制 (当前: 15 秒)

Dependencies: 地牢生成系统 (事件房是房间类型)、资源系统 (HP/金币/钥匙)
```

## 4. 事件-场景生成示例

```
示例 1: 破碎祭坛 + HP_RATIO=低
描述: "祭坛上刻着古老的符文, 似乎在渴望着什么..."
A: [献祭] 消耗 3 HP → 获得 8 金币 (价值判断: 残血换钱?)
B: [祈祷] 消耗 5 金币 → 回复 6 HP (价值判断: 花钱买命?)

示例 2: 发光水晶 + HAS_ELEMENT=Fire
描述: "当你靠近水晶, 你腰间的火焰附魔与之共鸣..."
A: [触碰] 水晶吸收你的火焰 → 你失去火焰附魔, 但 HP 全满
B: [释放] 你将火焰灌入水晶 → 下个房间全体怪物灼烧 3 回合

示例 3: 神秘商人 + GOLD=富裕 + FLOOR_NUM>=3
描述: "商人推了推兜帽: '你看起来能付得起好价钱...'"
A: [高阶药水] 20 金币 → HP 全满 + ATK+2 本层
B: [幸运符] 15 金币 → 暴击率 +15% 本层

示例 4: 扭曲传送门 + KILLS_THIS_FLOOR=多
描述: "传送门感受到你身上的杀气, 裂开得更大了..."
A: [进入] 跳过一个普通战斗房, 直接到下一个房间
B: [摧毁] 获得 10 金币, 但怪物感知到震动 → 下房多 1 怪

示例 5: 古代雕像 + FLOOR_NUM=1 (新手保险)
描述: "雕像看起来像是这个地牢的守护者..."
A: [触摸] 回复 10 HP (免费!)
B: [检查] 获得 1 把钥匙 (免费!)
    → 新手层: 两个选项都是正收益, 不会坑新手

示例 6: 陷阱走廊 - 默认 (无特殊条件)
描述: "走廊布满陷阱, 你能看到地板上的压力板..."
A: [强行通过] 失去 5 HP, 获得 8 金币
B: [小心拆除] 获得 1 把钥匙 (慢, 但安全)
```

## 5. 事件房平衡

### 5.1 后果数值对标

```
事件收益应该约等于 1 个战斗房的 50-70% 收益:
  战斗房: 3~5 金币 + 可能掉钥匙(20%)
  事件房: 5~10 金币等价物 (取中值)

因此:
  回复 HP 5~10 = 约 0.5 瓶药水 = 2.5~5 金币价值
  钥匙 1 把 = 10 金币价值
  
事件房的"风险"选项:
  损失 HP 3~10 = 等价于 1~3 次怪物攻击
  损失金币 5~15 = 相当于白打 1~2 个战斗房
```

### 5.2 稀有事件保底

```
每 3 层至少出现 1 个稀有后果的事件:
  - 获得随机遗物 (稀有, 约 5% 概率)
  - 下房全体怪物 HP-50% (稀有, 约 5% 概率)
  - 全层暴击率 +15% 永久 (超稀有, < 1% 概率)
  
稀有事件触发条件:
  - 玩家 HP_RATIO=低 时更容易触发"回复类稀有事件"
  - 玩家 GOLD=贫穷 时更容易触发"金币类稀有事件"
  - 本质上: DDA 隐性修正事件池权重
```
