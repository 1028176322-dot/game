## 2. Characters（35 张）— 待修复 ⚠️

| 维度 | 值 |
|------|----|
| 目标尺寸 | 192×768（竖长 sprite strip，多帧拼接） |
| 当前 prompt | `"Unified visual direction: bright cheerful cartoon animal adventure..."` 含场景描述（`soft forest light`）|
| 分类 | `characters` → `asset_kind = "character"`|
| 后处理 | 归入 `TRANSPARENT_CATEGORIES`、`MATTE_CATEGORIES` |
| 生成尺寸 | 因不是 `ORNAMENT_TYPES`，走非装饰框逻辑：目标≤128？192>128 → scale=1.5 → 288×1152 |

### 发现问题

1. **DETAIL_ANCHORS 缺少 characters 条目** — 管线构造 prompt 时不会追加 detail anchor，导致 AI 生成方向不明确。
2. **prompt 含场景描述** — "bright cheerful... soft forest light" 会让 AI 在 sprite strip 中画场景，造成背景残留。
3. **竖长 192×768 格式** — 这种 strip 格式（多帧动画竖排）在后处理时需要特殊处理，目前 `fit_rgba_to_canvas` 可能把各帧裁切不均匀。

### 修复建议

- 添加 DETAIL_ANCHORS["characters"] = "Full-body character sprite sheet with multiple animation frames stacked vertically, each frame on plain solid background, clean transparent edges."
- 重写 characters prompt：去掉场景描述，改为描述角色动作/姿势 + 透明纯色背景。
- 验证 `fit_rgba_to_canvas` 对 192×768 strip 的处理逻辑。

3. **体量大（120 张）** — 建议分 3 个子批（每批 40 张），出 contact sheet 逐个视觉验收。

### 修复建议

## 8. UI（173 张）— 已部分验证

| 子类型 | 数量 | 生成方式 | 状态 |
|--------|:----:|---------|:----:|
| procedural_ui（btn/card/panel/input/slot） | ~75 | UI Kit 程序化 | ✅ 8 张已入库 |
| icon 类（area_badge、avatar、map_icon 等） | ~98 | Agnes AI | ⏳ 2 张已入库 |

### UI icon 类的特殊问题

- 非 `KIND_PROCEDURAL_UI` 匹配的 UI 资源（如 `login_panel`、`event_panel`、`settings_panel`）走 AI 生成。
- 这些是完整的大型面板，需要完整场景/UI 风格，与 icon_item_key 等小图标不同。
- 需要区分"完整 UI 面板"和"小 UI 元素"：面板应有装饰细节，小元素应简洁。

### 修复建议

- 对 `_panel` 结尾的 UI 资源，prompt 允许适当装饰和渐变，但仍需透明边缘和居中。
- 对 avatar/badge/room_icon 等，保持 icon 模板。
