# ADR-003：为什么不能只靠文件名判断删除

- **状态**: Accepted
- **日期**: 2026-07-11
- **决策人**: 项目治理（多 AI 协作）

## 背景
最初脚本用 `BAD_PATTERNS` 匹配裸 `debug` / `tmp` / `backup` 来标记可删文件。实践发现：
- 裸 `debug` 会命中 `DebugPanel.ts`（已接入 `GameBootstrap`，Dev 门控，必须保留）、`debug_save_replay.test.ts`（正式测试）。
- 裸 `tmp` / `temp` 可能命中合法模块（某些模块名含 `temp`）。
- 「MD5 相同」≠「无人引用」：重复组的部分副本可能正被某脚本 import。

## 决策
- 文件名匹配**只用边界形式**：`_debug_` / `_tmp_` / `.backup_`，不匹配置换裸 `debug` / `tmp` / `temp`。
- 弱标记 `tmp_` / `temp_` 前缀**永不自动删**，仅作提示，须人工复核。
- 删除判定**联合多信号**：git tracked / ignored / Modified + `git grep` 全文引用数 + `package.json` scripts。
- "完全相同"不自动删：重复组仅列出，删除前须重新 grep 引用 + 检查 package.json。

## 后果
- 正面：避免误删 `DebugPanel.ts` 等核心文件；引用变化能被安全门拦截。
- 负面：标记精度下降（更多文件进入"需人工"队列），但安全优先。

## 反选项
- 纯文件名匹配：简单但已证明会误伤正式代码，否决。
