# ADR-001：为什么采用双层目录结构

- **状态**: Accepted
- **日期**: 2026-07-11
- **决策人**: 项目治理（多 AI 协作）

## 背景
仓库根 `E:\game` 与 `E:\game\回到地面` 存在同名子树（`docs/ tools/ assets/ art_source/ .workbuddy/` 等）。初次扫描易误判为"重复/冗余"，可能触发误删或"合并修复"。

## 决策
将仓库显式定义为**双层结构**：
- **根 `E:\game`** = 跨项目层：规划、审计、美术生成管线（`tools/` 下的 `fix_bg_prompts*` / `gen_missing_179` 等）。
- **`E:\game\回到地面`** = 项目层：Cocos 主工程（源码、引擎校验/CI 工具、游戏配置）。
两者职责互补，均为活跃文件，不属于需要清理的重复。

## 后果
- 正面：AI 不再把跨项目层误当冗余删除；根 `tools` 脚本（被 `export_runtime_replace_prompts.py` import）被确认为活管线。
- 负面：路径引用需严格区分两层，文档中须用相对路径并标注所属层；`docs/` + `docs/progress/` 同名属"方案+进度 stub"约定，不可删。

## 反选项
- 合并为单层：会破坏跨项目规划与项目工程的边界，且根层脚本与项目层脚本命名相同但职责不同，合并后需大量重命名，风险高。
