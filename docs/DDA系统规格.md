# DDA 动态难度系统 · 详细规格

> **版本**: v1.0 | **日期**: 2026-06-25 | **设计者**: GameDesigner
> **系统层**: 元层 P2 | **依赖**: 全局 (感知玩家状态后修正)
> **核心原则**: 玩家不可感知 DDA 的存在 (调整值对玩家隐藏)

---

## 1. 设计目的

**防止以下体验**:
- "我死了, 但不知道为什么" → 瓶颈检查
- "我连续死了 5 次了, 不玩了" → 挫败保护
- "这怪完全打不动" → 软锁保护
- "这层没有回复房, 我要死了" → 资源保底

**DDA 不是"降低难度", 而是"确保体验曲线不崩溃"**。好玩家仍然觉得难, 差玩家也不至于想删游戏。

---

## 2. 触发条件与调整

### 2.1 7 种触发条件

| 编号 | 触发条件 | 检测时机 | 调整动作 | 调整值 |
|------|----------|----------|----------|--------|
| D01 | 连续死亡 2 次 | 每次死亡 | 下一局初始层变异权重偏向利好 (薄雾/时空扭曲) | 权重 +15 |
| D02 | 连续死亡 5 次 | 每次死亡 | 下一局开局赠送"不朽石"(每层免死 1 次) | 1 次性 |
| D03 | 当前 HP < 30% 且 3 个房间内无回复房 | 进入新房间时 | 下次可到达房间中必有 1 个回复房 | 强制替换 |
| D04 | 未触发任何元素反应, 连续 3 场战斗 | 战斗结算 | 下个宝箱必出双元素附魔武器 | 引导发现 |
| D05 | Boss 前玩家 ATK 低于层数 × 2 | 进入 Boss 房前 | Boss 房前新增免费篝火 (回 5 HP) | 1 次性 |
| D06 | 同一怪物装配导致玩家死亡 3 次以上 | 死亡时 | 该怪物不再出现在后续局中 (当前种子) | 移除 |
| D07 | 玩家已清完该层所有房间, 但 HP < 20% | 进入 Boss 房前 | Boss 房前自动回 10 HP (剧情式: "你感到一股力量涌入") | 1 次性 |

### 2.2 调整幅度上限

```
DDA 调整值有软上限:

HP 补给:    最高不超过玩家最大 HP 的 40%
ATK 修正:   最高不超过 ±2 (玩家不可感知)
怪物 HP 修正: 最高不超过 ±20%
利好变异权重: 最高 +30
免费道具: 每局最多 1 件

超出上限 → 交给 playtest 观察是否正常
如果 playtest 发现 DDA 频繁触发 → 说明基础难度设计有问题, 不是 DDA 能解决的
```

## 3. 软锁保护

### 3.1 3 层保护机制

```
第一层: 商店必有特价药水
  → 任何时候进入商店, 必定有 1 瓶 3 金币的药水 (原价 5)
  → 防止: 玩家没钱买药, 慢性死亡

第二层: 无钥匙可开木箱
  → 宝箱房中, 如果没有钥匙, 木箱可免费开一次
  → 防止: 玩家无钥匙进宝箱房干瞪眼

第三层: 连续 5 次死亡送不朽石
  → D02 触发
  → 防止: 玩家失去信心
```

### 3.2 软锁保底 vs 正常体验

```
软锁保底是"防止游戏崩溃"的最后一道防线。
它不应该在正常游戏中频繁触发。

正常游戏中:
  - 商店药水 5 金币 → 大部分玩家买得起
  - 钥匙从战斗掉落 (20% 概率) → 每局约得 3~5 把
  - 连续 5 次死亡 → 在正常难度下不应频繁发生
  
如果 playtest 发现软锁保底频繁触发:
  → 说明基础数值有问题, 需要调整, 而不是依赖 DDA
```

---

## 4. DDA 的不可见原则

```
DDA 调整对玩家完全不可见:

不可做的: "检测到您连续死亡, 难度已降低" ❌
可做的: 暗置换房间类型, 但不告诉玩家

例外:
  免费篝火 (D05) → 可展示, 但用"剧情式"呈现
  "你在 Boss 房前发现了一个篝火, 似乎有人不久前在这里休息过"
  → 玩家不会觉得这是"系统在帮我"
```
