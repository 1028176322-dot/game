# Agent Contract — 10 条铁律

> **编码**: UTF-8
> 作用: AI Agent（Claude Code / Codex / Cursor / OpenHands / WorkBuddy）执行本项目的**最高约束**。每次动手前必须加载并遵守。
> 来源: `docs/2D转3D实施计划.md` §1.10。本文件是 §1.10 的权威展开版。

AI 在执行任何 2D→3D 相关任务时，**无条件遵守以下 10 条**。任何一条与任务卡冲突时，以本 Contract 为准并**停止报告**。

---

## 1. 禁止修改计划

不得修改 `docs/2D转3D实施计划.md`、各 `D0-x.md` / `demoN.md` 任务卡、`_architecture_report.md` 的内容。
计划是人工编写的约束，不是 AI 的可写对象。发现计划有误 → 停止并报告，等人工修订。

## 2. 禁止修改接口

接口签名/命名/职责以 `docs/2D转3D全面升级方案.md` §5.x 给出的为准。
不得新增接口、不得改名、不得改参数、不得合并/拆分方法。实现必须 1:1 贴合权威签名。

## 3. 禁止新增职责

每个模块只做其接口声明的事。
例：`GameContext` 只做服务注册/获取/销毁，不得顺手做配置加载（那是 ConfigDatabase 的事）；`Logger` 只做日志，不得做埋点上报。

## 4. 禁止猜测需求

任务卡未写明的需求，不得脑补。
缺信息（如某 getter 的返回类型在 `ConfigTypes` 缺失）→ 用 `unknown` + `// TODO` 占位并报告，**不得自行发明字段或语义**。

## 5. 发现冲突立即停止

以下任一情况出现 → **立即停止当前任务**，输出冲突点，等待人工：
- 与架构红线 1–6 冲突
- 与任务卡"禁止修改范围"冲突（需改范围外文件）
- 与设计文档 §5.x 签名冲突
- 多个权威来源互相矛盾

## 6. 任何超出范围必须报告

"允许修改范围"之外的任何改动（哪怕"只是顺手修个小 bug"）→ 必须报告，不得私自扩大修改面。
未列出的目录默认禁止修改。

## 7. 不能为了通过测试修改测试

测试失败 → 改**实现**使其通过，不得删断言、改预期值、注释用例、`xit` 跳过。
测试是约束，不是障碍。

## 8. 不能关闭 Lint

不得修改 lint 配置以绕过报错、不得 `// @ts-ignore` 掩盖类型错误（除非经人工书面豁免，且须记录理由）。
类型安全是架构约束的一部分。

## 9. 不能关闭 Validate

`npm run validate:all` 必须全过。不得改 `.gitignore` 绕过、不得临时禁用门禁脚本、不得 `--skip` 关键门禁。
validate 失败 = 任务未完成。

## 10. 不能降低架构约束

红线 1–6（禁止直连 PhysicsSystem / 禁止 switch(skillId) / 必须 ILifecycle / 禁止 new 服务 / 禁止 Math.random / 仅 SceneFlowService 可 loadScene）不可放宽、不可"暂时绕过"、不可"先跑通再改"。
任何降级请求 → 停止并报告。

---

## 执行前确认门禁（与 Execution Plan 配合）

- 写代码前必须输出 **Execution Plan** 并等待 `Plan Approved`（人工模式）或 `AUTO_APPROVE=false`（自动模式默认等待）。
- Plan 必须包含：将新增/修改/删除的文件、预计影响文件数、不会修改的禁止范围、预计步骤。
- 未获批准前，**不得创建/修改任何源文件**（仅允许只读扫描与生成报告）。

## 违反后果

违反任一铁律 = 任务失败，即使 `validate:all` 通过也不算完成。发现违规须主动回滚并报告。
