# D0-3 — 实现 ConfigDatabase（类型化查询，复用现有数据）

> 任务类型: 基础设施 / 配置 ｜ 依赖: D0-0, D0-2 ｜ Token Budget: 1 文件
> ★ 重点: 本任务是最典型的"禁止重复实现"落点。

## 输入
- `docs/2D转3D全面升级方案.md §5.3` 签名。
- `docs/ai-tasks/_architecture_report.md §5/§7`：`ConfigService` + `ConfigManager` **已做配置加载**，本任务必须复用，不得重写。
- 类型来源：`assets/scripts/config/ConfigTypes.ts` 既有类型。
- 查重对象：`ConfigDatabase` / `Database` / `Repository` / `DataCenter` / `ConfigManager`（已有，禁止再写）。

## 输出
- 新增 `assets/scripts/core/ConfigDatabase.ts`，导出 `class ConfigDatabase`：
  - `loadAll(): Promise<void>`（从 `ConfigService` 读取并做类型化封装，**不得自行 load**）
  - `getSkill(id): SkillConfig` `getMonster(id): MonsterConfig` `getBoss(id): BossConfig`
  - `getEffect(id): EffectConfig` `getAI(id): AIConfig` `getCamera(id): CameraConfig` `getAudio(id): AudioConfig`
  - 类型全部**复用** `config/ConfigTypes.ts` 既有类型，**不得新定义配置类型**。

## 严格约束
- §5.3 明确"System 不得自行 load"——`loadAll()` 仅封装 `ConfigService` 既有数据；**不得**新增方法、**不得**改名、**不得**增加"写入/热更"职责。
- **禁止重复实现**：绝不为配置加载新建 loader/reader；若 `ConfigService` 缺少某方法，报告并复用其公开 API，不得另起一套。
- 若某 getter 的配置类型在 `ConfigTypes` 中暂缺 → 返回 `unknown` 并标 `// TODO: add type in ConfigTypes`，**不得**在 `ConfigDatabase` 内新增字段。

## 允许修改范围
- 新增 `assets/scripts/core/ConfigDatabase.ts`

## 禁止修改范围
- `assets/scripts/config/**`（含 ConfigService / ConfigManager / ConfigTypes / ConfigSchemas / ConfigError）
- `assets/scripts/**` 除本文件与 `GameContext.ts` 外的全部

## 完成定义 (DoD)
- [ ] vitest：用 mock `ConfigService` 数据，`getSkill` 等返回正确类型；`loadAll` 调用一次且兼容既有结构
- [ ] 不破坏现有 `ConfigService.loadAll()` 行为（集成点在 D0-5 验证）
- [ ] `npm run validate:all` 9 门禁通过
- [ ] `assets/scripts/core/ConfigDatabase.ts` 未引入 `cc`
