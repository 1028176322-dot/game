# D0-0 — 建立 core/ 单测基座（vitest）

> 任务类型: 基础设施 / 测试基座 ｜ 依赖: 无（Demo0 第一步）｜ Token Budget: ≤3 文件

## 输入
- `回到地面/package.json`：当前**无 `test` 脚本、无测试框架**（见 `_architecture_report.md` §2）。
- 目标：建立对 `assets/scripts/core/` 纯 TS 模块（无 `cc` 依赖）的单元测试能力，让 `npm run test` 成为真实可跑门禁。

## 输出
- 新增 `回到地面/vitest.config.ts`：roots 指向 `tests/`，`environment: 'node'`，仅包含 `core/` 相关测试。
- 新增 `回到地面/tests/core/*.test.ts`：首批占位测试（可空，但 `npm run test` 必须 exit 0）。
- 修改 `回到地面/package.json`：新增 `devDependencies: { "vitest": "^2" }` 与 `scripts.test = "vitest run"`。

## 严格约束
- 仅引入 `vitest`；测试文件放 `tests/`，**不得**放进 `assets/scripts/`。
- 四大基础设施文件（D0-1~D0-4 产出）**不得因测试而引入 `cc` 依赖**（保持纯 TS，可在 node 运行）。
- **不得**改写现有业务代码以"适配测试"。
- 禁止重复实现：本任务不创建任何业务模块，仅测试基座。

## 允许修改范围
- 新增 `回到地面/vitest.config.ts`
- 新增 `回到地面/tests/**`
- 修改 `回到地面/package.json`（仅 `scripts` + `devDependencies`）

## 禁止修改范围
- `assets/scripts/**` 全部（含 `core/` 现有文件）
- `回到地面/tsconfig.json` 及其它配置
- 其它任何目录

## 完成定义 (DoD)
- [ ] `npm run test` 可运行且 exit 0（首批测试可为空）
- [ ] `npm run validate:all` 9 门禁通过
- [ ] `assets/scripts/core/` 四大文件（后续 D0-1~D0-4）确认未引入 `cc`
