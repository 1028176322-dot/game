# D0-2 — 实现 GameContext（ServiceLocator）

> 任务类型: 基础设施 / 依赖注入 ｜ 依赖: D0-0（vitest 基座）｜ Token Budget: 1 文件
> 注意: 执行顺序在 D0-1 之前，因为 `ILifecycle.initialize(ctx: GameContext)` 依赖其类型。

## 输入
- `docs/2D转3D全面升级方案.md §5.2` GameContext 签名 + 服务令牌示例。
- `docs/ai-tasks/_architecture_report.md §7`：确认无既有 ServiceLocator（安全新建；但 `GameManager`/`ConfigManager` 已有单例获取能力，**GameContext 不得重实现其单例管理**，只做服务注册/获取）。
- 查重对象：`ServiceLocator` / `GameContext` / `DIContainer` / `ServiceRegistry`。

## 输出
- 新增 `assets/scripts/core/GameContext.ts`，导出：
  - `class GameContext`：`register<T>(token: string, impl: T): void` / `get<T>(token: string): T`（未注册**必须抛错**）/ `onDestroy(): void`（反向销毁，后注册先销毁）
  - 13 个服务令牌常量：`ICollisionService` `IAssetCache` `IAnimationController` `ILogger` `IAudioService` `IConfigDatabase` `ISaveManager` `IDebugService` `ICameraBrain` `IEventBus` `IRuntimeState` `IReplayRecorder`（依 §5.2 列表；第 13 个由 §5.2 全量列出）
  - 内部 `private services = new Map<string, unknown>()`

## 严格约束
- 严格按照 §5.2 签名；**不得**新增方法、**不得**改名、**不得**增加职责（不得做生命周期广播——那是 LifecycleManager；不得提供 `new` 工厂——业务禁止 new，见红线 4）。
- `get` 未注册令牌**必须抛 `Error`**（非返回 undefined，防漏注）。
- `register` 同 token 二次注册**抛错**（防重复注入）。
- **禁止重复实现**：不重做 `GameManager.ensure` / `ConfigManager.getInstance` 的单例逻辑。

## 允许修改范围
- 新增 `assets/scripts/core/GameContext.ts`

## 禁止修改范围
- `assets/scripts/**` 除本文件外的全部；尤其 `config/**` `battle/**` `dungeon/**` `ui/**` `scene/**` `render/**` `assets/**`(服务) `app/**` `run/**` `utils/**`
