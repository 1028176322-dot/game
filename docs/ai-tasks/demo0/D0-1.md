# D0-1 — 实现 ILifecycle + LifecycleManager

> 任务类型: 基础设施 / 生命周期 ｜ 依赖: D0-2（GameContext 类型）｜ Token Budget: 1 文件

## 输入
- `docs/2D转3D全面升级方案.md §5.1` 接口 `ILifecycle` 与 `LifecycleManager` 签名。
- `图6.3` 生命周期状态图（initialize→enter→exit→pause→resume→destroy 六态）。
- `docs/ai-tasks/_architecture_report.md §6/§7`：确认无既有 LifecycleManager（安全新建）。

## 输出
- 新增 `assets/scripts/core/LifecycleManager.ts`，导出：
  - `interface ILifecycle`：`initialize(ctx: GameContext): void` / `enter(): void` / `exit(): void` / `pause(): void` / `resume(): void` / `destroy(): void`
  - `class LifecycleManager`：`register(s: ILifecycle): void` / `enterAll(): void` / `exitAll(): void` / `pauseAll(): void` / `resumeAll(): void` / `destroyAll(): void`（**逆序**销毁）
  - 内部 `private systems: ILifecycle[]`（按注册顺序）

## 严格约束
- 严格按照 §5.1 签名；**不得**新增方法、**不得**改名、**不得**增加职责（不得在此做 DI——DI 归 GameContext；不得在此加载资源）。
- `initialize(ctx)` 的 `ctx` 类型取自 D0-2 的 `GameContext`（同目录 import）。
- `destroyAll()` 必须按 `systems` 逆序遍历调用 `destroy()`（防依赖错乱）。

## 允许修改范围
- 新增 `assets/scripts/core/LifecycleManager.ts`

## 禁止修改范围
- `assets/scripts/**` 除本文件与 `GameContext.ts`（D0-2）外的全部
- 尤其 `battle/**` `dungeon/**` `ui/**` `scene/**` `render/**` `assets/**`(服务) `config/**` `app/**` `run/**` `utils/**`
