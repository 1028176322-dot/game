# D0-5 — GameBootstrap 接入 + 空场景验证

> 任务类型: 集成 / 验证 ｜ 依赖: D0-1~D0-4 全部 ｜ Token Budget: ≤2 文件（改 1 + 可能加测试）

## 输入
- D0-1~D0-4 产出（`GameContext` / `LifecycleManager` / `ConfigDatabase` / `Logger`）。
- `assets/scripts/core/GameBootstrap.ts` 现有 `startup()` 流程（`ConfigService.instance.loadAll()` → `ConfigManager.getInstance().loadAll()` → `AssetBundleService.instance.loadAssetMapFromResources()`）。
- `docs/ai-tasks/_architecture_report.md §5`：`GameBootstrap.startup()` 现有三步 load 不得破坏。

## 输出
- 修改 `assets/scripts/core/GameBootstrap.ts`：
  - 在 `startup()` 中**追加**（不改既有三步 load）：创建 `GameContext`，`register(ILogger, new Logger())`、`register(IConfigDatabase, new ConfigDatabase())`；创建 `LifecycleManager`；若 `ConfigDatabase` 实现 `ILifecycle` 则 `register` 进 LifecycleManager；调用 `lifecycle.enterAll()` 演示顺序；在 `onDestroy` 钩子中调用 `ctx.onDestroy()`。
  - 用 `Logger.channel("battle").info(...)` 打印生命周期链路，证明注入+广播打通。

## 严格约束
- 不得改动既有 `loadAll` 流程与 `goToMain`；仅**追加** GameContext 接入代码。
- `GameBootstrap` 仅可 `new` 四大基础设施（属允许），**不得** `new` 业务 System（红线 4）。
- 若 `ConfigDatabase` 不实现 `ILifecycle`，则不强制注册进 `LifecycleManager`（避免伪造生命周期）。

## 允许修改范围
- 修改 `assets/scripts/core/GameBootstrap.ts`（仅 `startup()` 追加 + `onDestroy` 钩子）

## 禁止修改范围
- `assets/scripts/**` 除 `GameBootstrap.ts` 与四大基础设施文件外的全部；尤其 `battle/**` `dungeon/**` `ui/**` `scene/**` `render/**` `assets/**`(服务) `config/**` `app/**` `run/**` `utils/**`

## 完成定义 (DoD)
- [ ] 运行空场景后，控制台按序打印 `Initialize → Enter → Pause → Resume → Exit → Destroy`（经 Logger，证明链路打通）
- [ ] `npm run test` 全绿
- [ ] `npm run validate:all` 9 门禁通过
