## 诊断记录：只显示背景、无主角/怪物时 7 条排查链

来源：用户分析。

### 排障顺序
1. **层级覆盖** — 背景 siblingIndex 比角色高，盖住一切。
   → 统一层级：BackgroundLayer(0) / TileLayer(1) / ActorLayer(2) / EffectLayer(3) / UILayer(4)，用 `ensureLayer()` 保证。
2. **坐标体系不一致** — 背景用 Canvas 坐标，角色用 Grid 坐标，跑到屏幕外。
   → 所有战斗单位走 `gridToWorld()` 统一转换。
3. **BattleManager 未真正 startBattle** — 背景是 DungeonSceneInstaller 加载，但战斗流程没触发。
   → 加启动链日志：`refs ready` / `dungeon generated` / `startBattle called` / `spawn monsters`。
4. **节点生成了但 SpriteFrame 为空** — 资源 ID 不匹配导致透明/空白。
   → `applyMonsterSprite()` 返回 null 时 warn 打印 `zoneId/monsterId/path`。
5. **背景太大挤没世界内容** — 背景铺满设计分辨率，角色层独立居中。
   → `fitBackground(bgNode)` 设尺寸/位置/siblingIndex，角色不要挂背景子节点。
6. **节点 active=false 或 scale=0** — 从 prefab/模板创建常见。
   → 生成后统一 `normalizeRenderableNode()` 确保 active/scale/transform。
7. **Camera cullingMask 没包含角色层** — 背景在 UI_2D，角色在 DEFAULT。
   → `setLayerRecursive(node, Layers.Enum.UI_2D)` 统一。
