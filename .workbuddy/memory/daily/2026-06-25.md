## 15:16 - 场景文件修复 (cameraPriority null 错误)

**问题**：清理 `library/` 和 `temp/` 后，现有场景文件的 UUID 格式 `_id` 与缓存关联断裂，`pointer-event-dispatcher.ts:220` 报 `cameraPriority null`

**根因**：旧场景文件使用 UUID 字符串作为节点 `_id`（如 `"c0y6F5f+pAvI805TdmxIjx"`），这些 UUID 在首次导入时生成并缓存到 library/。清理缓存后，引擎无法解析这些 UUID 引用的组件元数据。

**解决方案**：
1. 研究了 Cocos Creator 3.x UUID 压缩算法（23字符格式），实现 `compress_uuid()` 
2. 编写 `tools/regenerate_scenes.py`：使用整数格式 `_id`（与工作模板一致）+ **移除 `_globals` 字段**（包含已丢失的 skybox 资源引用）
3. 三个场景文件均已重新生成并验证：无 `_globals`，结构完整
4. 备份保留为 `*.scene.bak`（`splash_template.scene` 已删除）

**使用步骤**（在 Cocos Creator 编辑器中）：
1. 如果编辑器已打开 → **重启编辑器**（让引擎检测文件变更并重新导入）
2. 等待资源导入完成（观察底部进度条）
3. 对着 splash.scene 右键 → 点击 Play（运行）
4. 如果仍然报错，尝试 `开发者 → 清除所有缓存` 再重启
