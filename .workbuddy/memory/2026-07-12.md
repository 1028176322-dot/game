### 核查结论（探查代理 + 硬指标确认）
- **存在两种绑定风格，需区分**：

  - 新增 `pbrRoughness` 尝试键；
  - 打印 `touched materials: N renderers: M` 便于诊断。
  - `_autoFitScale` 增加 `MeshRenderer` 作为 `SkinnedMeshRenderer` 的 fallback（按 `getComponentInChildren` / `getComponent` 两级）。

## 23:06 修复自发光材质赋值：Color.fromHEX 类型崩溃

- **现象**：用户截图显示创建界面仍无模型；console 报错：
  - `Cannot create property 'r' on string '#3a3a3a'`
  - 伴随 `PromiseRejectionEvent`
- **根因**：`_makeMaterialsEmissive` 中对 `emissive` 传递了 `Color.fromHEX('#3a3a3a')`（Color 对象），但 Cocos 3.8 的 PBR material 对 `emissive` 属性期望的是 `Vec3`（或类型转换失败），导致内部 setter 把 Color 当字符串处理并崩溃；`PromiseRejectionEvent` 是 Sentry 对同一抛错的包装。
- **修复**：`assets/scripts/render/CharacterModelAssembler.ts`
  - 把 `emissive` 从 `Color.fromHEX('#3a3a3a')` 改为 `new Vec3(0.23, 0.23, 0.23)`（0-1 RGB）。
  - 把 `emissiveScale` 从 `0.45` 改为 `0.5`。
  - 将 `trySet` 改为独立的 `setMatProp` helper，并只保留 `setProperty` + 直接属性赋值两条路径，每处都独立 `try/catch`。
- **验证**：`check_ts_static.py` All static checks passed ✅；`encoding_audit --ci` issues=0/p0=0 ✅。
- **用户侧动作**：停止预览 → 重新运行预览 → 进入创建冒险者界面。预期这次能看到 3D archer 模型；如果仍看不到，请贴 `touched materials: N renderers: M` 的 N 值。
