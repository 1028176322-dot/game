# 启动屏设计方案

> **版本**: v1.0 | **日期**: 2026-06-24
> **用途**: 微信小游戏审核强制要求 | **目标平台**: 微信小游戏
> **包体约束**: 单张图 < 100KB | **尺寸**: 1280×720px（16:9）

---

## 1. 微信启动屏规范

| 要求 | 说明 |
|------|------|
| 展示时长 | 至少 2 秒（可配置 2~5 秒）|
| 尺寸 | 1280×720px（推荐），或 750×1334px（竖版）|
| 文件格式 | PNG / JPG |
| 文件大小 | < 200KB（本项目目标 < 100KB）|
| 显示内容 | 游戏名称 + 公司/团队名称 + 版本号 |
| 禁止内容 | 诱导点击、虚假信息、第三方平台 LOGO |
| 关闭方式 | 点击任意位置可跳过（需代码实现）|

## 3. 实现方案

### 3.1 Cocos Creator 实现

```
1. 在 Cocos Creator 中创建启动场景 (splash.scene)
2. 添加 Sprite 组件用于显示启动图
3. 添加点击事件监听：点击任意位置 → 跳转主场景 (main.scene)
4. 自动计时 2 秒后允许跳过（不等待点击也可自动跳转）
5. 记录启动次数，首次启动显示 3 秒，后续 2 秒

启动场景生命周期:
  onLoad → 加载启动图 → 显示 2 秒 → 可跳过 → 加载主场景
                                           ↓
                                    加载完成后跳转
```

### 3.2 微信小游戏适配

```
// app.js / game.js 中的启动流程:

1. wx.showLoading({ title: '加载中...' })
2. 检查是否有缓存的启动图资源
   - 有 → 直接显示
   - 无 → 用本地默认图（内置在主包中，< 50KB）
3. 同时后台加载主场景资源
4. 启动图显示 ≥ 2 秒后允许跳过
5. 主场景资源加载完成 → 自动跳转
6. 隐藏 loading
```

### 3.3 包体优化

| 策略 | 效果 |
|------|------|
| 启动图内置在主包而非远程加载 | 避免网络延迟导致启动慢 |
| 使用 256 色 PNG 而非全彩 | 文件缩小 40% |
| 装饰插图为单色剪影 | 文件大小 < 30KB |
| 启动场景不加载额外脚本 | 减少首帧加载时间 |

## 4. 启动屏状态矩阵

| 状态 | 行为 | 用户感知 |
|------|------|----------|
| 首次启动 | 显示 3 秒 + 自动 + 点击跳过 | "加载中…" |
| 二次启动 | 显示 2 秒 + 点击跳过 | 快速进入 |
| 资源加载慢 | 启动图保持直到加载完成 | 启动图不消失 |
| 资源加载失败 | 启动图显示 5 秒后弹出"重试"按钮 | 点击重试 |
| 点击跳过 | 立即跳转主场景（如果资源已加载）| 无缝过渡 |
