## 四、遗留问题与注意事項

### 4.1 需要处理的���题

| 问题 | 影响 | 建议处理方式 |
|------|------|-------------|
| 怪物只有 idle 帧 | 缺少 attack/hit/death 动画 | 后续按需补帧（可先只用一个状态图）|
| 终结Boss文件名是通用名 | `boss_guardian_forest` 应为 `boss_forestGuardian` | 需批量重命名映射到配置表名称 |
| 已有资源来自两次AI生成 | 两批资源的画风可能不一致 | playtest 时观察视觉一致性 |
| 事件房背景缺失 (6 张) | 事件房进入后无背景图 | 优先级较低, 可先用战斗房背景代用 |
| 无音效/音乐 | 游戏全程静音 | 上线前必须补齐, 否则玩家体验极差 |

### 4.2 风格一致性评估

| 维度 | 评估 | 说明 |
|------|------|------|
| 角色画风 | ⚠️ 基本一致 | 5 个角色均为像素风, 但细节精度有差异 |
| 怪物画风 | ⚠️ 基本一致 | 36 个怪物色调分区正确, 像素精度有差异 |
| UI 风格 | ✅ 一致 | 5 个通用 UI 组件的色值统一 |
| 色调分区 | ✅ 正确 | 6 区域色调符合设计规格 |
| 文件命名 | ✅ 合规 | 已全部转小写, 无中文无空格 |
| 尺寸规范 | ⚠️ 少数异常 | 4 个怪物为 64×64 而非 48×48 (voidRift/deerElite/等) |

### 4.3 优先级建议

```
第 1 优先级 (上线拦路虎):
  ① 音效/音乐 (47 个) — 没有声音, 游戏体验崩塌
  ② 战斗 HUD (9 个) — 没有血条/技能按钮, 无法操作
  ③ 强化房 UI (36 个) — 核心肉鸽交互

第 2 优先级 (内容完整性):
  ④ 图标 (80 个) — 物品/技能无图标, 玩家看不懂
  ⑤ 特效 (47 个) — 战斗/元素反应无反馈

第 3 优先级 (锦上添花):
  ⑥ 场景背景 (12 个) — 可用单色背景兜底
  ⑦ 字体 (2 个) — 可用系统字体兜底
  ⑧ 事件房背景 (6 个) — 可用战斗房背景代用
```
